# Constitutional repository coherence orientation 001

Repository authority wins.

This orientation does not modify code, the Book, graph artifacts, or tests. It asks the current Book, question graph, implementation, tests, reports, and history what constitutional questions their combined coherence now makes unavoidable.

## Method boundary

The repository generated candidate questions through recurring evidence shapes rather than preferred topology:

- Book clauses were read for preserved standing and negative authority.
- The current question graph was read for live relations, disconnected shores, refusal edges, and Unknown neighbors.
- Circulation and cache current-state artifacts were read for implementation-backed handoffs and remaining realization boundaries.
- Characterization and topology reports were read for recurring role, road, lens, occurrence, and standing distinctions.
- Tests and runtime anchors were treated as implementation evidence only where reports or Book anchors identify them as current witnesses.

Classification vocabulary used below:

```text
missing implementation
!= disconnected implemented shores
!= under-projected constitutional relation
!= unresolved topology
!= unsupported expectation
```

## Candidate question A — When may an admitted inquiry become a bounded question artifact rather than merely correspond to one?

### Repository-generated question

What constitutional act, owner, and evidence boundary allow admitted inquiry movement, bounded inquiry frontier standing, or an Inquiry Need to become an individual bounded question artifact without collapsing inquiry standing into question standing?

### Evidence causing the repository to ask it

The Book now preserves that question-shaped text is not automatically inquiry, and that a bounded question carries identity, provenance, scope, evidence demand, and lawful stop conditions without answer, outcome, unbounded investigation, or repository-knowledge promotion. It also preserves that inquiry origination requires bounded translation and that question graph relations are plural and non-collapsing.

The current graph shows several neighboring shores that almost meet but are not lawfully connected:

- Addressable pressure has no current direct implementation bridge to Inquiry Need.
- Inquiry Need can reach a bounded inquiry frontier through selected need and coherent boundary testimony, but it is stopped from becoming a formed question, inquiry opening, observation authorization, action, recording, or mutation.
- Admitted inquiry movement has only a correspondence or unrealized handoff to an individual bounded question unless separate admission-to-artifact evidence exists.
- The bounded inquiry frontier explicitly lacks an individual bounded question formulation bridge.

The circulation current state reports a tested selected path through bounded constitutional question production, projection, view selection, composition request, composition result, and rendering, but the same table marks generic operator-ingress parsing, general natural-language translation/admission, universal admission owner, semantic inquiry admission, and general examination selector as missing realization.

History reinforces the pressure: recent commits recovered the constitutional question graph, corrected evidence-fidelity overreach, added terrain and connection surveys, and projected only one plural non-collapsing question-graph clause rather than a lifecycle engine.

### Classification

`disconnected implemented shores` plus `unresolved topology`.

This is not merely missing implementation. The repository already has bounded-question artifacts and tested downstream consumers, and it also has Inquiry Need/frontier machinery. The unresolved issue is the constitutional relation and owner that would connect these shores without treating any adjacent standing as automatic question formulation.

## Candidate question B — Which consumer-side acceptances are lawful roads, and which are only lenses, renderings, or call adjacency?

### Repository-generated question

When may a producer output be consumed as a constitutional road by a later boundary, and what consumer-side validation, scope, and standing effect must be present before that relation is named more strongly than lens, representation, or adjacency?

### Evidence causing the repository to ask it

The characterization pass distinguishes act, constraint, testimony, artifact, constructor, lens, and road. It says a road is a producer assertion accepted by a consumer for a declared purpose and requires compatible identity, provenance, invariants, applicability, and consumer acceptance; adjacency and constructibility are insufficient. It leaves unresolved which producer-consumer pairs beyond inspected anchors have sufficient consumer-side validation to be named constitutional roads.

The Book's question graph clause now states that a relation exists only where source standing, target standing, crossing evidence or warrant, authority boundary, and surviving limits are preserved for that local relation. It also denies automatic carrying of stronger standings such as answer, sufficiency, completion, retirement, receipt, reliance, correction, reopening, execution authority, mutation authority, or ontology closure.

Selection topology repeats the same pressure locally: constitutional view selection gives registered view names representation-standing for composition and is bounded to that downstream purpose; reuse by another consumer would require that consumer to accept registered view names selected by exact-key comparison. It also distinguishes representation selection from standing-bearing consideration selection.

The circulation realization table identifies tested handoffs for one selected path but leaves missing realization for a delivery record, receipt proof, reliance proof, response-to-new-pressure owner, correction, supersession, reopening, and response admission authority. This means multiple artifacts are renderable or adjacent to consumers, but the repository refuses to infer uptake or road standing from availability.

### Classification

`under-projected constitutional relation` plus `unresolved topology`.

The recurring relation exists in Book doctrine, reports, and selected implementation witnesses, but the general constitutional test for road naming is not fully projected or inventoried. This is not unsupported expectation, because reports repeatedly identify live producer-consumer pressures; it is also not only missing implementation, because some roads are implemented and tested while their general relation remains bounded and plural.

## Candidate question C — Must standing-changing read-only acts be recorded, or can validated non-mutating establishment suffice?

### Repository-generated question

Which constitutional changes of standing require event-ledger occurrence evidence, and which may be lawfully established by validated read-only boundaries that preserve `mutates_cluster=false` and do not mutate cluster truth?

### Evidence causing the repository to ask it

The characterization pass leaves unresolved whether every constitutional change of standing must have a recorded event, or whether some validated read-only establishment boundaries are sufficient without ledger recording. It also leaves unresolved whether act should remain broad enough to include read-only bounded selection and establishment, or whether occurrence-without-mutation needs a distinct durable kind.

The repository's operational instructions preserve the same boundary for diagnostics: diagnostic findings should prefer `diagnostic_run:<id>` scoped subjects, distinguish event-ledger writes from cluster mutation, and preserve `mutates_cluster=false` for read-only diagnostics. That is not Book doctrine by itself, but it is current repository authority for operational surfaces.

The circulation realization table repeatedly treats bounded-question production, projection, selection, composition request, composition result, rendering, and cache admission as read-only or non-mutating where tested, while the current state also says standing changes are local and act-bound. The cache current-state update further narrows SQLite cache standing to disposable read-model requirements, rebuild-from-ledger authority, and `mutates_cluster=false` read-only admission; a cache hit, miss, reset, or rebuild is not event history and does not append ledger events or mutate the cluster.

### Classification

`under-projected constitutional relation` plus `unsupported expectation` boundary.

The unsupported expectation would be either universal recording for every standing change or universal sufficiency of read-only validation. Repository evidence supports neither universal rule. What is unavoidable is the need to distinguish occurrence evidence, preservation, read-only establishment, event-ledger writes, and cluster mutation at the constitutional boundary.

## Candidate question D — What proof, owner, and authority convert delivery or operator response into receipt, reliance, correction, supersession, reopening, or new admitted movement?

### Repository-generated question

What separate evidence and authority allow outward representation or delivery to become recipient uptake, and what separate boundary allows operator response to correct, supersede, reopen, or admit new movement rather than merely re-enter as source-attributed pressure?

### Evidence causing the repository to ask it

The circulation current state explicitly says bounded results may become outward representations, delivery creates only delivered or available representation standing, and operator response may re-enter only as new pressure. Receipt, understanding, reliance, action authority, responsibility transfer, correction, supersession, reopening, mutation, and completion require separate evidence and authority.

The question graph repeats the same negative edges: delivery or availability does not prove receipt, understanding, or reliance; operator response does not correct, supersede, reopen, or mutate a prior artifact absent separate authority; completion does not imply retirement.

The frontier records Seed Egress and recipient re-entry as unresolved, including proof mechanics for receipt, understanding, reliance, correction/reopening owner maps, response taxonomy, durable records, and complete neighboring-inquiry topology.

### Classification

`unresolved topology` with some `missing implementation`.

The constitutional negative boundary is already strong. The unresolved part is not whether delivery implies reliance; it does not. The unavoidable question is what positive proof and owner topology would be required when the repository does need to recognize uptake, correction, supersession, reopening, or new admitted movement.

## Candidate question E — Which standing vocabularies should become durable constitutional coordinates, and which remain local posture strings or presentation labels?

### Repository-generated question

When recurring standing words, Unknown kinds, diagnostic statuses, or implementation/report names recur across Book, graph, tests, and reports, what evidence decides whether they become durable constitutional coordinates rather than local posture strings, implementation evidence names, or presentation vocabulary?

### Evidence causing the repository to ask it

The question graph evidence-fidelity correction restores literal evidence names such as `QuestionFamily`, `Question Shape`, and `Survey Warrant Family` only where they identify implementation or report evidence, while preserving Book-facing terms as non-canonical. The Book itself says question forms, methodological warrants, public answer families, and individual bounded questions occupy different resolutions and do not automatically admit, select, dispatch, examine, answer, complete, retire, or reopen another.

Repository dimensionality work says standing is a recurrent macro family but not a simple scalar dimension. The strongest local coordinates include standing kind, standing reason, consumer-relative admissibility, reliance permission, movement permission, truth status, verification status, unknown reason, and forbidden inference. It leaves unresolved which standing vocabularies should become durable enums and which remain local posture strings.

SQLite friction work shows the same risk in implementation form: local status strings and `LIKE` matching can preserve Unknown and mismatch distinctions in a bounded witness, but naming-driven classification is fragile and does not prove a universal enum system.

### Classification

`under-projected constitutional relation` plus `unsupported expectation` boundary.

The unsupported expectation is that recurring labels should automatically become repository knowledge or canonical ontology. The equally unsupported opposite is that absence from one vocabulary proves repository absence. The evidence calls for a constitutional promotion boundary for recurring vocabularies.

## Candidate question F — What is the current constitutional status of disposable read-model caches as consumers await production evidence?

### Repository-generated question

How should the repository constitutionally characterize disposable SQLite read-model caches that are implementation-backed, tested for read-only admission and rebuild behavior, but still lack concrete production consumer evidence and complete owner maps?

### Evidence causing the repository to ask it

The cache boundary reality frontier says no Book projection is warranted. The implemented cache boundary is limited to schema validity, cache identity, version and event horizon, payload materialization, rebuild from ledger authority, and read-only admission through `mutates_cluster=false`. Remaining pressure is concrete production consumer evidence, not replacement metadata or generalized cache-family doctrine.

The circulation current state repeats that current projection, summary, and derived-index SQLite cache tables are disposable read models. Incompatible schemas are dropped and recreated rather than interpreted, migrated, or repaired with `ALTER TABLE`; rebuilt projections derive from surviving ledger events, preserve read-model-only cache standing and `mutates_cluster=false`, and append no false events.

### Classification

`missing implementation` for production consumers and owner maps, not `under-projected constitutional relation` yet.

The current relation is already narrowed and implementation-backed. The repository asks this question because consumers are anticipated, but the frontier itself says future Book projection would require realization-neutral constitutional evidence beyond the implemented cache seam.

## Selection posture

Repository coherence does not clearly dominate toward a single candidate.

Candidate A is strong because it joins the newest question-graph recovery, Book projection, existing bounded-question implementation, Inquiry Need/frontier implementation, and repeated Unknown bridges. Candidate B is similarly strong because the same road-versus-lens problem recurs across characterization, selection topology, question graph relations, and circulation realization. Candidate C is also live because read-only standing changes, diagnostic recording boundaries, event-ledger occurrence, and cache read-model admission repeatedly touch the same mutation boundary.

The evidence that would distinguish them is not architectural preference but repository pressure:

- Candidate A would dominate if the next reports/tests/history show multiple consumers awaiting a lawful Inquiry Need/frontier/admitted-movement to bounded-question artifact bridge.
- Candidate B would dominate if more producer-consumer pairs fail because consumer-side acceptance is undefined or over-inferred from adjacency.
- Candidate C would dominate if tests or diagnostics expose inconsistent treatment of read-only establishment, event recording, and cluster mutation.
- Candidate D would dominate if egress, delivery, response, correction, or reopening surfaces become operationally present.
- Candidate E would dominate if recurring posture strings or evidence names begin driving cross-surface behavior without a promotion boundary.
- Candidate F would dominate only if production consumers begin relying on disposable caches in ways not covered by current cache identity, horizon, payload, rebuild, and read-only checks.

Plural orientation should therefore be preserved. The currently unavoidable constitutional field is the relation among local standing changes, lawful consumer acceptance, and preservation of inherited limits across adjacent but non-collapsible shores.

Repository coherence orientation 001 complete.
