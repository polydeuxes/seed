## Candidate inspection discipline

Each recurring candidate is inspected independently before comparison. The inspection asks only:

```text
What artifact is observed?
Who emits it?
Who receives it?
What lawful transition enables it?
Why is it an observation?
Why is it not merely presence?
Why is it not evidence?
Why is it not history?
Why is it not projection?
Could it lawfully be the organism's first observation?
```

## Candidate 1: operator testimony

### What artifact is observed?

Operator testimony is observed as operator-supplied prose, pressure, prompt content, runtime user input, or an inquiry note depending on the implemented path that receives it.

### Who emits it?

The operator emits it.

### Who receives it?

A bounded local responsibility may receive it: runtime input handling, inquiry-note preservation, decision input composition, or another implemented intake path. Current evidence does not recover one universal receiver.

### What lawful transition enables it?

The lawful transition is not truth acquisition. It is the bounded transition from Null / no present representation to externally supplied material being present to an existing local responsibility.

### Why is it an observation?

Operator testimony can become observation-like only in the weak sense that Seed may have a bounded record that the operator said or supplied something. If a competent local surface records an event or note, Seed can observe the existence of the testimony as testimony.

### Why is it not merely presence?

It is more than mere presence only when a local surface preserves a record with inspectable content, provenance, identity, or event/note shape. Without preservation, it remains transient presentation or external occurrence.

### Why is it not evidence?

It is not automatically evidence for the truth of what it says. Repository evidence repeatedly separates operator concern, testimony, pressure, and prose from fact, verification, authority, implementation readiness, and mutation. At most, it can be evidence that an operator utterance or pressure exists.

### Why is it not history?

It is history only if recorded in a history-bearing surface. Inquiry-note preservation is explicitly outside event-ledger runtime state projection in the reviewed evidence. Runtime `input.user_message` can be append-only event history if that path records it, but operator testimony as a general candidate is not itself constitutional history.

### Why is it not projection?

Projection is a derived view over preserved state or event history. Operator testimony is source-side material or preserved input; a projection may later include or summarize it, but the testimony is not the projection.

### Could it lawfully be the organism's first observation?

Only for a specific path where the first locally preserved record is a bounded observation that the operator supplied testimony. Recurring evidence does not support operator testimony itself as the unique first constitutional observation. It survives only as a path-specific possible first observed artifact: `operator supplied this bounded representation`, not `the testimony content is true`.

## Candidate 2: bounded external representation

### What artifact is observed?

A bounded external representation is observed as a locally bounded representation: a preserved inquiry note, runtime input event, `InputArtifact`, provider-local accepted input, or other local intake artifact.

### Who emits it?

An external participant, provider, file, runtime caller, operator, document source, tool result, platform source, or source adapter may emit or supply the representation. The emitter is path-specific.

### Who receives it?

An existing local responsibility receives it: inquiry orientation, runtime input handling, input inspection, provider-local intake, observation import validation, or observation source collection. The repository does not recover a universal external-representation subsystem.

### What lawful transition enables it?

The lawful transition is:

```text
Null / unknown / uncommitted
→ bounded external representation exists for an already implemented local responsibility
→ that responsibility may preserve, inspect, classify, orient, translate, or stop only within its authority
```

### Why is it an observation?

It is an observation only when the receiving responsibility can truthfully preserve or report that the representation exists with bounded content/provenance. The observed object is not the represented claim's truth; it is the representation's bounded presence to Seed.

### Why is it not merely presence?

It is not merely presence when it crosses into a repository-visible local artifact or record: inquiry note, input event, file artifact, accepted provider shape, or validated observation object. The repository can inspect that artifact later or use it as bounded orientation.

### Why is it not evidence?

It may become evidence for the fact that a representation was present, but it is not automatically evidence for the external claim's correctness, semantic interpretation, ownership, responsibility, or actionability. Evidence-bearing status depends on local admission rules and provenance.

### Why is it not history?

Bounded representation is prior to, or separate from, history unless recorded in a history-bearing surface. A preserved inquiry note is not event-ledger history; a runtime input event can be event history; an `InputArtifact` may be audit metadata. Therefore representation and history cannot be collapsed.

### Why is it not projection?

The representation is source or intake material. Projection is derived from event history, facts, or preserved state. A representation may feed a later projection only through implemented local rules.

### Could it lawfully be the organism's first observation?

Yes, but only in a narrowed characterization: the first constitutional observation could be Seed observing a bounded external representation as present to a competent local responsibility. However, recurring evidence has not recovered one universal artifact, emitter, receiver, or event kind. Therefore this candidate survives as the recurring observation grammar, not as a unique first observation instance.

## Candidate 3: preserved signal

### What artifact is observed?

A preserved signal is a bounded signal-like record: raw note, input event, provider output fragment, file metadata, source sample, or diagnostic/visibility output preserved by a competent surface.

### Who emits it?

The emitter is the source that produced the signal: operator, provider, file, runtime caller, platform API, tool, document, or another local surface.

### Who receives it?

The local preservation or intake responsibility receives it. No universal signal receiver is recovered.

### What lawful transition enables it?

The transition is bounded preservation after Null: a signal is no longer absent or transient because a local authority preserved it without overpromoting it.

### Why is it an observation?

It is an observation when Seed can lawfully say: this signal was received or preserved with this source/path/content/metadata. The observation concerns the signal's occurrence or preservation, not the truth of any represented world claim.

### Why is it not merely presence?

Preservation gives the signal durable or inspectable shape. Mere presence could be unrecorded; preserved signal can be reviewed, bounded, and used for orientation or later evidence assessment.

### Why is it not evidence?

Preserved signal can be evidence, but evidence requires a claim relation and authority. A signal preserved without admission remains signal/preservation, not sufficient evidence for truth, fact, responsibility, or implementation.

### Why is it not history?

Some preserved signals enter append-only history; others, such as inquiry-note probe records, remain outside event-ledger state projection. Preservation alone does not establish constitutional history.

### Why is it not projection?

A preserved signal is input material. Projection is downstream derived state or view over recorded material.

### Could it lawfully be the organism's first observation?

Possibly, if the first lawful local record is best characterized as observing a preserved signal. But `preserved signal` is broader and less exact than the recurring external-representation grammar. It survives as compatible vocabulary, not as the strongest unique answer.

## Candidate 4: first preserved event

### What artifact is observed?

The artifact is the first append-only event lawfully recorded by a competent local responsibility, with event identity, kind, workspace, actor, payload, and provenance/causation fields where applicable.

### Who emits it?

The competent event-producing local responsibility emits it: runtime input handling, observation ingestion, fact/admission surface, or another owner that lawfully appends an event.

### Who receives it?

The append-only event ledger receives it. Later projection, audit, and read-only views may consume it.

### What lawful transition enables it?

The lawful transition is:

```text
no preserved append-only constitutional history
→ first lawful append-only event
→ first preserved constitutional history
```

### Why is it an observation?

It is not necessarily an observation. It may be an input event, observation event, evidence event, fact event, decision event, or other event kind. It can be observed later by a read/projection/inspection surface, but eventness alone is not observation.

### Why is it not merely presence?

An append-only event is durable, identified, replayable, and projection/audit relevant. It is more than source presence.

### Why is it not evidence?

Some events are evidence events; others are not. Event existence can evidence that something was recorded, but it does not automatically evidence the truth of external content or the validity of a claim.

### Why is it not history?

It is history when it is append-only event history. This candidate fails the `not history` distinction because its constitutional identity is primarily first preserved history, not first observation.

### Why is it not projection?

The event is projection input. Projection is derived from replaying or reading event history.

### Could it lawfully be the organism's first observation?

Only if the first event kind is `observation.observed` or otherwise locally characterized as an observation. Recurring evidence explicitly rejects one universal first event kind and says observation is not always first. Therefore first preserved event is rejected as the unique first constitutional observation.

## Candidate 5: observation-level artifact / canonical Observation

### What artifact is observed?

The artifact is a canonical `Observation` or an observation event produced by a valid observation source/import/ingestion contract, with source, subject, predicate, value, confidence, metadata, dimensions, and time/provenance shape where supported.

### Who emits it?

An observation source, observation import path, or provider/source adapter emits it after satisfying its local contract.

### Who receives it?

Observation ingestion or another observation-consuming surface receives it; event ledger and evidence/fact surfaces may receive downstream events if ingestion occurs.

### What lawful transition enables it?

The transition is from accepted provider/source material through local validation into a valid Observation, and possibly from Observation into observation/evidence/fact events according to local ingestion rules.

### Why is it an observation?

This is the most literal observation candidate. It has the repository's canonical observation shape and, when ingested, can create observation/evidence events while preserving provenance and boundaries.

### Why is it not merely presence?

Canonical Observation is structured, validated, source-attributed, and claim-shaped. It is not simply that material arrived.

### Why is it not evidence?

Observation and evidence are separated. Observation can generate evidence or support facts through ingestion, but observation does not equal evidence or fact by identity.

### Why is it not history?

Observation becomes history only when recorded as an append-only event. A constructed Observation object and an event-ledger observation record are related but distinct.

### Why is it not projection?

Observation is input/support material. Projection is derived current state or view over preserved events/facts/evidence.

### Could it lawfully be the organism's first observation?

Yes for paths whose first locally relevant record is a valid Observation. But recurring Null-first evidence shows operator-prose and file-intake paths can preserve bounded representation before canonical Observation, and first-history evidence rejects observation as universal first event. Therefore canonical Observation survives as a path-specific first observation possibility, not as Seed's recurring first constitutional observation.

## Candidate 6: repository-visible evidence

### What artifact is observed?

The artifact is implementation evidence, tests, diagnostics, reports, observations, compatibility behavior, app output, or bounded handoff material visible in the repository.

### Who emits it?

Repository authors, tests, runtime surfaces, diagnostic surfaces, app commands, prior investigations, or implementation modules emit it.

### Who receives it?

A recovery investigation, authority evaluation, audit, or local admission boundary receives it.

### What lawful transition enables it?

The lawful transition is evidence-gated movement: evidence can move a bounded subject from Null / unsupported / candidate toward supported local status only through a competent boundary.

### Why is it an observation?

Repository-visible evidence may be observed by investigators or surfaces. It can also include observations. But evidence as a category is not the same as observation.

### Why is it not merely presence?

Evidence is more than presence when it supports a claim under provenance and authority. Mere repository text or output is not evidence until a bounded claim and authority relation are identified.

### Why is it not evidence?

This candidate is evidence by identity when properly admitted, so it fails the requested distinction. It is not the first observation; it is the material that can support or reject a first-observation characterization.

### Why is it not history?

Evidence may be historical or current, preserved or generated. It is not automatically append-only constitutional history.

### Why is it not projection?

Evidence may feed projection, but projection is a derived current view or state. Repository-visible evidence is support material.

### Could it lawfully be the organism's first observation?

No as a unique answer. Repository-visible evidence is the authority used by this characterization, not the organism's first constitutional observation. It is rejected as category confusion.

## Comparison

### Surviving candidates

1. **Bounded external representation observed as present to a competent local responsibility** survives as the strongest recurring grammar for first constitutional observation. It survives only with strict limits: no universal artifact, no universal receiver, no truth of represented content, no automatic observation event, no automatic evidence/fact, no attachment, no inquiry graph, no runtime sequence.
2. **Preserved signal** survives as compatible but weaker vocabulary. It points to the same boundary when phrased as a locally preserved signal whose existence can be observed, but it is less precise than bounded external representation.
3. **Operator testimony** survives only as a path-specific instance of bounded external representation: Seed may first observe that operator testimony was supplied or preserved, not that the testimony is true.
4. **Canonical Observation** survives only for paths where a valid observation-source contract emits an Observation before any other locally relevant preserved artifact. It does not survive as universal.

### Rejected candidates

1. **First preserved event** is rejected as the unique first observation because recurring evidence says no universal first event kind is supported and observation is not always first. Its strongest identity is first preserved constitutional history when an append-only event is recorded.
2. **Repository-visible evidence** is rejected as the first observation because it is the authority used for characterization, not the organism's own first observation. It may include observations but is not identical to observation.
3. **Operator testimony as truth** is rejected. Testimony may orient inquiry or be preserved as testimony, but it does not become truth, fact, implementation, authority, or mutation by being supplied.
4. **Bounded external representation as a universal object or subsystem** is rejected. The repository supports distributed local artifacts and responsibilities, not a central receiver.

## Recurring observation grammar

The recurring grammar recovered is:

```text
bounded source/emitter
+ bounded external representation or signal
+ competent local receiver
+ preservation / inspection / validation authority
+ provenance or source/path/content limits where available
+ negative authority over truth, fact, semantic interpretation, ownership, runtime action, and promotion
=> Seed may lawfully observe that the representation/signal is present or preserved,
   or may stop if the local boundary does not support even that much.
```

More compactly:

```text
Seed's earliest recurring observation grammar is observation of bounded presence,
not observation of truth.
```

The observed artifact is the bounded representation as received/preserved/validated by a local responsibility. The observation does not automatically concern the represented world claim.

## Recurring observation boundaries

- **Presence is not truth.** A representation can be present without its content being true.
- **Presence is not canonical Observation.** Some paths preserve notes, input events, or input artifacts before any Observation object exists.
- **Observation is not evidence.** Observation can support evidence creation but is not identical to evidence.
- **Evidence is not fact.** Evidence may support local movement only through competent boundaries.
- **Event history is not observation by default.** A first event may be input, observation, evidence, fact, or another kind.
- **History is not projection.** Append-only event history feeds projection; projection is derived view/state.
- **Projection is not verification.** Projected current state or view is not truth by itself.
- **Operator testimony is not repository knowledge.** It can be preserved as testimony or pressure, but not promoted by authorship.
- **Local receiver is not universal owner.** Inquiry orientation, runtime input, input inspection, provider translation, and observation sources have different authorities.
- **Lawful stop remains valid.** If a receiver cannot support observation, the repository must preserve unknown/non-promotion rather than inventing a first observation.

## Preserved unknowns

- The repository has not recovered one universal first constitutional observation instance.
- The repository has not recovered one universal first event kind.
- The repository has not recovered one universal receiver, intake owner, or constitutional observation owner.
- The repository has not proven that every newly created Seed must first encounter operator testimony, file input, runtime input, provider output, or canonical Observation.
- The repository has not determined whether future evidence may ratify a specific first observation for a specific app-created Seed path.
- The repository has not determined whether inquiry-note preservation should ever become constitutional history; current evidence keeps it outside event-ledger projection.
- The repository has not determined whether `bounded external representation` should become implementation vocabulary; current evidence supports it as characterization grammar over distributed local artifacts.
- The repository has not determined a startup, begin, lifecycle, boot, scheduler, or universal observation process.

## Confidence

**High confidence** that recurring evidence rejects canonical Observation, first preserved event, operator testimony, repository-visible evidence, projection, history, or truth as the unique universal first constitutional observation.

**High confidence** that recurring evidence supports bounded external representation / preserved signal becoming present to a competent local responsibility before stronger claims in some paths.

**Medium-high confidence** that the recurring first-observation grammar is `observation of bounded presence, not observation of truth`.

**Medium confidence** that `bounded external representation` is the best compact vocabulary, because it is strongly supported as distributed characterization but not as one implementation primitive.

**Low confidence** for any unique first observation instance, universal receiver, universal event kind, or universal sequence.
