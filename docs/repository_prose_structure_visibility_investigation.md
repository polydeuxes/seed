# Repository Prose Structure Visibility Investigation

## Purpose and boundary

This investigation asks whether repository prose observation can begin with structural visibility rather than semantic interpretation.

This is investigation only.

It does not implement prose ingestion, NLP, meaning extraction, shape discovery, ontology work, workflow systems, or automation.

Repository authority remains repository-visible document structure and existing repository observation philosophy.

## Central finding

Repository evidence supports a conservative starting position:

```text
prose
    -> observable structure
```

before:

```text
prose
    -> meaning
```

Repository-visible structure already exists in documentation.

That structure appears observable without requiring semantic interpretation.

The strongest supported conclusion is:

```text
document structure
is repository-visible
```

while:

```text
document meaning
```

remains outside the scope of this investigation.

## Deterministic structure versus semantic interpretation

Repository evidence supports a meaningful distinction.

### Structure

Questions such as:

```text
is yaml present?

how many headings exist?

what heading levels exist?

how many sections exist?
```

can be answered deterministically.

### Interpretation

Questions such as:

```text
what does this document mean?

what claim is being made?

what concepts are present?
```

require semantic interpretation.

The former appears repository-visible.

The latter does not.

## Would structure-only visibility be useful?

Repository evidence partially supports yes.

Reason:

Visibility can exist independently of interpretation.

Examples:

```text
classification coverage
```

provides useful visibility before deeper interpretation.

Similarly:

```text
structure coverage
```

may provide visibility before prose interpretation.

Potential value:

```text
coverage

consistency

inventory

navigation
```

without requiring semantic understanding.

## Structure-first observation and repository philosophy

Repository evidence strongly supports compatibility.

Existing repository observation frequently follows:

```text
observe first

accumulate

evaluate later
```

Examples:

```text
host observations

filesystem observations

capability observations

classification observations
```

The observation itself does not require final interpretation.

Structure-first prose observation follows the same pattern.

```text
observe structure

accumulate visibility

evaluate later
```

This resembles existing repository behavior.

## Counterexamples

### Structure with limited value

Some structural observations may provide little value.

Examples:

```text
exact word count

exact line count
```

Visibility exists.

Utility may be limited.

Therefore:

```text
observable
    != useful
```

### Structure that pressures interpretation

Some structures may appear simple but implicitly invite semantic analysis.

Examples:

```text
heading names

yaml field names
```

The structure itself is observable.

Meaning remains a separate step.

### Structure without consistency

Different documentation styles may reduce comparability.

Examples:

```text
custom headings

missing metadata

non-standard layouts
```

Visibility remains possible.

Interpretation becomes harder.

## Relationship to earlier findings

The recurrence-first and observation-first investigations suggested:

```text
observation
    -> accumulation
    -> evaluation
```

Structure-first prose observation appears compatible with that pattern.

The observation remains:

```text
structure visible
```

Evaluation may occur later.

No semantic interpretation is required for the observation itself.

## Supported conclusions

1. Repository documents already contain observable structure.
2. Structural visibility appears possible without semantic interpretation.
3. YAML presence, metadata presence, heading hierarchy, and section structure are repository-visible observations.
4. Useful visibility can be obtained from coverage, consistency, inventory, and navigation information.
5. Structure-first observation is compatible with existing repository observation philosophy.
6. Observation does not require immediate interpretation.
7. Structural visibility is a weaker and more conservative claim than meaning extraction.

## Unsupported conclusions

- Structure alone reveals document meaning.
- Structural visibility requires NLP.
- Structural visibility requires ontology promotion.
- Structural visibility requires shape discovery.
- Structural visibility automatically produces useful insights.
- Heading structure alone determines document intent.
- Metadata presence determines document correctness.
