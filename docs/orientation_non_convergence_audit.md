# Orientation Non-Convergence Audit

## Purpose

This audit investigates where repository work fails to support the orientation
interpretation, what evidence remains better explained without orientation, and
whether orientation is only compression over existing repository concepts.

This is an audit only. It is not a reconciliation, frontier, implementation
proposal, ontology proposal, identity proposal, agency proposal, governance
proposal, interface redesign, policy proposal, or remediation plan. It does not
defend orientation and does not attack orientation. Repository authority wins
over this audit. Referenced documents retain authority for their scoped claims.

## Method

The review treated named documents as starting points, not a closed corpus. It
searched repository maps, indexes, adjacent observations, frontiers,
reconciliations, authority documents, runtime surfaces, and tests for evidence
that weakens, contradicts, duplicates, or fails to require the orientation-object
interpretation.

## Executive Findings

1. **Orientation does not converge as an implemented runtime primitive.** Runtime
   surfaces are organized around facts, subject/predicate/value support,
   contradictions, context selection, source navigation, explanation, and state
   summaries. They do not require an orientation object to function.
2. **Orientation is strongest as handoff and understanding prose.** The strongest
   support appears in documents about current work position, active edge,
   preservation failure, working-state activation, and operator understanding.
   That evidence shows a need to preserve continuation meaning, but not
   necessarily a distinct orientation object.
3. **Reference point remains a competing explanation.** Many examples explained
   as orientation can be explained as significance relative to operator, entity,
   current concern, continuity concern, repository, source surface, or authority
   boundary.
4. **Selection-first explanations are stronger in many documents.** The chain
   from pressure to selection to active edge to current work position often
   explains why a thing becomes current without requiring orientation as a
   separate concept.
5. **Continuity and preservation explain several orientation-like cases more
   directly.** Preservation documents ask what survived, what failed to survive,
   and what path or lineage remains available. These questions can be answered
   without making orientation the primary object.
6. **Orientation has high compression risk.** It may compress reference point,
   current concern, active edge, selection rationale, attention target,
   navigation route, continuity object, lineage, and preserved work position into
   one ordinary-language term.
7. **The orientation-object observation strengthens the pro-orientation side,
   but does not settle this audit.** It identifies question, frontier, current
   concern, active edge, continuity concern, and sometimes contradiction as the
   strongest orientation-object patterns; adversarial review still finds that
   many of those patterns can be decomposed into existing repository concepts.


## Orientation-Object Observation Review

`orientation_object_observation.md` is now included in the reviewed corpus. It
is explicitly exploratory and says orientation object is not promoted as
canonical vocabulary. Its working description tests whether something lets a
participant, document, surface, or continuation know what present work is about,
what counts as relevant, or what would make a next move intelligible.

The document provides the strongest direct support for orientation language in
this review. It identifies question, frontier, current concern, active edge,
continuity concern, and sometimes contradiction as the strongest
orientation-object patterns. It also finds that entity can serve as an
orientation object but should not be treated as the general form of orientation,
and that operator questions and navigation can orient work without making the
operator a universal concern subject.

The same document also weakens over-promotion of orientation. It preserves
exploratory status, refuses identity, agency, governance, schema, interface,
policy, and ontology promotion, and leaves unresolved whether orientation should
ever become a reconciled concept. For this audit, that means the document is
strong evidence that orientation is useful observational language, but not proof
that orientation survives as a standalone architectural object after reference
point, current concern, active edge, selection, continuity, preservation,
navigation, and authority have been considered.

## Contradiction Inventory

### Subject-first interpretations

Repository runtime and knowledge surfaces repeatedly use subject-centered
structures without orientation. Facts are grouped by subject, predicate, and
value; contradictions are detected across same-subject/same-predicate competing
values; explanations query subject and predicate; source navigation rows expose
support subjects and paths.

This weakens orientation as a universal explanation. For belief, conflict,
explanation, and source lookup, a subject-first model is sufficient. The subject
may be an entity, module, host, endpoint, or documentation identity. Orientation
adds little unless the question is about why that subject matters now.

Subject-first evidence does not disprove orientation. It bounds it: orientation
is not required for basic representational, evidential, or contradiction work.

### Entity-first interpretations

Entity, alias, host, endpoint, capability, repository module, and source-path
work explain many cases that orientation might otherwise absorb. Identity and
entity-boundary documents focus on how names, endpoints, aliases, and targets
are derived or kept distinct. The stronger explanation in those areas is entity
resolution, not orientation.

Entity-first evidence is especially strong where the problem is:

- whether two observed names identify the same thing;
- whether a target is a host, endpoint, filesystem subject, module, or source;
- whether evidence supports a particular subject;
- whether projection should preserve aliases without hiding provenance.

Orientation is weaker here because the repository already has more precise
vocabulary for identity, aliasing, subject, and evidence support.

### Authority-first interpretations

Authority documents explain many non-convergence cases without orientation.
Documentation authority, input-source authority, operator authority, projection
authority, view authority, and capability authority determine what a surface may
claim, infer, expose, execute, or decide.

Where the question is whether a document may own a conclusion, whether a surface
may mutate state, whether operator goals authorize policy, or whether a runtime
view may act as truth, authority-first explanation is more precise than
orientation. Orientation can even weaken clarity if it suggests that being
oriented toward a concern grants authority to resolve it.

### Selection-first interpretations

Selection-first explanations are among the strongest alternatives. Selection
work already distinguishes selected from available, active from preserved,
attention from selection, selected current concern from non-selected remainder,
and current output from broader repository memory.

Current work position and active edge may look orientation-shaped, but much of
their behavior can be explained as selected pressure plus preserved rationale.
Selection answers:

- why this concern is current;
- why adjacent concerns remain non-current;
- why this context entered the working set;
- why a future participant should resume here rather than elsewhere.

Orientation adds explanatory value only if it names something selection and
rationale do not already preserve.

### Continuity-first interpretations

Continuity-first explanations are strong where the repository asks what survives
change and what lets future work resume safely. Continuity does not require a
stable orientation object; it can concern a question, finding, lineage, active
edge, handoff position, or recognizable thread through changed artifacts.

This weakens orientation when the evidence is about survival across sessions,
frontier succession, or handoff recognizability. In those cases, continuity is
the more direct concept, while orientation may be shorthand for the particular
continuity content that must be preserved.

### Preservation-first interpretations

Preservation-first explanations are strong in preservation failure, discovery
path preservation, interpretation candidate preservation, architectural findings
preservation, and non-selected remainder preservation. These documents ask what
is saved, what is lost, what is discoverable, and what should not be retried.

Orientation is unnecessary when the evidence is specifically about preserving:

- lineage;
- discovery path;
- critique sequence;
- non-selected candidates;
- rejected compression;
- historical finding state;
- documentation memory;
- authority boundary;
- rationale for why a path was not taken.

Preservation may preserve orientation-like material, but preservation explains
the repository behavior without making orientation primary.

## Non-Convergence Findings

### Orientation is absent from core runtime contracts

The most important non-convergence finding is absence. The implemented runtime
contracts inspected during this audit do not expose an orientation object. They
operate over facts, evidence, state projection, contradictions, context
selection, explanations, source navigation, events, and summaries.

Absence is not disproof. However, it means orientation has not converged into the
runtime architecture the way fact support, contradiction detection, source
navigation, and context selection have.

### Orientation is unnecessary for fact support and contradiction visibility

Fact support and contradiction visibility require subject, predicate, value,
evidence, confidence, exclusivity, source type, and recency. They do not need an
orientation object. A contradiction can be visible because claims conflict, not
because a participant is oriented toward it.

Orientation may explain why a contradiction becomes current work. It does not
explain the contradiction itself better than existing contradiction vocabulary.

### Orientation is unnecessary for source navigation

Source navigation explains how a participant reaches supporting files, modules,
paths, and evidence. Navigation work already separates path, source, support,
query, and surface responsibility. Orientation can describe the human experience
of being less lost, but route construction and source lookup do not require it.

### Orientation is unnecessary for authority boundaries

Authority questions are not orientation questions. A surface either has authority
to claim, infer, present, decide, mutate, or execute, or it does not. Orientation
language risks obscuring these boundaries by making a well-oriented response
sound authorized when authority has not been established.

### Orientation is weaker than reference point for significance questions

When evidence asks why a condition matters, reference point is often stronger.
A disk forecast, host issue, documentation gap, or unresolved question matters
relative to something: operator concern, entity dependency, current work,
continuity need, repository memory, or authority boundary. That explanation
requires a reference point, not necessarily orientation.

### Orientation is weaker than active edge for live frontier pressure

Active edge already names the currently pulling unresolved boundary. If
orientation means the work is directed toward that boundary, it may duplicate
active edge. If it means something broader, the repository has not settled that
broader role.

### Orientation is weaker than current work position for handoff state

Current work position already carries selected context, active concern,
rationale, constraints, unresolved pressure, validation state, and safe next
movement. If orientation means preserved position for continuation, current work
position is the more precise repository term.

### Orientation is weaker than continuity for survival-through-change questions

Continuity asks whether a thread remains recognizable after artifact, participant,
or session change. Orientation may be one thing continuity preserves, but the
repository evidence does not require orientation to explain continuity itself.

### Orientation is weaker than preservation for memory and discovery-path cases

Preservation documents explain why final conclusions are insufficient without
lineage, rejected paths, candidate remainder, critique sequence, and historical
context. Orientation may describe one preserved quality, but discovery path and
lineage explain the actual evidence more directly.

## Compression Findings

Orientation may be fully or partly explainable as a compression over the
following repository concepts.

| Candidate compression | Evidence strength | Audit finding |
| --- | --- | --- |
| Reference point | Strong | Many orientation-like cases ask what makes a condition significant. Reference point explains significance without adding a new object. |
| Current concern | Strong | If orientation means what work is about now, current concern is already available. |
| Active edge | Strong | If orientation means the current live pull, active edge is more specific. |
| Selection | Strong | If orientation means why this concern entered current work, selection and selection rationale explain it. |
| Attention / target | Moderate to strong | Attention-target work already asks what focus is about and distinguishes target from relevance and active context. |
| Navigation | Moderate | If orientation means knowing where to go, navigation and source-navigation surfaces already own routeability. |
| Continuity | Strong | If orientation means safe continuation across sessions, continuity and handoff vocabulary are already present. |
| Preservation | Strong | If orientation means what must survive, preservation and lineage explain the repository pattern. |
| Working state | Moderate | If orientation means available continuation context, working state and activation surfaces already cover much of it. |
| Authority boundary | Moderate | If orientation means knowing what can be claimed or acted upon, authority vocabulary is more precise. |

The strongest compression finding is that orientation appears most useful when a
document is trying to say several of these at once. That usefulness is also the
risk: the term can hide which underlying repository concept is doing the work.

## Current Work Position / Active Edge Audit

Current work position and active edge provide the strongest evidence in favor of
orientation-like language, but they do not force convergence on orientation.

### Existing explanation without orientation

Current work position can be explained as:

```text
selected current concern
+ active edge
+ relevant constraints
+ authority boundaries
+ validation state
+ selection rationale
+ non-current remainder context
+ safe continuation position
```

Active edge can be explained as:

```text
unresolved pressure
+ selected frontier
+ current inquiry boundary
+ continuation relevance
+ adjacent non-selected pressures
```

Neither explanation requires orientation as an independent object. Both may
produce what readers experience as orientation, but that does not prove an
orientation object exists.

### Where orientation may still add value

Orientation may still add value if it captures the composite reader-facing effect
of current work position plus active edge plus constraints. This audit does not
validate that value. It only notes that the value, if present, appears composite
rather than primitive.

## Preservation Audit

Preservation evidence is better explained by lineage, continuity, discovery path,
and documentation memory in several areas.

### Lineage

Lineage explains how a finding, document, or concern came to be. Orientation is
not needed to preserve parentage, sequence, dependency, or derivation.

### Continuity

Continuity explains why the work remains recognizable after change. Orientation
may be one continuity payload, but continuity is the stronger explanation for
handoff and survival-through-change questions.

### Discovery path

Discovery path explains why preserving only the final finding loses important
critique, false starts, contradiction exposure, and compression removal. This is
not merely orientation; it is evidence about the path that made understanding
change.

### Documentation memory

Documentation memory explains how repository documents preserve previous work,
non-selected alternatives, frontiers, authority boundaries, and historical
status. Orientation may describe reader benefit, but memory and preservation
explain the artifact behavior.

## Critical Distinctions Under Review

### Orientation != reference point

This distinction does not fully survive review. Many orientation-like claims can
be rewritten as reference-point claims: something matters relative to an
operator, entity, current concern, continuity concern, repository surface, or
authority boundary. Orientation may require a reference point; reference point
does not require orientation.

### Orientation != current concern

This distinction partly survives. Current concern can be a concrete object of
work without the broader context needed for safe continuation. However, when
orientation simply means what the work is about now, current concern is enough.

### Orientation != active edge

This distinction partly survives. Active edge is narrower and pressure-shaped;
orientation, if retained, would likely be broader. But many examples of
orientation in current work prose appear to mean active edge plus selected
rationale.

### Orientation != selection

This distinction survives better than some others. Selection is an operation or
result of choosing among candidates, while orientation is ordinary-language
positioning toward work. Still, when orientation explains why a concern is
current, selection rationale often carries the explanatory load.

### Orientation != continuity

This distinction partly survives. Continuity is survival and recognizability
through change, while orientation is more about present direction or position.
But handoff and continuation evidence often needs continuity, not orientation, to
explain the repository behavior.

## Required Tensions

| Tension | Audit finding |
| --- | --- |
| Orientation vs reference point | Reference point is often the cleaner explanation for significance. Orientation risks becoming an umbrella over reference-sensitive concern. |
| Orientation vs current concern | Current concern explains what is current. Orientation adds only if it carries surrounding position, constraints, and rationale. |
| Orientation vs active edge | Active edge explains live unresolved pressure. Orientation duplicates it when used to mean current pull. |
| Orientation vs continuity | Continuity explains handoff recognizability. Orientation may be one payload but not the continuity mechanism. |
| Orientation vs preservation | Preservation explains what survives and what fails. Orientation may describe a preserved quality but not the preservation pattern. |
| Orientation vs authority | Authority determines what may be claimed or done. Orientation must not imply authority. |
| Orientation vs selection | Selection explains narrowing and currentness. Orientation risks hiding selection rationale unless decompressed. |

### What this audit adds

This audit adds an adversarial non-convergence view. It asks where orientation is
absent, unnecessary, weaker than another explanation, confusing, overlapping, or
compressive. It inventories subject-first, entity-first, authority-first,
selection-first, continuity-first, and preservation-first explanations that do
not require orientation.

### What this audit should avoid duplicating

This audit should not redefine orientation, reference point, active edge, current
work position, continuity, preservation, authority, selection, identity, agency,
or Seed ontology. It should not propose implementation remediation. It should
not promote orientation from observation language into runtime architecture.

## Strongest Evidence Against Orientation

1. Core runtime surfaces do not expose orientation as a primitive.
2. Subject/predicate/value fact support and contradiction detection work without
   orientation.
3. Source navigation works through paths, subjects, supports, and queries.
4. Authority work answers claim and action boundaries more precisely than
   orientation.
5. Selection and selection rationale already explain why one concern becomes
   current while alternatives remain preserved.
6. Continuity and preservation explain handoff, memory, lineage, and discovery
   path without requiring orientation.
7. Reference point explains significance in many examples that might otherwise
   be described as orientation.
8. The orientation-object observation supplies the strongest direct support for
   orientation, especially orientation without settled entity subject, but it also
   preserves exploratory status and avoids ontology promotion.

## Strongest Non-Convergence Findings

- Orientation has not converged into implementation contracts.
- Orientation has not converged into a single documentation authority surface.
- Orientation has not converged as distinct from reference point, current concern,
  active edge, or continuity.
- Orientation appears in prose as a reader-facing or handoff-facing quality more
  than as a governed architectural object.
- Areas with the strongest operational precision use other concepts: subject,
  entity, fact support, contradiction, source path, selection, authority,
  continuity, and preservation.

## Strongest Compression Findings

- Orientation can compress reference point plus current concern.
- Orientation can compress active edge plus current work position.
- Orientation can compress selection rationale plus attention target.
- Orientation can compress continuity plus preservation payload.
- Orientation can compress navigation route plus documentation memory.
- Orientation can compress authority awareness, even though authority itself must
  remain separate.

## Strongest Overlap Findings

- Orientation overlaps with reference point in significance questions.
- Orientation overlaps with current concern in currentness questions.
- Orientation overlaps with active edge in live-pressure questions.
- Orientation overlaps with selection in narrowing questions.
- Orientation overlaps with attention target in aboutness questions.
- Orientation overlaps with continuity in handoff questions.
- Orientation overlaps with preservation in survival and memory questions.
- Orientation overlaps with navigation in routeability questions.

## Strongest Ambiguity Findings

- It is unclear whether orientation names an object, relation, state, effect,
  reader experience, handoff payload, or compression over several concepts.
- It is unclear whether orientation is something preserved, something produced by
  selection, something required for continuity, or something inferred by a future
  participant.
- It is unclear whether every current concern has orientation or whether some
  current concerns are adequately represented by subject, question, active edge,
  or selection rationale alone.
- It is unclear whether orientation can be absent while reference point,
  selection, and continuity remain intact.
- It is unclear whether orientation should ever be used in authority-sensitive
  contexts, because it may imply more coherence than the repository has granted.

## Duplicate-Work Risks

1. Repeating reference-point findings under a new label.
2. Repeating current-work-position findings under a new label.
3. Repeating active-edge findings under a new label.
4. Repeating continuity and handoff findings under a new label.
5. Repeating preservation and discovery-path findings under a new label.
6. Repeating selection-rationale findings under a new label.
7. Repeating navigation-route findings under a new label.
8. Accidentally creating an ontology or identity proposal while trying to explain
   orientation.

## Unresolved Observations

- The repository contains strong orientation-like language, especially around
  current work, active edge, working-state activation, preservation failure, and
  operator understanding, but this audit did not find convergence on orientation
  as an independent architectural object.
- Direct review of `docs/orientation_object_observation.md` strengthens the
  evidence that repository prose can use orientation-object language, but does
  not resolve whether orientation is a standalone architecture concept or a
  compression over existing surfaces.
- Orientation may survive as a useful prose compression for future participants,
  but compression usefulness is not the same as architectural validation.
- Orientation may name the experienced result of several repository mechanisms
  rather than a mechanism itself.
- The strongest remaining question is whether any repository case requires
  orientation after reference point, current concern, active edge, selection,
  attention target, continuity, preservation, navigation, and authority have all
  been applied.

## Closing Audit Finding

Orientation does not fail completely: repository prose, especially the
orientation-object observation, shows repeated need to preserve where work is,
what pulls it, why it matters, and how a future participant can continue. But
adversarial review shows that much of this evidence remains explainable by
existing repository concepts. Orientation therefore does not currently survive
as a validated standalone architectural finding in this audit. It remains an
unresolved compression candidate over reference point, current concern, active
edge, selection, attention, navigation, continuity, preservation, authority
awareness, and documentation memory, even though the orientation-object
observation is strong evidence that the compression is useful and recurrent.
