# Compression Candidate Observation

## Purpose

This observation investigates repository evidence surrounding `compression`.

It asks:

```text
What repository concepts appear compressed?

What repository concepts absorbed pressure without decompression?
```

This is an observation, not a frontier, reconciliation, vocabulary proposal,
ontology proposal, governance proposal, workflow proposal, runtime proposal,
schema proposal, implementation proposal, memory-system proposal, or
learning-system proposal.

The document does not define compression canonically. It does not decide whether
compression is good or bad. It does not reconcile repository ontology. It
records candidate evidence, pressure signals, decompression examples, pressure
survivors, stable concepts, and uncertainty.

## Method And Authority Boundary

The investigation reviewed repository documentation that already discusses
compression, decompression, distinction emergence, preservation, lineage,
continuity, current work position, active edge, object / role / operation
pressure, concept stability, fact / claim boundaries, inquiry, attention, and
persistence.

Primary evidence included:

- `documentation_compression_observation.md`
- `understanding_claim_and_decompression_observation.md`
- `knowledge_and_understanding_distinction_observation.md`
- `discovery_path_preservation_observation.md`
- `preservation_failure_observation.md`
- `continuity_frontier.md`
- `current_work_position_frontier.md`
- `active_edge_frontier.md`
- `object_role_and_operation_frontier.md`
- `object_role_operation_pressure_test.md`
- `concept_stability_audit.md`
- `lineage_distinction_observation.md`
- adjacent fact / claim, attention, persistence, inquiry, and preservation
  documents where they materially supported or challenged a candidate.

Existing reconciliations and frontiers keep their scoped authority. This
observation compares evidence only to describe where pressure appears to have
accumulated and what happened next.

## Candidate Pattern Observed

Repository evidence supports a recurring but non-universal pattern:

```text
concept
    -> recurring unexplained pressure
    -> contradiction / ambiguity / insufficient explanatory power
    -> distinction appears
    -> understanding changes
```

The strongest examples are not merely cases where a new word appeared. They are
cases where an existing concept stopped answering a concrete repository question
safely. The pressure often came from questions such as:

- Is this reality, representation, claim, support, or authority?
- Is this preserved information, continuity, lineage, identity, or active work
  position?
- Is this the cause of attention or the thing receiving attention?
- Is this an object, role, operation, relationship, or mixed context?
- Is this knowledge-like preservation or understanding-like preservation?

The pattern is not universal. Some concepts absorbed pressure without obvious
splitting. Some broad concepts remained useful because they were already scoped,
source-bound, or relational enough to carry pressure without hiding a necessary
operational distinction.

## Strongest Compression Candidates

### Fact

`Fact` is a strong compression candidate, but the evidence does not show only
one reading.

Pressure accumulated when repository work needed to distinguish reality from
representation and support from authority. Fact-oriented language could suggest
settled reality even when Seed held a claim with evidence, scope, freshness,
source authority, and possible contradiction. Reality / fact / claim work
therefore separated facts in reality from claims in representation and preserved
claim-centric representation without denying facts in reality.

What failed was not the ordinary usefulness of `fact`. What failed was using a
single fact-shaped term to answer representation, verification, authority,
confidence, and reality questions at once.

The distinction that appeared was:

```text
reality may contain facts
representations contain claims
facts in Seed are normalized claim forms
support is not the same as truth or authority
```

The distinction reduced pressure where the repository needed to represent stale,
uncertain, contradicted, future-facing, or differently supported knowledge. It
also preserved a boundary: `Fact -> Claim` should not be read as facts never
exist. The stronger observation is that repository `fact` language was partly
compressed and partly incomplete for claim-centric representation.

### Working State

`Working state` is a strong compression candidate.

Pressure accumulated around continuation. Preserved text, summaries, or facts
could survive while the next participant still lacked the live question, active
constraint, unresolved pressure, validation state, selection rationale, or next
safe move. That made `working state` carry too much: stored context, activity
context, handoff context, current position, current frontier, active edge, and
continuation readiness.

What failed was treating retained context as enough to resume work safely.

Distinctions that appeared include:

```text
working state -> continuation context
working state -> current work position
current work position -> selected active context + live frontier + constraints
active edge -> currently pulling boundary of unresolved work
```

The distinction reduced pressure by explaining why a handoff can preserve facts
but still fail to preserve the actionable work position.

### Lineage

`Lineage` is a strong compression candidate.

Pressure accumulated when visible artifact relationships were asked to prove
more than they could show. A link, dependency, index route, or commit cluster can
show artifact lineage, but not necessarily inquiry movement, observer working
memory, prompt influence, causal discovery order, or the challenge sequence that
changed understanding.

What failed was treating visible documentation linkage as sufficient evidence
for discovery path or causal inquiry history.

Distinctions that appeared include:

```text
artifact lineage
inquiry lineage
observation lineage
discovery-path lineage
```

The distinction reduced pressure by letting repository participants preserve
what a document graph can show without overclaiming what it cannot show.

### Attention

`Attention` appears moderately to strongly compressed.

Pressure accumulated because repository work needed to distinguish why attention
moved from what attention was about. Selection pressure, trigger, target,
frontier, gap, risk, operator intent, active context, and focus could all sit
under attention-like language.

What failed was saying only that something had attention when the repository
needed to know whether the cause was a contradiction, operator request, missing
evidence, risk, unresolved failure, inherited frontier, or candidate object of
inquiry.

Distinctions that appeared include:

```text
attention trigger -> why attention becomes active or moves
attention target -> what receives attention
selection rationale -> why a target is selected rather than merely present
active edge -> selected pressure currently pulling work
```

The distinction reduced pressure for inquiry routing and selection explanation,
while leaving ordinary `attention` useful as broad navigation language.

### Persistence

`Persistence` appears compressed where it was asked to cover storage, identity,
survival, continuity, and participation.

Pressure accumulated from handoff and continuity work. Stored artifacts can
remain while represented understanding, active pressure, or current work
position does not. Conversely, a question or frontier can persist conceptually
while wording changes.

What failed was equating material retention with conceptual survival.

Distinctions that appeared include:

```text
persistence != storage
persistence != identity
continuity asks what remains recognizable across change
preserved artifact != preserved active work position
```

The distinction reduced pressure by explaining why retention can be necessary
without being sufficient for continuation.

### Understanding

`Understanding` is a strong compression candidate because repository evidence
shows it carrying claim, support, explanation, scope, path, caveat, and
continuation meanings.

Pressure accumulated when participants used final conclusions but still lacked
why those conclusions were safe, what support changed, what hidden distinctions
appeared, or what unknown unknowns were exposed. Understanding also came under
pressure where conclusions survived but discovery paths did not.

What failed was treating `I understand X` as a simple fact-like state.

Distinctions that appeared include:

```text
understanding as claim-like posture
understanding support path
understanding transition
knowledge-like preservation vs understanding-like preservation
conclusion preservation vs discovery-path preservation
```

The distinction reduced pressure by allowing understanding to be described as
supported, scoped, revisable, partially preserved, or weakened when support paths
are missing.

### Knowledge

`Knowledge` appears compressed in a weaker and more bounded way than
`understanding`.

Pressure accumulated where repository work needed to distinguish preserved
knowledge artifacts from a participant's ability to use, explain, continue from,
or revise them. Knowledge could mean represented claims and relationships, the
repository's preserved material, or a participant's usable understanding of that
material.

What failed was treating preserved knowledge as automatically equivalent to
understanding.

Distinction that appeared:

```text
knowledge-like preservation -> what is represented or preserved
understanding-like preservation -> how a participant can use, explain, or
continue from what is represented
```

The distinction reduced pressure by explaining why repository knowledge can
exist while current work orientation or understanding remains incomplete.

### Object

`Object` is a strong candidate for pressure, but only a candidate for
compression.

Pressure accumulated when `object` language was asked to classify durable things,
contextual participation, activities, relationships, and mixed inquiry artifacts.
The object / role / operation lens helped, but the pressure test found it
incomplete around relationship semantics, support, contradiction, authority,
lineage, and working-state connections.

What failed was treating every represented concern as simply an object.

Distinctions that appeared include:

```text
object -> durable represented thing
role -> contextual participation
operation -> what happens to represented knowledge
relationship -> connection claim not reducible to object identity
```

The distinction reduced pressure for many examples, but not all. The pressure
test is important evidence that decompression can expose remaining pressure
rather than fully resolve it.

### Frontier

`Frontier` appears compressed where it names unresolved work, inquiry object,
active edge, status, next direction, and boundary of premature implementation.

Pressure accumulated as more documents used frontier language to preserve active
questions without treating them as settled architecture. Current-work-position
and active-edge work increased the pressure by asking which frontier is live now
and what boundary is actively pulling attention.

What failed was treating every frontier as equally active, equally routed, or
equally actionable.

Distinctions that appeared include:

```text
frontier as unresolved boundary
active edge as currently pulling boundary
current work position as selected active continuation state
preserved frontier as not necessarily active work
```

The distinction reduced pressure by separating a documented unresolved area from
the live continuation position.

### Inquiry

`Inquiry` appears compressed as process, object family, lineage, pursuit,
frontier, and preserved unresolved understanding.

Pressure accumulated because repository documents repeatedly preserve questions,
gaps, tensions, audits, findings, reconciliations, and frontiers as more than a
linear process. They can persist, branch, resume, constrain later work, and
carry lineage.

What failed was treating inquiry only as a process that produces knowledge.

Distinctions that appeared include:

```text
inquiry process
candidate inquiry object
inquiry lineage
finding as bridge between inquiry and knowledge
frontier as candidate inquiry state or object
```

The distinction reduced pressure by explaining why unresolved pursuit can remain
architecturally relevant without proposing an inquiry runtime or ontology.

## Pressure Survivors

### Observation

`Observation` appears to absorb pressure without decompression better than many
other concepts.

Pressure accumulated around reality boundaries, source authority, interpretation,
freshness, batching, event granularity, and documentation observation. What
survived was the source-bound posture: observation reports rather than becomes
reality. The repository often adds adjacent distinctions around evidence,
claim, interpretation, source, and projection instead of decomposing observation
itself.

Observation did not require decomposition in the reviewed evidence because its
authority boundary already limits what it can claim. It can carry pressure by
remaining a report with source and scope rather than a truth, fact, or conclusion.

### Authority

`Authority` appears to absorb pressure because it already behaves relationally
and scope-dependently.

Pressure accumulated around source authority, documentation authority, operator
authority, view authority, and architecture authority. What survived was not a
single universal authority object, but the repeated rule that authority is
scoped to source, document, surface, or decision context.

Authority did not obviously require decomposition because repository documents
already tend to ask `authority for what, in which scope?` The pressure is handled
through boundaries rather than by splitting authority into a new object family.

### Relationship

`Relationship` appears to survive pressure partly because it is already a
connection claim rather than a hidden object.

Pressure accumulated around identity, support, contradiction, lineage,
dependency, belongs-to, runs-on, and working-state connections. Some pressure
moved to specific relationship semantics, but the broad concept remained stable:
a relationship preserves a connection claim without overclaiming identity.

Relationship did not require broad decomposition in the reviewed evidence
because its usefulness comes from not collapsing connection into object sameness.
Pressure produced more careful semantics, not a replacement for relationship.

### Claim

`Claim` is one of the strongest pressure survivors.

Pressure accumulated around fact, evidence, support strength, assertion strength,
contradiction, future claims, agency, ownership, causality, and explanation.
What survived was the central posture that represented knowledge is claim-like:
supportable, scoped, revisable, contradictable, and separable from reality.

Claim did not require decomposition in the reviewed evidence because it already
absorbs uncertainty. Instead of splitting claim apart, the repository usually
adds support, strength, authority, provenance, contradiction, or projection
boundaries around it.

### Evidence

`Evidence` also appears stable under pressure.

Pressure accumulated around provenance, trust, support, source authority,
freshness, and claim support. What survived was the role of evidence as preserved
support material rather than the represented claim itself.

Evidence did not require decomposition because the repository repeatedly uses it
to preserve why a claim is available for consideration while keeping evidence
separate from fact, claim, truth, and authority.

## Critical Examples

### Example 1: Fact -> Claim

The evidence supports a mixed finding.

`Fact` was compressed where it carried representation, support, authority, and
reality pressure. It was also incomplete where early fact language lacked enough
claim-centric boundaries for stale, uncertain, contradicted, sourced, or future
knowledge.

The decompression did not abolish fact. It moved repository representation
toward claim-centric language and left fact as either reality-side condition or
normalized claim form depending on scope.

### Example 2: Working State -> Current Work Position

The pressure was continuation failure.

Repository evidence shows that preserved state can fail to preserve the live
position. The distinction appeared when documents needed to know not just what
was known, but what work was active, what was unresolved, what constraints still
mattered, and what next move would be safe.

The decompression reduced pressure by separating retained context from
continuation-ready orientation.

### Example 3: Lineage Expansion

The pressure was evidentiary overclaim.

Artifact links could not safely prove inquiry causality, discovery sequence,
prompt influence, or understanding transitions. Lineage expanded into artifact,
inquiry, observation, and discovery-path lineage so that each surface could keep
its own authority.

The decompression reduced pressure by preserving visible lineage while naming
blind spots.

### Example 4: Understanding Claim

The pressure was support-path loss.

A participant may have a conclusion but not the support, distinction, path,
active pressure, or caveat that made it understandable. Understanding therefore
came to behave like a claim-like posture: supportable, scoped, revisable, and
challengeable.

The decompression reduced pressure by making it possible to say that
understanding can strengthen, weaken, or partially survive without becoming a
binary fact.

### Example 5: Claim As Stable Under Pressure

`Claim` remained stable despite heavy pressure because its shape already absorbs
uncertainty. A claim can be supported, contradicted, stale, future-facing,
scoped, weak, strong, selected, projected, or revised without ceasing to be a
claim.

The pressure did not produce broad decomposition because surrounding concepts
absorbed the needed distinctions: evidence, support strength, assertion strength,
authority, projection, corroboration, contradiction, and fact promotion.

## Required Findings

### Strongest Compression Candidates

The strongest candidates are:

1. `Working state` — because continuation pressure exposed a clear gap between
   preserved information and current work position.
2. `Lineage` — because visible artifact linkage was repeatedly insufficient for
   discovery-path and inquiry-history claims.
3. `Understanding` — because support-path, caveat, and decomposition evidence
   changed what `understanding` could safely mean.
4. `Fact` — because claim-centric representation separated reality, claim,
   support, fact normalization, and authority.
5. `Persistence` — because storage, identity, continuity, and conceptual
   survival were repeatedly not equivalent.
6. `Attention` — because trigger and target pressures separated why attention
   moved from what received attention.

### Strongest Pressure Signals

The strongest signals are:

- preserved artifacts exist but continuation still fails;
- a term must answer both reality-side and representation-side questions;
- a visible relationship is asked to prove causal discovery history;
- a conclusion survives but the reason it became understandable does not;
- a concept is asked to cover object, role, operation, and relationship at once;
- selection occurs but the trigger, target, and rationale cannot be separated;
- storage survives but recognizable continuity does not.

### Strongest Decompression Examples

The strongest decompression examples are:

- fact / claim / support / authority separation;
- working state / continuation context / current work position / active edge;
- lineage as artifact, inquiry, observation, and discovery-path lineage;
- attention as trigger and target, with selection rationale nearby;
- persistence as not storage and not identity;
- understanding as claim-like, supported, scoped, revisable, and path-dependent.

### Strongest Pressure Survivors

The strongest pressure survivors are:

- `Claim`, because uncertainty, revision, support, contradiction, and scope fit
  its existing shape.
- `Observation`, because source-bound reporting limits overclaim.
- `Evidence`, because provenance/support material remains distinct from claim
  and truth.
- `Authority`, because scoped authority boundaries handle pressure without one
  universal split.
- `Relationship`, because connection claims already avoid identity collapse.

### Strongest Stable Concepts

The strongest stable concepts are `claim`, `observation`, and `evidence`.
Their stability appears to come from bounded responsibility rather than lack of
pressure. Each survives because it does not need to do every job: observation
reports, evidence preserves support, and claim represents supportable assertion.

### Strongest Examples Where Pressure Did Not Produce Decomposition

- Claim pressure produced support and strength boundaries around claims rather
  than a replacement for claim.
- Observation pressure produced source, interpretation, freshness, and evidence
  boundaries rather than replacing observation.
- Authority pressure produced scoped authority surfaces rather than a single new
  authority taxonomy in this observation.
- Relationship pressure produced relationship semantics and identity boundaries
  rather than abandoning relationship.

### Strongest Examples Where Decomposition Reduced Pressure

- Working state decompression reduced continuation ambiguity.
- Lineage decompression reduced evidentiary overclaim.
- Fact / claim decompression reduced reality-representation ambiguity.
- Understanding decompression reduced conclusion-versus-support-path ambiguity.
- Attention decompression reduced trigger-versus-target ambiguity.
- Persistence decompression reduced storage-versus-continuity ambiguity.

## Tensions Preserved

### Compression vs Breadth

A broad concept is not necessarily compressed. `Observation` and `claim` remain
broad but stable because their responsibilities are bounded. Breadth becomes
compression when the broad term must answer incompatible or hidden questions.

### Compression vs Abstraction

An abstract concept is not necessarily compressed. `Authority` is abstract, but
repository evidence often handles it through scoped authority boundaries. The
compression signal appears when abstraction hides necessary distinctions rather
than simply generalizing across cases.

### Compression vs Ambiguity

Ambiguity can signal compression, but ambiguity alone is not enough. Some
frontiers intentionally preserve ambiguity because evidence is unsettled.
Compression is better supported where ambiguity repeatedly causes failed
continuation, unsafe overclaim, or explanatory loss.

### Compression vs Explanation Load

A concept may be compressed when every use requires increasing explanation to
avoid overclaim. `Fact`, `lineage`, `working state`, and `understanding` show
this pattern. Decompression reduced explanation load by placing distinctions
where later readers could reuse them.

### Decomposition vs Invention

Not every decomposition is evidence of prior compression. Some distinctions may
be new exploratory inventions, local clarifications, or temporary scaffolding.
This observation treats a decomposition as stronger evidence only when it
responded to documented pressure and reduced a documented failure mode.

### Stable Concept vs Compressed Concept

Stable concepts can face heavy pressure. `Claim` faced more pressure than many
compressed candidates, but it survived because it already represented a
supportable, revisable, scoped assertion. Stability appears linked to bounded
responsibility, not to absence of challenge.

### Pressure Absorption vs Pressure Splitting

Some concepts absorb pressure by keeping a tight boundary and letting adjacent
concepts carry distinctions. Others split because the original term cannot carry
multiple jobs safely. `Claim` absorbed pressure; `working state` split under
continuation pressure.

## Unresolved Observations

- The repository has not canonically defined compression, and this document does
  not do so.
- Some decompressions may reflect maturing investigation rather than evidence
  that the earlier concept was compressed.
- `Object`, `frontier`, and `inquiry` remain mixed cases: each shows pressure,
  but the evidence is less settled than for working state, lineage,
  understanding, persistence, attention, or fact.
- Pressure survivors may later decompress under a different investigation. Their
  current stability is an observed state, not a guarantee.
- The candidate pattern is useful but not universal. Repository evidence does
  not support treating every distinction-emergence example as compression.

## Closing Observation

Repository evidence suggests compression may appear when a useful concept
silently carries multiple responsibilities until concrete work asks it to answer
a question it cannot answer safely. Decompression appears strongest when a new
distinction preserves the useful part of the old concept while reducing a
specific failure mode: overclaim, continuation loss, authority confusion,
lineage overreach, support-path loss, or trigger / target ambiguity.

The opposite pattern also matters. Some concepts remain stable because their
boundaries already let pressure accumulate without hiding necessary distinctions.
The repository evidence therefore supports both compression candidates and
pressure survivors.
