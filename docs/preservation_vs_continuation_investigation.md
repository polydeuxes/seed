# Preservation Versus Continuation Investigation

## Central finding

Repository evidence supports a meaningful distinction:

```text
preservation
    -> static relationship family

continuation
    -> temporal relationship family
```

This is not yet a first-class repository model.

However it explains a recurring pattern across investigations.

Preservation repeatedly became visible through implementation-backed surfaces.

Continuation repeatedly required stronger temporal evidence, continuity evidence, historical evidence, lineage evidence, or movement evidence.

## Preservation evidence

Strong implementation-backed examples:

```text
unknown remains unknown

classification remains unknown

limitation remains explicit

evidence remains attached

visibility class remains constrained

diagnostic inspection preserves unresolved state
```

Common shape:

```text
state exists
boundary or limitation prevents unsupported change
state remains visible
```

Preservation can be observed at a single point in time.

## Continuation evidence

Repository-visible continuation evidence is weaker.

Candidates reviewed:

```text
inquiry note continuity

audit continuity

state continuity

projection continuity

operator continuity

investigation continuity
```

Strongest repository-backed examples:

```text
recorded inquiry note
    -> later inquiry orientation retrieval

projection version continuity
    -> later projection build uses same accumulated repository state

operator continuation handoffs
    -> later investigations continue prior investigation context
```

Unlike preservation, continuation requires at least two positions in time.

Common shape:

```text
state A
    -> later state B
```

or:

```text
artifact A
    -> later artifact A
```

with enough evidence to justify identity across time.

## Preservation without continuation

Repository evidence supports this.

Examples:

```text
classification remains unknown
```

No later classification event is required.

```text
evidence remains attached
```

No later inquiry continuation is required.

```text
limitation remains explicit
```

No temporal chain is required.

General form:

```text
state preserved
without demonstrating later state
```

This is why preservation became visible earlier.

## Continuation without preservation

Repository evidence also supports this.

Examples:

```text
pressure continues
but transforms
```

```text
investigation continues
but conclusions change
```

```text
projection continues
while contents evolve
```

General form:

```text
identity continues
state changes
```

Continuation therefore does not require preservation.

## Preservation versus continuity

Repository evidence suggests:

```text
preservation
    -> retained state

continuity
    -> retained identity across time
```

A thing may preserve state without proving continuity.

A thing may demonstrate continuity while changing state.

## Preservation versus lineage

Lineage asks:

```text
where did this come from?
```

Preservation asks:

```text
what remained unchanged?
```

Repository evidence for preservation is much stronger than repository evidence for lineage.

## Preservation versus movement

Movement asks:

```text
how did inquiry move?
```

Preservation asks:

```text
what remained visible despite pressure to change?
```

Movement requires temporal structure.

Preservation does not.

## Preservation versus transformation

Transformation:

```text
A becomes B
```

Preservation:

```text
A remains A
```

Continuation:

```text
A later remains identifiable as A
```

These are distinct.

## Counterexamples

### Appears preserved but does not continue

```text
unknown remains unknown
```

No evidence of later inquiry.

```text
limitation remains explicit
```

No temporal chain.

### Appears to continue but is not preserved

```text
pressure continues
but transforms
```

```text
investigation continues
but reframes the question
```

```text
operator continuity exists
but repository state changes
```

## Supported conclusions

1. Repository evidence supports preservation as a recurring static relationship family.
2. Repository evidence partially supports continuation as a temporal relationship family.
3. Preservation can exist without continuation.
4. Continuation can exist without preservation.
5. Preservation is currently easier to ground than continuation.
6. Preservation evidence is stronger than lineage, movement, or transformation evidence.
7. Preservation and continuation are related but distinct families.

## Unsupported conclusions

- Preservation and continuation are equivalent.
- Preservation requires temporal evidence.
- Continuation requires preserved state.
- Lineage and continuation are the same thing.
- Movement and continuation are the same thing.
- Preservation proves continuity.

## Open questions

- Is retained identity the minimal requirement for continuation?
- Can continuation become repository-visible without lineage?
- Is continuity a subtype of continuation or a separate concept?
- Can preservation relationships become first-class before continuation relationships?
- Is preservation the first observable relationship family because it avoids temporal claims?

## Acceptance answers

### What distinguishes preservation from continuation?

```text
preservation
    -> retained state

continuation
    -> retained identity across time
```

### Can preservation exist without continuation?

Yes.

### Can continuation exist without preservation?

Yes.

Identity can continue while state changes.

### Does repository evidence support preservation as a static relationship family?

Yes.

### Does repository evidence support continuation as a temporal relationship family?

Partially yes.

### Are preservation and continuation members of the same family or different families?

Repository evidence currently supports them as related but distinct families:

```text
preservation
    -> static

continuation
    -> temporal
```
