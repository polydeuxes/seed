# Work Shape And Orientation Observation

## Purpose

This document observes whether repository work appears in recurring shapes and
whether those shapes seem to require different orientation structures.

It is observation only. It is not a reconciliation, frontier, workflow proposal,
task ontology, implementation proposal, operator model, execution architecture,
schema proposal, remediation plan, or recommendation.

The central questions are:

```text
Do different repository work shapes require different orientation structures?
```

```text
Why do some repository activities continue successfully with weak pressure,
weak active edge, or weak continuation posture, while others fail without them?
```

```text
Does repository evidence support different categories of work that orient
differently?
```

The review did not assume that work shapes exist, that the examples in the
prompt survive review, or that any orientation subset is universal.

## Authority And Method Boundary

Repository evidence was treated as authority. Reconciliations were treated as
stronger boundary records where they establish distinctions or ownership.
Frontiers were treated as unsettled pressure surfaces. Observations were treated
as pattern evidence. Runtime and tests were used only as implementation-facing
read-model evidence.

No runtime changes were made. No runtime changes are proposed.

The review preserves these distinctions where repository evidence supports them:

```text
work shape != workflow
work shape != task
orientation != work shape
continuation != all work
lookup != resumption
inventory != active work
observation != recommendation
navigation != selection
```

These distinctions are observational here. They are not introduced as canonical
vocabulary.

## Major Findings

### 1. Repository evidence supports recurring work shapes, but not a canonical taxonomy

The strongest observed pattern is that repository work does not always present
as one uniform activity. Different documents preserve different work postures:

- lookup or source navigation asks where something is and what surface can answer
  a question;
- inventory asks what exists within a bounded population;
- observation records what repository evidence appears to show;
- audit checks a surface against a boundary, expectation, or implementation
  state;
- reconciliation relates claims, boundaries, authority, and implementation-facing
  evidence;
- frontier exploration keeps an unresolved edge available without settling it;
- continuation and handoff preserve enough current position for later work;
- recommendation or selection-facing surfaces explain why one candidate or path
  was chosen without making selection truth or execution authority;
- implementation support read-models expose deterministic state, context,
  source-navigation, or summary surfaces without becoming the work itself.

These shapes are not workflows in the reviewed evidence. They do not prescribe
ordered stages, a lifecycle, a task system, or an agent architecture. They are
observed postures of repository work that recur across document families.

### 2. Orientation appears compositional and work-shape dependent

Repository evidence does not support a universal orientation subset that is
required for every work shape. The orientation bundle reviewed in
`orientation_bundle_load_bearing_observation.md` appears load-bearing when work
must remain continuable across drift, restart, handoff, or unresolved pressure.
The same bundle appears partially unnecessary when the activity is a bounded
lookup, inventory, or read-only audit with a clear authority boundary.

A cautious observation is:

```text
Orientation reconfigures according to what would make the current activity fail.
```

For bounded lookup, failure often means looking in the wrong place, so reference
point, query surface, and boundary matter more than active edge or continuation
posture.

For inventory, failure often means overcounting, undercounting, crossing scope,
or treating an inventory as a decision. Boundary, population, inclusion rules,
and validation state matter more than pressure or next safe move.

For observation, failure often means overstating evidence, reconciling too soon,
or converting observed patterns into proposals. Evidence boundary, authority
boundary, and claim strength matter more than selection rationale.

For reconciliation, failure often means mixing authorities or collapsing distinct
concepts. Boundary, reference point, relation among claims, and validation state
matter strongly; active edge may appear only as the unresolved relation being
reconciled.

For frontier exploration, failure often means prematurely settling the frontier
or losing the open pressure that made it worth preserving. Active edge,
pressure, boundary, non-resolution posture, and continuation all appear stronger.

For continuation or handoff, failure often means a later participant can read the
content but cannot safely resume. Current concern, pressure, active edge,
selection rationale, validation state, and next safe move appear strongest here.

### 3. Weak pressure, weak active edge, or weak continuation posture is sometimes harmless

Some activities continue successfully with weak pressure because they are
anchored by a stable reference point and bounded question. A source-navigation
lookup can succeed if the query, target, and boundary are clear. It does not
need to preserve why the repository is investigating that target now.

Some activities continue successfully with weak active edge because they are not
at an unresolved edge. An inventory can report a bounded set; an audit can report
pass/fail or supported/unsupported findings; a map can route readers to existing
surfaces. In those shapes, the absence of an active edge is not necessarily a
failure.

Some activities continue successfully with weak continuation posture because
they are terminal or consumable as reference. A completed inventory or bounded
audit may not need a next safe move if it is not meant to transfer active work.
The evidence becomes problematic only when a later participant is expected to
resume an active investigation from it.

### 4. The same weakness becomes failure in continuation-facing work

The failure documents make the opposite pattern visible. Where work must be
resumed, preserved content alone can be insufficient. A handoff or state summary
may contain facts, answers, references, or conclusions, but later work can still
reconstruct the missing orientation if it lacks:

- what concern is current;
- what pressure selected it;
- what reference point anchors it;
- what boundary governs it;
- what unresolved edge remains active;
- why this path was selected over nearby paths;
- what has already been validated;
- what movement remains safe.

This explains why the same orientation element can be optional in one work shape
and load-bearing in another. The element is load-bearing when its absence forces
reconstruction, drift, duplicate work, or unsafe continuation.

## Work-Shape Findings

The following shapes appear repeatedly enough to be useful observations. The
labels are descriptive only.

### Lookup / source navigation

Observed surfaces include source-navigation tests, source navigation design and
audit documents, knowledge navigation documents, and map/index documents.

Likely necessary orientation elements:

- reference point or query;
- target surface;
- authority boundary;
- enough validation to know whether the lookup found a match.

Often optional or weak without failure:

- pressure;
- active edge;
- continuation posture;
- selection rationale beyond why the lookup target was chosen.

Absent-without-failure pattern:

- A lookup can answer "where is this?" without preserving the larger current
  concern, as long as the lookup result is not treated as resumption context.

### Navigation / map use

Navigation appears broader than lookup. It can route from a question to a layer,
map, concept, document family, or implementation surface.

Likely necessary orientation elements:

- question or starting point;
- reference layer;
- boundary between navigation and selection;
- destination evidence or map relation.

Often optional or weak without failure:

- active edge, unless navigation is toward an unresolved frontier;
- continuation, unless the map is used to resume active work;
- pressure, unless the route is selected because of a live concern.

Observed distinction:

```text
navigation exposes possible paths; selection explains why one path is active now.
```

### Inventory

Inventory appears in capability, policy, execution, source, documentation,
pending-action, and audit documents.

Likely necessary orientation elements:

- bounded population;
- inclusion/exclusion boundary;
- current observed state;
- validation state or count/status criteria;
- authority of the source being inventoried.

Often optional or weak without failure:

- pressure;
- active edge;
- next safe move;
- selection rationale.

Duplicate-work risk:

- Inventories reduce reconstruction by making a population visible, but they do
  not by themselves preserve why one item should become active work.

### Observation

Observation documents repeatedly preserve evidence and pattern without deciding
architecture.

Likely necessary orientation elements:

- evidence boundary;
- authority boundary;
- claim-strength restraint;
- inspected documents and search terms;
- unresolved observations.

Often optional or weak without failure:

- recommendation;
- implementation posture;
- task sequencing;
- complete continuation posture.

Strong absence boundary:

- Observation can succeed precisely because it does not recommend, reconcile, or
  propose remediation.

### Investigation

Investigation appears as a broader activity that can include lookup, inventory,
audit, observation, and frontier exploration.

Likely necessary orientation elements:

- current concern or question;
- search path or evidence plan;
- boundary of what is and is not being decided;
- validation state for what has been checked;
- unresolved observations.

Variable elements:

- pressure is strong when the investigation is selected from alternatives;
- active edge is strong when the investigation is frontier-facing;
- continuation is strong when the investigation is not completed in one pass.

### Audit

Audit surfaces repeatedly compare an implementation, document family, or
read-model behavior against a known boundary or expectation.

Likely necessary orientation elements:

- audit target;
- expected boundary or invariant;
- evidence surface;
- finding status;
- validation state.

Often optional or weak without failure:

- active edge, unless the audit exposes unresolved architecture;
- next safe move, unless the audit is intended as continuation context;
- pressure, if the audit target is already explicit.

### Reconciliation

Reconciliation documents relate claims, authority, concepts, and implementation
surfaces. They are stronger than observations where they establish boundaries.

Likely necessary orientation elements:

- reference point;
- authority boundary;
- relation among claims or surfaces;
- distinction being preserved;
- validation state;
- sometimes pressure if a conflict or gap selected the reconciliation.

Often optional or weak without failure:

- next safe move;
- active edge after the reconciliation settles a boundary;
- selection rationale if the reconciliation target is already named.

### Frontier exploration

Frontiers preserve an unresolved edge. They often contain pressure, questions,
and nearby authority boundaries without settling architecture.

Likely necessary orientation elements:

- active edge;
- unresolved pressure;
- boundary against premature implementation or settlement;
- reference point in existing documents;
- continuation or future revisit posture.

Often optional or weak without failure:

- validation state may remain partial;
- next safe move may be broad rather than procedural.

### Continuation / resumption / handoff

Continuation-facing documents provide the strongest evidence for a fuller
orientation bundle.

Likely necessary orientation elements:

- current concern;
- pressure;
- reference point;
- authority boundary;
- active edge;
- selection rationale;
- validation state;
- continuation posture or next safe move.

Often optional or weak without failure:

- Very little is consistently optional when the goal is safe resumption. The
  exact form varies, but missing relation among these elements repeatedly appears
  to cause reconstruction or drift.

Observed distinction:

```text
lookup != resumption
```

A lookup can locate a surface. Resumption needs the relation that made that
surface relevant to active work.

### Recommendation / selection-facing work

Recommendation and selection surfaces appear adjacent to but distinct from
navigation and observation. They explain why one candidate was chosen or ranked
without making that selection truth, policy, execution, or authority.

Likely necessary orientation elements:

- candidate set;
- selection criterion or rationale;
- boundary that selection is not truth or execution;
- validation state of candidates;
- reference point for the need or surface being selected for.

Variable elements:

- pressure is strong when selection answers "why this now";
- active edge is strong when selection chooses an unresolved investigation;
- continuation is strong only when selected work must be resumed later.

### Implementation support / read-model review

Runtime-facing read-model documents and tests expose state summary, context
views, source navigation, trace summaries, and projection summaries. In this
review they were used only as evidence of surfaces and boundaries.

Likely necessary orientation elements:

- read-only boundary;
- source of truth;
- surface responsibility;
- validation state through tests;
- precise target surface.

Often optional or weak without failure:

- pressure;
- active edge;
- continuation posture.

Important boundary:

- Implementation-facing evidence can show what surfaces exist and how they are
  bounded, but this observation does not convert that evidence into runtime
  change.

## State Summary Review

The original State Summary pressure appears to have been an operator-facing
understanding and resumption problem, not merely a display problem.

The work shape appears mixed:

```text
read-model summary + operator understanding surface + continuation support
```

Orientation information that already existed in repository surfaces included:

- projected State as source material;
- summary counts and current state categories;
- read-only surface boundaries;
- trace and explanation surfaces for decisions and outcomes;
- context-view boundaries for provider-facing selection.

Orientation information that appeared missing or weak included:

- why this subset mattered to the current concern;
- which pressure made the summary useful now;
- what active edge or unresolved state the summary should preserve;
- how the summary related to safe continuation rather than just display;
- which validation state should be visible to prevent overconfidence;
- how to distinguish answer availability from answer governing current work.

This explains why State Summary pressure repeatedly attracted orientation
language. A summary that only inventories state can succeed as inventory but fail
as continuation support. If the operator is trying to resume or decide what is
currently significant, the summary needs more than counts or facts; it needs
relations that preserve current relevance, boundary, validation, and safe next
movement.

## Current Work Position Review

Current Work Position does not appear purely universal or purely
work-shape-specific.

Evidence supports a mixed observation:

```text
Current Work Position is a continuation-facing surface that becomes broadly
useful when work must preserve present relevance, but it is not required for
every bounded lookup, inventory, or terminal audit.
```

It appears universal only for work that must answer:

```text
Where are we in this work, and what remains safe to do next?
```

It appears unnecessary or lighter when work only asks:

```text
Where is this symbol?
What exists in this bounded set?
Does this surface respect its boundary?
What does this document observe?
```

## Active Edge Review

Active Edge does not participate equally across all work shapes.

It appears strongest in:

- frontier exploration;
- continuation;
- handoff;
- unresolved investigations;
- selection among possible next inquiries.

It appears weaker or absent without failure in:

- lookup;
- bounded inventory;
- completed audits;
- map/index maintenance;
- read-only implementation surface review.

A likely reason is that active edge answers what unresolved thing is currently
being worked. Some work shapes are not centered on an unresolved thing; they are
centered on locating, counting, checking, or reporting bounded evidence.

## Boundary Review

Boundary appears unusually strong across multiple work shapes.

The reviewed evidence suggests several reasons:

1. Boundary prevents authority mixing. Observation, reconciliation, frontier,
   audit, runtime, test, and read-model surfaces carry different authority.
2. Boundary prevents activity mixing. Lookup, navigation, selection,
   recommendation, observation, reconciliation, and implementation support can
   touch the same artifacts while doing different work.
3. Boundary prevents execution leakage. Runtime-facing documents repeatedly
   distinguish read-only surfaces, recommendations, handoffs, registered
   operations, policy, and execution.
4. Boundary supports weak-pressure work. A bounded lookup or inventory can
   succeed without strong pressure because boundary supplies enough containment
   to avoid drift.
5. Boundary supports high-pressure work. Continuation and handoff need boundary
   to prevent unsafe movement from preserved information into unauthorized or
   unsupported action.

Boundary therefore appears less work-shape-dependent than active edge or next
safe move. It is still not declared universal here; the observation is only that
it recurs across the widest range of reviewed work shapes.

## Duplicate-Work Findings

The repository shows duplicate-work pressure when orientation is reconstructed
rather than preserved.

Observed duplicate-work patterns include:

- repeated reconstruction of orientation bundles around concern, pressure,
  reference point, boundary, active edge, continuation, next safe move,
  selection rationale, and validation state;
- repeated separation of observation from reconciliation, recommendation, and
  implementation proposal;
- repeated clarification that navigation, selection, and continuation are not the
  same activity;
- repeated restatement that read-only summaries or maps are not execution,
  policy, truth, or workflow systems;
- repeated re-inspection of State Summary and Current Work Position surfaces to
  determine whether they preserve active relevance or only display state.

The duplicate-work finding is not that the repository needs a new system. The
observation is that missing work-shape awareness may cause later work to rebuild
which orientation elements matter for the current activity.

## Why The Orientation Bundle Reconfigures

The bundle appears to reconfigure because the failure condition changes by work
shape.

```text
lookup failure: wrong place or unsupported match
inventory failure: wrong population or boundary crossing
observation failure: overclaiming or proposing beyond evidence
reconciliation failure: authority mixing or collapsed distinctions
audit failure: unchecked boundary or unsupported finding
frontier failure: premature settlement or lost unresolved edge
handoff failure: preserved content without resumable position
continuation failure: inability to identify safe next movement
selection failure: chosen path without rationale or candidate boundary
```

The elements that prevent each failure become load-bearing in that shape. Other
elements can remain weak without immediate failure.

## Work-Shape Distinction Review

The critical distinctions mostly survive review, but only as observations:

- `work shape != workflow`: reviewed shapes describe posture and load-bearing
  orientation, not ordered process.
- `work shape != task`: a single user request can contain lookup, observation,
  audit, and continuation support; the shape is not identical to the task.
- `orientation != work shape`: orientation is the relation bundle that helps a
  shape proceed; it is not the category of work.
- `continuation != all work`: some work is terminal, bounded, or reference-like.
- `lookup != resumption`: locating a surface is not the same as restoring active
  position.
- `inventory != active work`: listing what exists does not by itself select what
  is live.
- `observation != recommendation`: reporting evidence does not choose a path.
- `navigation != selection`: exposing paths does not explain why one path is
  active now.

The unresolved part is whether repository language should continue using these
as informal distinctions or whether existing documents already cover them with
other concepts. This document does not decide that.

## Unresolved Observations

- The repository does not establish a canonical list of work shapes.
- The reviewed evidence supports different orientation needs, but not a precise
  minimum subset for each shape.
- Some documents combine shapes, especially State Summary, Current Work Position,
  handoff, and operator understanding surfaces.
- It remains unclear whether Current Work Position should be read as a general
  continuation surface, a work-shape-dependent surface, or both.
- Boundary appears unusually strong across shapes, but this review does not prove
  it is universal.
- Active Edge appears weaker in lookup, inventory, and completed audit work, but
  can become strong if those activities are used to resume an unresolved
  investigation.
- Pressure may be supplied implicitly by a named document, prompt, audit target,
  or bounded question; weak explicit pressure is not always weak orientation.
- Validation state appears under different names across audits, observations,
  summaries, tests, and read-models, making cross-shape comparison uneven.
- Repository evidence does not settle whether work-shape awareness would reduce
  duplicate orientation reconstruction; it only shows repeated reconstruction.

## Closing Observation

Repository evidence supports a cautious answer:

```text
Different repository work shapes appear to orient differently.
```

The evidence is strongest where continuation, handoff, frontier exploration, and
operator understanding fail without richer orientation. It is weaker, but still
visible, where lookup, inventory, map use, and bounded audits proceed with
lighter orientation because boundary, reference point, and validation are enough
for that shape.

The repository therefore does not currently support a universal orientation
subset. It supports an observational pattern in which orientation elements become
load-bearing when their absence would cause the specific work shape to drift,
overclaim, cross authority, duplicate work, or fail to continue.
