# Lexical Recurrence Visibility Investigation

## Purpose and boundary

This investigation asks whether language recurrence can become repository-visible without crossing into semantic interpretation.

This is investigation only.

It does not implement NLP, LLM analysis, semantic search, claim extraction, intent inference, authority inference, concept extraction, ontology promotion, classification systems, recommendation systems, or knowledge graphs.

Repository authority wins.

## Central finding

Repository evidence supports a meaningful distinction between:

```text
language visibility

language recurrence visibility

semantic interpretation
```

The strongest supported position is:

```text
lexical recurrence
is potentially observable
without semantic interpretation
```

provided the observation remains limited to:

```text
strings
terms
phrases
identifiers
references
labels
```

and does not attempt to infer:

```text
meaning
concepts
claims
intent
authority
```

Repository authority wins.

## Relationship to language visibility

Prior investigations established:

```text
language-bearing artifacts exist

documentation contains language

operator requests contain language

language can be preserved

language can be cited

language can be attributed

language can be routed
```

Language visibility answers:

```text
what language exists?
```

Lexical recurrence visibility would answer:

```text
what language recurs?
```

This mirrors the distinction established earlier between:

```text
structure visibility
    -> what structure exists

recurrence visibility
    -> what structure recurs
```

Repository authority wins.

```text

```

Examples:

```text

```

The investigation does not claim these terms represent concepts.

Only that the strings recur.

Repository authority wins.

## Lexical recurrence

Candidate shape:

```text
string
    -> occurrence count
```

Examples:

```text
term
    -> recurrence

phrase
    -> recurrence

identifier
    -> recurrence

reference
    -> recurrence
```

This resembles structural recurrence:

```text
section label
    -> recurrence

front matter key
    -> recurrence
```

but the observed unit differs.

Structural recurrence observes structure.

Lexical recurrence observes language fragments.

Repository authority wins.

## Structural recurrence versus lexical recurrence

### Structural recurrence

Observed unit:

```text
heading
section
front matter key
skeleton
code fence language
```

Question:

```text
what structural elements recur?
```

### Lexical recurrence

Observed unit:

```text
word
term
phrase
identifier
string
```

Question:

```text
what language fragments recur?
```

### Distinction

Structural recurrence can exist even if labels differ.

Lexical recurrence can differ even when structure is identical.

Therefore:

```text
structural recurrence
    !=
lexical recurrence
```

Repository authority wins.

## Lexical recurrence versus semantic recurrence

A stronger distinction appears necessary.

### Lexical recurrence

Examples:

```text
same string

same phrase

same identifier
```

Questions:

```text
does the text recur?
```

### Semantic recurrence

Examples:

```text
same idea

same concept

same meaning
```

Questions:

```text
does the meaning recur?
```

Repository evidence strongly separates these.

Examples:

```text
candidate != fact

visibility != ownership

descriptive language != authority
```

The repository repeatedly avoids promoting language similarity into semantic authority.

Therefore:

```text
lexical recurrence
    !=
semantic recurrence
```

is currently the strongest supported position.

Repository authority wins.

## Visibility opportunities

If lexical recurrence exists, the following operator questions become answerable.

### Frequency

```text
Which terms recur most frequently?

Which phrases recur most frequently?
```

### Distribution

```text
Which terms are common?

Which terms are rare?

Which terms appear once?
```

### Coverage

```text
How broadly is a term used?

How many documents contain a phrase?
```

### Drilldown

```text
Where does a phrase occur?

Which documents contain a term?
```

### Membership

```text
Which documents belong to the set
containing exact term X?
```

These questions appear analogous to existing structural visibility layers.

Repository authority wins.

## Counterexamples

### Recurrence immediately pressures meaning

Many recurring terms are not interesting as strings.

Examples:

```text
preservation

identity

equivalence
```

Investigation pressure naturally becomes:

```text
what do they mean?
```

rather than:

```text
how often do they occur?
```

This is the strongest semantic pressure.

### Term recurrence may be mistaken for concept recurrence

Example:

```text
identity
```

Recurring use of the string does not establish:

```text
identity concept recurrence
```

Different documents may use the same word differently.

Repository authority wins.

### Different strings may represent similar meaning

Examples:

```text
continuation

continuity
```

or:

```text
observation

observed
```

Lexical recurrence would treat these as distinct.

Semantic recurrence would pressure them together.

This boundary demonstrates why lexical recurrence is weaker than semantic recurrence.

### Identifiers are safer than prose

Identifiers such as:

```text
RepositoryArtifactFact

RelationshipFact
```

have clearer exact-string identity.

They create less semantic ambiguity than free-form prose.

Repository authority wins.

## Pressure toward semantics

Lexical recurrence appears capable of remaining observable.

However recurring pressure exists toward:

```text
concept extraction

claim extraction

semantic clustering

ontology promotion

authority inference
```

Examples:

```text
term recurrence
    -> concept candidate

phrase recurrence
    -> shape candidate

vocabulary recurrence
    -> ontology pressure
```

These transitions are not justified by lexical recurrence alone.

Repository authority wins.

## Supported conclusions

1. Language visibility and lexical recurrence visibility are meaningfully distinct.
2. Exact strings, terms, phrases, identifiers, references, and labels appear observable without semantic interpretation.
3. Lexical recurrence is distinct from structural recurrence because the observed unit differs.
4. Lexical recurrence is distinct from semantic recurrence because identical strings do not guarantee identical meaning.
5. Lexical recurrence could support frequency, distribution, membership, coverage, and drilldown style visibility.
6. Lexical recurrence can remain observable if it remains limited to exact language fragments.
7. Recurring pressure exists toward semantics, but lexical recurrence alone does not justify concept extraction, claim extraction, authority inference, or ontology promotion.

## Unsupported conclusions

- Lexical recurrence proves concept recurrence.
- Lexical recurrence proves semantic recurrence.
- Lexical recurrence proves claim recurrence.
- Lexical recurrence proves authority.
- Lexical recurrence proves ontology candidates.
- Recurring vocabulary automatically identifies repository concepts.
- Lexical recurrence can safely collapse synonyms.
- Lexical recurrence can safely infer meaning.

## Acceptance answers

### Can language recur without interpretation?

Supported position:

```text
yes
```

Exact strings can recur without requiring interpretation.

### What is lexical recurrence?

```text
recurrence of exact observed language fragments
```

such as:

```text
words
terms
phrases
identifiers
references
```

### How is lexical recurrence different from structural recurrence?

```text
structural recurrence
    -> recurring structure

lexical recurrence
    -> recurring language fragments
```

### How is lexical recurrence different from semantic recurrence?

```text
lexical recurrence
    -> same text

semantic recurrence
    -> same meaning
```

### Can lexical recurrence remain observable rather than interpretive?

Potentially yes, if bounded to exact language fragments and recurrence statistics.

### What visibility would lexical recurrence add?

```text
frequency

distribution

membership

coverage

drilldown
```

for language fragments.

### Where does lexical recurrence create pressure toward semantics?

Most strongly at:

```text
concept extraction

semantic clustering

claim extraction

authority inference

ontology promotion
```

where recurring strings begin to be treated as recurring meaning.

Repository authority wins.
