# Recurrence-First Shape Discovery Investigation

## Central finding

Repository evidence partially supports a recurring pattern:

```text
observation
    -> recurrence
    -> candidate
    -> investigation
    -> supported distinction
```

This differs from:

```text
definition
    -> promotion
```

Many successful inquiry branches appear to begin with repeated observations before stable terminology exists.

Repository authority does not support automatic promotion.

Repository authority does support accumulation of supporting evidence.

## Concept-first versus recurrence-first

### Concept-first

Shape:

```text
term exists
    ->
definition proposed
    ->
repository evaluates fit
```

Pressure originates from terminology.

The repository then attempts to determine whether the term deserves authority.

### Recurrence-first

Shape:

```text
observation repeats
    ->
pattern noticed
    ->
candidate distinction emerges
    ->
terminology stabilizes later
```

Pressure originates from evidence.

Terminology follows observation.

### Repository comparison

Recent investigations repeatedly followed the second pattern.

Examples:

```text
preservation

continuation

state

identity

equivalence
```

The investigations did not begin with stable definitions.

They began with recurring observations appearing across documents, audits, reconciliations, and inquiry branches.

## Historical repository examples

### Preservation

Observed recurrence:

```text
unknown preservation

classification preservation

evidence preservation

relationship preservation

context preservation
```

The recurrence appeared before preservation received a stable working description.

Investigation followed recurrence.

### Preservation versus continuation

The distinction emerged from repeated friction.

Observed pattern:

```text
some findings appeared state-oriented

some findings appeared time-oriented
```

Only later did terminology stabilize around:

```text
preservation

continuation
```

### State versus identity

Observed recurrence:

```text
same thing

changed state
```

and:

```text
same state

different thing
```

The distinction emerged through counterexamples.

The terminology became stable after repeated observation.

### Justified equivalence

Observed recurrence:

```text
related
    != identical
```

appeared repeatedly in identity audits.

Only later did the inquiry branch converge on:

```text
boundary creation

boundary collapse

equivalence
```

Again, recurrence preceded stable terminology.

Naming followed.

## What would a shape observation mean?

Repository evidence does not support ontology promotion.

However it partially supports a weaker possibility.

### Candidate definition

A shape observation is:

```text
recurring structure
observed across evidence
without requiring ontology promotion
```

### Candidate examples

#### Recurring contrast

```text
A
    !=
B
```

Examples:

```text
boundary != unknown

state != identity

preservation != continuation

relationship != identity
```

The recurrence is the contrast pattern.

Not the specific terms.

#### Recurring relation

Examples:

```text
boundary
    -> limitation
    -> unknown
```

```text
preservation
    -> retained state
```

```text
continuation
    -> retained identity
```

The recurrence is the relational structure.

Not necessarily the promoted concepts.

#### Recurring inquiry form

Examples:

```text
A without B

B without A

counterexample search

boundary investigation
```

These inquiry forms appear repeatedly.

#### Recurring evidence pattern

Examples:

```text
observation
    -> counterexample
    -> distinction
```

```text
observation
    -> reconciliation
    -> boundary
```

The recurrence may exist independently of specific terminology.

## Observation, evidence, fact, claim, candidate, shape, ontology

Repository evidence suggests meaningful separation.

### Observation

```text
something noticed
```

### Evidence

```text
supporting material
for an observation
```

### Fact

```text
supported repository statement
```

### Claim

```text
assertion evaluated against evidence
```

### Candidate

```text
not yet supported enough
for authority
```

### Shape

Candidate interpretation:

```text
recurring structure
not yet ontology
```

### Ontology

```text
repository-governed concept
relation
state
authority
behavior
or claim target
```

Repository authority currently appears much stricter for ontology than for observations.

## Can shape discovery remain non-promoted?

Repository evidence supports:

```text
possibly
```

Reason:

The repository already contains categories that remain candidate-oriented.

Examples:

```text
candidate

unsupported claim

investigation

reconciliation
```

These can exist without promotion.

A shape observation appears closer to:

```text
candidate observation
```

than:

```text
ontology declaration
```

Repository evidence currently supports the candidate interpretation more strongly.

## Why begin with repository prose?

Repository evidence strongly supports this.

### Shared vocabulary

Repository prose already contains:

```text
repository authority

boundary

unknown

support

claim

evidence
```

### Shared authority model

Repository prose follows:

```text
observation

evidence

support

counterexample
```

This creates a common evaluation framework.

### Shared investigation style

Recent inquiry work repeatedly uses:

```text
counterexamples

boundary tests

non-equivalence tests

relationship tests
```

### Shared document structure

Inquiry reports repeatedly use:

```text
finding

counterexample

supported conclusion

unsupported conclusion
```

### Shared evidence expectations

Repository prose already understands:

```text
repository authority wins
```

This makes recurrence easier to evaluate consistently.

### External prose problem

External prose introduces:

```text
unknown authority

unknown terminology

unknown assumptions
```

Repository prose provides a controlled environment.

## Counterexamples

### Recurrence that may be noise

Not all repetition is meaningful.

Examples:

```text
frequently used wording

fashionable terminology

author preference
```

may recur without describing structure.

Recurrence alone is insufficient.

### Important concepts without recurrence

Repository evidence supports some concepts because they are architecturally necessary.

Examples may include:

```text
specific authority boundaries

specialized implementation owners
```

These may be important without broad recurrence.

Therefore:

```text
importance
    != recurrence
```

### Risk of ontology expansion by another name

A major counterexample is:

```text
recurrence
    automatically implies promotion
```

Repository evidence does not support that.

The repository repeatedly requires:

```text
support

counterexample review

boundary validation
```

before stronger claims are accepted.

Therefore recurrence alone cannot justify ontology expansion.

## Supported conclusions

1. Recent successful inquiry branches often began with recurring observations rather than stable definitions.
2. Several distinctions appeared before terminology stabilized.
3. Repository inquiry already behaves more like evidence accumulation than immediate promotion.
4. A repository-compatible shape observation could be interpreted as recurring structure observed across evidence.
5. Repository prose is a stronger initial corpus than external prose because vocabulary, authority boundaries, and evidence expectations are already shared.
6. Recurrence-first discovery appears compatible with existing observation -> evidence -> support philosophy.
7. Recurrence does not automatically justify promotion.
8. Shape observations can be interpreted as candidate-level observations rather than ontology declarations.

## Unsupported conclusions

- Every recurrence is meaningful.
- Every recurrence is a shape.
- Every shape deserves ontology promotion.
- Stable terminology is unnecessary.
- Repository prose is sufficient for all future inquiry.
- Shape observations already exist as a first-class repository concept.
- Recurrence alone is evidence of truth.

## Acceptance answers

### Do successful repository discoveries tend to emerge from recurrence?

Partially yes.

Many recent inquiry branches began with repeated observations before stable terminology existed.

### Can recurring structures emerge before stable terminology?

Repository evidence supports yes.

Recent preservation, continuation, identity, and equivalence work displayed recurring distinctions before terminology stabilized.

### What would a repository-compatible shape observation mean?

A recurring structure observed across evidence without requiring ontology promotion.

### Can shape discovery remain candidate-only and non-promoted?

Repository evidence supports that possibility.

The candidate interpretation is currently stronger than an ontology interpretation.

### Why begin with repository prose instead of external prose?

Shared vocabulary, authority boundaries, investigation style, document structure, and evidence expectations already exist.

### Does recurrence-first discovery fit existing repository philosophy?

Yes, at least partially.

The repository already prefers:

```text
observation
    -> evidence
    -> support
```

over:

```text
definition
    -> immediate authority
```

Recurrence-first discovery appears closer to the existing philosophy than concept-first promotion.
