# Unresolvedness Observation

## Status

Exploratory observation only.

This document investigates whether recent inquiry work exposed a deeper
repository pattern around `unresolvedness`. It does not modify implementation,
Inquiry Orientation, State Summary, continuation behavior, handoff behavior,
runtime concepts, schemas, policy, ontology, canonical terminology, or repository
priority.

`Unresolvedness` is not treated here as canon, ontology, runtime state, a
first-class primitive, or a proposed implementation concept. The term is tested
as descriptive language for a recurring pattern in prior inquiry, active-edge,
current-work-position, continuation, handoff, concern, visibility, knowledge
navigation, understanding, learning, attention, and pressure documents.

Candidate observation under investigation:

```text
State
    contains what is known

Lens
    provides ways of viewing what is known

Orientation
    relates participants to what is currently relevant

Inquiry
    may move through unresolvedness
```

This candidate is intentionally weaker than a conclusion. Repository authority
wins wherever the candidate fails.

## Question

Central questions:

```text
What is unresolvedness?

Does repository evidence support unresolvedness as a useful descriptive concept?

How does unresolvedness relate to inquiry, concern, active edge, current work
position, orientation, and continuation?
```

A secondary question follows from the recent `Inquiry As Movement Observation`:
if inquiry may be movement, pursuit, navigation, or relation to a gap rather than
an object, what is inquiry moving through or relative to? `Unresolvedness` is one
candidate answer, not an assumed answer.

## Unresolvedness investigation

Repository evidence supports `unresolvedness` as useful descriptive language,
but not as a settled concept with a single definition.

The strongest evidence is cumulative rather than lexical. Recent inquiry work
repeatedly describes inquiry through active questions, gaps, tensions, unknowns,
investigation targets, comparison requests, pressure, pursuit, and movement. The
`Inquiry As Movement Observation` explicitly tests whether inquiry may be
movement through unresolvedness, but it preserves that framing as exploratory.
This document therefore treats that prior observation as evidence of a recurring
pattern, not as authority that unresolvedness exists as a separate concept.

The repository pattern is broader than ordinary uncertainty. Several inspected
documents describe situations where something may be known, visible, or preserved
while still not settled enough for continuation. Examples include:

```text
known information without clear current work position
preserved references without active continuation
visibility loss without ownership or expectation authority
candidate comparison reference without accepted baseline authority
active question or gap without implementation priority
inquiry note preserved as prose without inferred intent
handoff preserving unresolved tensions without creating authority
```

These examples suggest that unresolvedness may name a relation or condition in
which work has not yet settled, rather than only a missing fact. It can appear as
some combination of:

```text
gap
difference
uncertainty
unknown
incomplete relation
missing comparison
unfinished movement
unsettled understanding
unsettled authority
unsettled continuation position
```

No inspected evidence supports forcing these into one definition. A safer
reading is that `unresolvedness` may be an umbrella description for conditions
that keep inquiry, attention, concern, or continuation from being complete.

## Concern relationship

The visibility branch supports concern as operator-significant interpretation,
not as ontology, alerting, remediation, policy, expectation, or ownership.

The branch progression is important:

```text
visibility
    ->
continuity
    ->
candidate baseline
    ->
accepted baseline
    ->
concern
```

More precisely, repository reconciliation says concern may emerge at several
points in or beside that chain. Visibility change, continuity break, baseline
deviation, accepted-reference deviation, expectation violation, ambiguity,
staleness, degraded evidence quality, or operator investigation context can all
produce concern without collapsing concern into expectation or ownership.

That evidence supports a cautious relationship:

```text
concern may be attention or significance directed toward unresolvedness
```

However, concern is not merely another name for unresolvedness. Concern appears
when unresolvedness or difference matters in a scoped operator context. Some
unresolvedness may remain inert, archival, theoretical, or not currently
operator-significant. Some concern may arise from clearly known bad state rather
than from unknowns. The safer phrasing is:

```text
Concern can expose, select, or intensify unresolvedness, but does not equal it.
```

Concern can also behave like pressure created by unresolvedness. The visibility
branch began from operator pain: storage surfaces appeared empty despite
measurements, identity boundaries blocked expected visibility, and mount-loss
examples mattered operationally before ownership or expectation authority was
available. That pressure did not authorize a new policy or runtime behavior, but
it did make the unresolved boundary worth investigating.

## Active Edge relationship

`Active Edge Frontier` asks what is currently pulling work forward. Its recurring
vocabulary includes active question, active gap, active contradiction, active
frontier, active relationship, selected tension, active concern, current work
position, next safe move, and continuation point.

This is strong evidence that Active Edge is adjacent to unresolvedness. The
active edge often appears where unresolvedness is concentrated enough to pull
work forward. It may also appear where unresolvedness is visible enough to be
carried through continuation.

Possible exploratory relationship:

```text
Active Edge may be the currently live edge of unresolvedness.
```

But this should remain weaker than a definition. The same frontier explicitly
keeps open whether Active Edge is an object, role, operation, relationship,
current work position, attention state, selection result, priority, or inquiry
object. It also warns against confusing active edge with repository priority.
Some active edges may involve risk, authority boundary, validation state,
implementation cleanup, or next-safe-move pressure rather than pure unknownness.

A safer observation is:

```text
Unresolvedness helps explain why an edge can be active, but does not by itself
explain selection, priority, authority, or safe movement.
```

## Current Work Position relationship

`Current Work Position Frontier` asks what position current work occupies and
what must survive for work to feel continuous. It characterizes current work
position as a possible selected, bounded, continuation-relevant orientation
around what is active, why it is active, what unresolved pressure it occupies,
what boundaries constrain it, and what movement is safe.

That evidence supports the candidate that Current Work Position may describe the
current location or relation of work to unresolvedness. It is not merely the
unresolvedness itself. It also includes selection, boundary, rationale,
authority, validation state, known non-goals, and next-safe-move information.

Possible relationship:

```text
unresolvedness = unsettled terrain or pressure
current work position = where the work currently stands relative to that terrain
```

This remains exploratory. Some current work positions may be anchored in
implementation cleanup, branch closure, accepted reference preservation, or
handoff consumption rather than an abstract gap. Also, repository evidence warns
that current work position is not established ontology and should not become a
silent authority carrier.

## Inquiry relationship

Prior inquiry work supports separating `inquiry note` from `inquiry`. The
implementation stores `InquiryNoteRecord` as raw prose and renders an
`InquiryOrientationView`; it does not implement an `Inquiry` runtime object. The
authority boundary says the note is not a fact, claim, goal, tool need,
requirement, capability, decision, proposal, plan, authorization, command,
runtime instruction, intent, concern, recommended action, or next safe move.

Unresolvedness helps explain that distinction:

```text
inquiry note = preserved expression related to unresolvedness
inquiry = possible movement, pursuit, or relation around unresolvedness
orientation view = read-only relation between the note and already projected or
preserved material
```

This framing also helps explain continuation and handoff behavior. Continuation
can fail even when information is preserved because the unresolved edge, live
reasoning branch, selection rationale, or next safe move may be missing. Handoff
can preserve unresolved tensions, active investigation, navigation intent, and
safe movement without creating the authority behind those items.

Unresolvedness may therefore explain why inquiry-as-object evidence remains
weak: the repository currently preserves notes, views, frontiers, findings,
handoffs, working state, and current positions around inquiry-like movement, but
it does not yet settle a single inquiry object.

## Orientation relationship

Orientation appears to relate participants to what is currently relevant rather
than to create State truth or runtime authority. Lens/orientation work suggests:

```text
State answers what exists in projected state.
Lens answers how projected state or preserved knowledge is being viewed.
Orientation answers where attention is currently directed or how a participant is
related to relevant knowledge.
```

Unresolvedness fits this only as a possible target or pressure of orientation.
Orientation may orient a participant toward unresolvedness, away from it, around
its boundary, or through a selected portion of it. But orientation is broader: it
can involve relevance, attention, activation, continuity, participant relation,
and safe continuation even where the underlying matter is already known.

## Continuation and handoff relationship

Continuation documents repeatedly show that preserved information is not enough.
A future participant may know the architecture, references, and summary while
still lacking the current work-position needed to continue. Continuation context
therefore preserves active work, current investigation, live reasoning branch,
open question or tension, risks, constraints, and next safe move.

Unresolvedness helps describe what must survive in those cases:

```text
not just what was known
but what remained unsettled enough to govern the next move
```

Handoff documents similarly preserve selected active context, unresolved
tensions, rationale, boundaries, authority trail, and continuation instructions.
That does not make unresolvedness a handoff object. It suggests that handoff may
need to preserve a relation to unresolvedness so continuation does not become
reconstruction.

## Alternative explanations

### Unresolvedness is merely overloaded language

Supported in part. The term can easily absorb too much: uncertainty, gap,
frontier, active edge, concern, tension, unfinished work, and unclear authority.
This document should not normalize it as canonical vocabulary. Its value, if any,
is as a temporary observation label for a recurring pattern.

### Unresolvedness is another name for uncertainty

Not fully supported. Uncertainty is one form of unresolvedness, but repository
examples include known differences, visible changes, preserved but inactive gaps,
accepted-reference questions, incomplete relations, missing authority, and
continuation positions. These are not all simple uncertainty.

### Unresolvedness is another name for concern

Not supported. Concern requires scoped operator significance or interpretive
importance. Unresolvedness can exist without currently becoming concern, and
concern can arise from a known condition rather than an unknown one.

### Unresolvedness is another name for Active Edge

Not supported. Active Edge asks what currently pulls work forward. Unresolvedness
may be the material, terrain, or pressure at that edge, but active edge also
includes activation, selection, attention, next safe move, and current focus.

### Unresolvedness is another name for frontier

Not supported. Frontier documents often preserve unresolvedness, but a frontier
is a documentation or investigation boundary. Unresolvedness may appear inside a
frontier without being identical to frontier form.

### Inquiry-as-movement is incorrect

Still possible. The repository contains object-like inquiry evidence in frontier
language, lineage language, inquiry notes, and orientation views. Current
evidence is stronger for movement than for a runtime inquiry object, but the
movement framing remains exploratory and could be revised by later
reconciliation.

### No separate concept exists

Possible and plausible. Existing terms may already be enough:

```text
question
gap
tension
unknown
concern
frontier
active edge
current work position
continuation context
handoff context
orientation
```

If those terms can carry the distinctions without confusion, unresolvedness may
remain unnecessary vocabulary.

## Uncertainties

- Whether `unresolvedness` adds clarity or merely bundles existing terms.
- Whether unresolvedness is best described as a condition, relation, pressure,
  terrain, lack, gap, or interpretive effect.
- Whether concern can ever be cleanly separated from unresolvedness in operator
  contexts where significance is what makes unresolvedness visible.
- Whether Active Edge is best read as concentration of unresolvedness,
  visibility of unresolvedness, selected pressure, or a separate continuation
  phenomenon.
- Whether Current Work Position should be phrased as location relative to
  unresolvedness or as broader selected orientation that only sometimes includes
  unresolvedness.
- Whether inquiry-as-movement will survive later inquiry ontology work.
- Whether using `unresolvedness` risks introducing ontology by implication even
  when the document explicitly refuses that move.

## Non-conclusions

This document does not conclude that:

```text
Unresolvedness is canon.
Unresolvedness should become ontology.
Unresolvedness should become runtime state.
Unresolvedness should become a first-class repository primitive.
Unresolvedness should be added to Inquiry Orientation.
Unresolvedness should be added to State Summary.
Unresolvedness should change continuation or handoff behavior.
Concern is unresolvedness.
Active Edge is unresolvedness.
Current Work Position is unresolvedness.
Inquiry is definitely movement through unresolvedness.
```

The strongest non-conclusion is that repository evidence currently supports
`unresolvedness` only as an exploratory descriptive lens over recurring inquiry,
concern, active-edge, current-work-position, continuation, and handoff patterns.
It may help explain why inquiry increasingly appears as movement, pursuit,
navigation, relation to a gap, or relation to unfinished understanding rather
than as an object. It does not yet deserve architectural promotion.
