# Documentation Compression Observation

## Purpose

This observation investigates where repository documentation appears to have
compressed distinctions, and how those compressions were discovered.

It is an observation, not a frontier, reconciliation, terminology standard,
vocabulary proposal, ontology proposal, governance recommendation, workflow
proposal, schema proposal, or implementation proposal.

Compression is not treated as error. A compressed term can be useful when it
keeps communication compact, preserves an unsettled question, or lets work begin
before every distinction is visible. The question here is narrower: where did a
term appear singular, what later pressure exposed multiple meanings, and what
can repository evidence show about that discovery path?

## Method

The observation reviewed repository documentation surfaces that preserve either
explicit distinctions or evidence of distinction-emergence pressure. The review
included foundational ontology documentation, claim-oriented architecture,
inquiry and attention frontiers, trigger and target frontiers, object / role /
operation work, persistence and continuity work, current-work-position and
active-edge work, lineage observations, discovery-path preservation, observation
surface limits, and adjacent claim-support and working-state documents.

This document preserves documentation authority boundaries. It does not reconcile
terms across documents and does not decide which vocabulary should win where two
documents use different language.

## High-Level Finding

Repository documentation repeatedly shows a pattern in which an initially useful
term becomes less precise under investigation pressure. The pressure is usually
not abstract wordsmithing. It appears when a document needs to answer a concrete
question the compressed term cannot answer safely:

1. Is this represented as reality, claim, fact, support, or authority?
2. Is this preserved information, surviving pressure, lineage, identity, or
   continuity?
3. Is this an active work position, a historical summary, a roadmap status, or a
   handoff artifact?
4. Is attention being moved by a trigger, or is something receiving attention as
   a target?
5. Is lineage an artifact relationship, an inquiry relationship, an observation
   surface, or a discovery path?
6. Is an apparent object actually an object, a contextual role, an operation, or
   a relation?

The strongest observed discovery pattern is:

accepted term -> challenge or task pressure -> contradiction, ambiguity, or
insufficient explanatory power -> hidden distinction exposed -> terminology
expands -> later understanding changes.

The pattern is not universal. Some compressed language remains useful as an
umbrella, especially when the repository is intentionally preserving uncertainty
rather than resolving it.

## Compression Candidate Findings

### Fact

`Fact` is one of the strongest compression examples. The foundational ontology
now distinguishes claim and fact explicitly: claims are central knowledge
primitives, facts are normalized claim forms, and `Claim != Fact` is a core
boundary. Earlier fact-oriented language appears to have come under pressure
when repository work needed to distinguish reality from representation,
assertion from verification, and support from authority.

The strongest pressure came from claim-centered architecture and reality / fact /
claim reconciliation. The pressure question was not merely what word to use. It
was whether Seed could safely represent uncertain, stale, contradicted, sourced,
or future-facing knowledge if `fact` carried too much authority. Claim-support
work then exposed another hidden distinction: a claim is not its support, and
support strength is not identical to assertion strength or truth.

Observed decompression:

- fact -> claim;
- fact -> normalized claim form;
- claim -> supportable assertion;
- support -> evidence / authority / strength relationship;
- contradiction -> pressure between claims rather than simple fact replacement.

Compression remains useful where implementation or projection surfaces need a
compact normalized representation. It becomes risky where readers might treat a
represented fact as reality, verification, or authority.

### Working State

`Working state` is another strong compression example. The continuation context
reconciliation treats working state as the immediate work-position needed after
interruption, but it also separates it from repository state, architecture,
summary, transcript, status, roadmap, and authority. Later continuity and
current-work-position work places pressure on the phrase by asking what must
survive for active work to resume.

The pressure was continuation failure: preserved documents, summaries, or facts
can exist while the participant still cannot tell what is active, what question
is live, what constraint matters next, or what should not be retried without
revalidation. That pressure exposed that `working state` could mean stored
context, activity context, current work position, active edge, validation state,
selection rationale, or next safe move.

Observed decompression:

- working state -> continuation context;
- working state -> activity context;
- working state -> current work position;
- current work position -> selected active context, live frontier, constraints,
  next safe move, and unresolved pressure;
- active edge -> the currently pulling boundary among preserved concerns.

Compression remains useful when orienting a participant quickly. It becomes less
precise when the repository needs to ask whether the active position survived,
whether status is enough, or whether a handoff preserved actionable orientation.

### Lineage

`Lineage` expanded into multiple surfaces. The documentation lineage observation
initially treated lineage as visible relationships among recent documentation
clusters: dependencies, routing, downstream references, bridge documents,
consolidations, and follow-on frontiers. The observation surface audit then
placed pressure on that result by asking what the lineage observation could and
could not see. The lineage distinction observation decompressed the term further
into artifact lineage, inquiry lineage, observation lineage, and discovery-path
lineage.

The pressure was evidentiary rather than terminological. A reference edge,
frontmatter dependency, index placement, or commit grouping can show a visible
artifact relationship without proving causal discovery order, prompt influence,
observer working memory, or the exact path by which understanding changed.
Discovery-path preservation added another pressure: final findings can preserve
what was concluded while losing the challenge, contradiction, or assumption that
made the conclusion understandable.

Observed decompression:

- lineage -> artifact lineage;
- lineage -> inquiry lineage;
- lineage -> observation lineage;
- lineage -> discovery-path lineage;
- lineage evidence -> dependency, routing, prose reference, commit grouping,
  repeated downstream reference pattern;
- lineage blind spot -> prompt lineage, observer working memory, pre-metadata
  history, executed observation history.

Compression remains useful when routing readers through document families. It is
less precise when asked to prove causality, preserve critique, reconstruct
inquiry movement, or explain how a distinction was discovered.

### Attention

`Attention` appears to have been useful as a broad selection concept before the
repository separated why attention moves from what receives attention. The
selection and attention frontier established active selection pressure. The
attention trigger frontier investigated why attention shifts or becomes active.
The attention target frontier investigated what receives attention.

The pressure came from selection and inquiry work: it was not enough to say that
a concern had attention. Repository work needed to distinguish the cause or
condition that made something salient from the object, claim, gap, frontier, or
operation being attended to.

Observed decompression:

- attention -> selection pressure;
- attention -> trigger;
- attention -> target;
- attention -> current focus or active edge;
- attention target -> candidate object / role / operation pressure.

Compression remains useful for ordinary navigation. It becomes less precise when
asking whether a frontier is active because of operator intent, contradiction,
risk, missing evidence, available capability, or inherited inquiry pressure.

### Persistence

`Persistence` came under pressure when the repository had to distinguish storage,
identity, survival, continuity, lineage, and work-position preservation. The
persistence frontier explicitly separates persistence from storage and identity.
The continuity frontier then treats persistence as insufficient for questions
about surviving understanding, pressure, relation, or work position across
change.

The pressure came from continuation and frontier succession. Stored artifacts may
survive without preserving active orientation. A question, contradiction,
finding, or frontier pressure may survive even when the original wording or
artifact changes. That makes persistence useful but not always enough.

Observed decompression:

- persistence -> survival of represented item or pressure;
- persistence -> not storage;
- persistence -> not identity;
- continuity -> recognizable survival through change;
- lineage -> evidence or explanation for continuity, not continuity itself;
- current work position -> preservation of active orientation, not just stored
  material.

Compression remains useful when asking whether something endured at all. It is
less precise when asking what survived, whether it remained recognizable, and
whether a future participant can use it.

### Observation

`Observation` is a stable foundational term, but repository evidence shows
pressure around what an observation artifact preserves. Observation can name an
evidence-producing act or artifact, while discovery-path work asks whether the
challenge that produced a finding is preserved. The observation surface audit
also shows that a documentation observation can see some surfaces while missing
others.

The pressure was critique preservation. A final observation can record a finding
without preserving the discovery path: what was challenged, what assumption was
exposed, what contradiction mattered, and why the final distinction became
visible. This did not replace observation; it exposed a neighboring preservation
concern.

Observed decompression:

- observation -> observed claim / artifact output;
- observation surface -> what the observation could see;
- blind spot -> what the observation could not see;
- discovery path -> challenge, contradiction, assumption exposed, boundary
  clarified;
- critique -> pressure that makes hidden distinctions visible.

Compression remains useful because not every observation must become a discovery
retrospective. It becomes less precise where future readers need to understand
how a finding became visible rather than only what the finding says.

### Object, Role, And Operation

`Object` is a strong additional compression example. The object / role /
operation frontier and consistency audit show that many candidate concepts can
look object-like until placed under pressure. The pressure test asks whether a
candidate is a durable thing, contextual participation, operation, relationship,
or multiple of these at once.

The pressure came from concepts such as evidence, claim, support, inquiry,
attention, persistence, and lineage needing different treatment depending on
whether they are represented as stable objects, roles in a context, operations,
or relationships. The repository did not resolve all candidates; it preserved
the pressure.

Observed decompression:

- object -> durable represented thing;
- role -> contextual participation or function;
- operation -> transformation or action-like process;
- relationship -> edge or connection claim;
- candidate concept -> may occupy more than one interpretation under pressure.

Compression remains useful when a broad concept must be discussed before its
role is known. It becomes less precise when architecture needs to decide whether
something should be stored, related, executed, selected, or treated as a claim.

### Understanding

`Understanding` appears to remain under compression pressure. Continuity work
uses phrases such as surviving understanding, but then notes that the phrase may
be too narrow because what survives can include unresolved gaps, contradictions,
authority boundaries, work positions, evidence, provenance, relationships,
purpose, or need pressure.

The repository evidence therefore suggests `understanding` may currently carry
multiple meanings: represented knowledge, recognized continuity, live inquiry
orientation, preserved pressure, ability to resume work, and explanatory meaning.
This observation does not propose a replacement. It only notes that the term is
still under pressure.

## Critical Examples

### Example 1: Fact -> Claim

Pressure exposed: fact-oriented language could not safely represent support,
staleness, contradiction, authority boundary, and reality / representation
separation. Claim-centered architecture decompressed `fact` by making claims the
central primitive and treating facts as normalized claim forms rather than direct
truth transfer.

Discovery mode: architectural contradiction and authority pressure.

### Example 2: Working State -> Current Work Position

Pressure exposed: preserving documents, summaries, or status does not guarantee
that a future participant knows what is currently active. The need to resume work
safely exposed a distinction between stored context and current work position.

Discovery mode: continuation failure and handoff pressure.

### Example 3: Lineage Expansion

Pressure exposed: documentation references, dependencies, routing, and commit
grouping show visible artifact relationships, but they do not prove inquiry
movement, observer memory, prompt influence, discovery path, or causality.
Lineage therefore expanded into artifact, inquiry, observation, and discovery
path surfaces.

Discovery mode: evidence-surface audit and causality caution.

### Example 4: Observation -> Discovery Path

Pressure exposed: an observation can preserve a final finding while losing the
critique path that made the finding understandable. Discovery-path preservation
therefore distinguished final finding from challenge, contradiction, assumption
exposed, and boundary clarified.

Discovery mode: critique-driven decompression.

### Example 5: Attention -> Trigger / Target

Pressure exposed: saying that something has attention does not say why attention
moved or what is receiving it. The trigger and target frontiers split this broad
term into at least cause-of-attention and object-of-attention concerns.

Discovery mode: selection and inquiry pressure.

## Required Findings

### Strongest Compression Examples

1. `Fact`, because repository authority now explicitly separates claim, fact,
   support, authority, contradiction, and reality.
2. `Working state`, because continuation work repeatedly distinguishes current
   position from summary, transcript, status, roadmap, architecture, and stored
   repository state.
3. `Lineage`, because later observations identify multiple lineage surfaces and
   evidence limitations.
4. `Attention`, because trigger and target became separate frontiers.
5. `Persistence`, because continuity work shows storage and identity are not
   enough to explain survival through change.

### Strongest Distinction-Emergence Examples

- claim / fact / support / authority;
- working state / current work position / active edge;
- artifact lineage / inquiry lineage / observation lineage / discovery-path
  lineage;
- attention trigger / attention target;
- persistence / continuity / identity / storage / lineage;
- object / role / operation / relationship.

### Strongest Critique-Driven Decompressions

- observation -> discovery path, because critique preservation asks what
  challenge made the finding visible;
- lineage -> observation surface and blind spot, because a prior observation was
  audited for what it could not see;
- fact -> claim, because claim-centered architecture arose from pressure against
  over-authoritative fact language;
- object -> role / operation, because pressure tests challenge candidate object
  stability.

### Strongest Pressure-Driven Decompressions

- continuation failure drove working-state decomposition;
- support and contradiction drove fact decomposition;
- selection pressure drove attention trigger / target decomposition;
- frontier succession and handoff survival drove persistence / continuity
  decomposition;
- evidentiary limits drove lineage decomposition.

### Terms Still Under Compression Pressure

- `understanding`, because it may name knowledge, recognition, continuity,
  explanatory grasp, inquiry orientation, or preserved pressure;
- `continuity`, because its boundary with persistence, lineage, inquiry, handoff,
  and work position remains unresolved in frontier form;
- `active edge`, because it is routed as frontier pressure rather than reconciled
  ontology;
- `lineage`, because artifact, inquiry, observation, and discovery-path surfaces
  are distinct but not fully reconciled;
- `object`, because candidates can remain object-like until role or operation
  pressure is applied.

### Places Where Compression Remained Useful

- `attention` remains useful as a broad navigation term before trigger and target
  questions matter;
- `working state` remains useful for compact continuation orientation;
- `lineage` remains useful for document-family routing;
- `persistence` remains useful for asking whether something endured at all;
- `fact` remains useful in normalized implementation or projection contexts when
  its claim boundary is preserved;
- `understanding` remains useful as a human-readable umbrella while the
  repository avoids premature reconciliation.

## Discovery Pattern Findings

Repository evidence supports a recurring but non-universal discovery pattern:

1. A term is accepted because it is useful enough for current work.
2. A later investigation asks a sharper question.
3. The term proves unable to carry all needed distinctions without ambiguity.
4. A contradiction, continuation failure, evidence gap, selection pressure, or
   audit limitation exposes hidden meanings.
5. A new document names the distinction or preserves the pressure.
6. Later documents route to the new distinction while warning against promoting
   frontier language into canonical ontology too early.

The strongest examples of this pattern are fact -> claim, working state ->
current work position, lineage -> multiple lineage surfaces, attention -> trigger
/ target, and persistence -> continuity.

The pattern is not universal because some documents intentionally preserve
compressed language as frontier vocabulary. In those cases, compression is a way
to keep an unsettled question visible without pretending that the repository has
a canonical answer.

## Tensions Observed

| Tension | Observation |
| --- | --- |
| Compression vs precision | Compressed terms help work proceed, but pressure exposes ambiguity when documents need authority boundaries. |
| Compression vs communication | Umbrella words are easier to route and discuss, but they can hide which distinction a reader needs. |
| Compression vs understanding | A compact final finding can preserve conclusion while losing the challenge that made understanding change. |
| Umbrella term vs distinct concept | Attention, lineage, persistence, and working state remain useful umbrellas even after subdistinctions emerge. |
| Stable vocabulary vs evolving vocabulary | Foundational boundaries stabilize some terms while frontiers intentionally leave others unsettled. |
| Useful simplification vs hidden ambiguity | Simplification is useful until it supports an unsafe inference, such as treating a claim as verified reality. |
| Terminology vs preserved meaning | Changing names is less important than preserving the pressure, distinction, or meaning that caused the change. |

## Unresolved Observations

- Repository evidence does not always preserve the prompt, challenge, or working
  memory that caused a document to be written.
- Commit grouping, routing, and frontmatter can show relationship but cannot by
  themselves prove causality or exact discovery order.
- `Understanding` appears under pressure, but the repository has not yet isolated
  whether it is one concept, several concepts, or only a reader-facing umbrella.
- Continuity, lineage, persistence, current work position, and active edge remain
  connected but not reconciled.
- Some compressed terms may be intentionally preserved because premature
  precision would imply authority the repository does not yet have.
- This observation cannot determine whether every decompression was necessary;
  it can only report where documentation evidence shows pressure and later
  distinction.

## Final Observation

The repository does not merely accumulate new terms. Many new documents appear
after a prior term is pressured by contradiction, continuation need, evidentiary
limits, critique, or selection pressure. The important movement is not vocabulary
expansion by itself. The durable movement is that hidden distinctions become
visible, and later readers gain a better chance of seeing why a once-stable term
was not enough for the question being asked.
