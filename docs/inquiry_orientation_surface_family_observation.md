# Inquiry Orientation Surface-Family Observation

## Status

Exploratory observation only.


## Background

Two investigation branches recently stabilized.

### Inquiry Orientation V1

Implementation inspection established that Inquiry Orientation V1 currently orients against:

```text
projected fact support
source-navigation fact support
```

It does not orient against documentation, observations, reconciliations, frontiers, State Summary materials, or repository prose directly.

### State Summary Decomposition

Recent decomposition work separated several kinds of pressure that had historically landed on State Summary.

Repository evidence now distinguishes at least:

```text
projected-State accounting
lens/view interpretation
HomeOps-style operational inspection
participant-orientation / entrypoint pressure
```

## Observation

These two branches may describe adjacent parts of the same phenomenon.

The State Summary investigations repeatedly found that operators were asking different kinds of questions through the same broad surface.

Examples:

```text
What exists?
What is prominent?
What looks unhealthy?
Where should I begin?
What is relevant right now?
```

Those questions are not obviously owned by the same authority surface.

Meanwhile Inquiry Orientation V1 demonstrated that preserved operator prose can be related to existing repository material without promoting that prose into intent, concern, recommendation, action, plan, or command.

## Candidate Connection

Do not assume this is correct.

Investigate whether the next pressure is not:

```text
better matching
```

but:

```text
surface-family orientation
```

Illustrative shape:

```text
operator prose
    -> orientation
    -> possible surface families
```

rather than:

```text
operator prose
    -> recommendation
```

or:

```text
operator prose
    -> action
```

## Example

Illustrative only.

```text
Why does node115 keep showing up first?
```

Possible reading:

```text
State accounting?        weak

Entity prominence?       stronger

Lens/View pressure?      stronger

Orientation pressure?    possible
```

The value of such a reading would not be determining operator intent.

The value would be helping distinguish which repository surface family appears related to the inquiry.
