# Historical Inquiry Pressure Investigation

## Purpose and boundary

This investigation asks whether repository evidence can distinguish historically important inquiry pressure from currently active inquiry pressure.

This is an investigation only. It does not implement inquiry graphs, workflow systems, closure systems, planning systems, automation, or pressure classification mechanisms.

Repository authority remains implementation-backed behavior, tests, executable diagnostics, and existing repository-visible documents.

## Central finding

Repository evidence supports a meaningful distinction between:

```text
active pressure

residual pressure

historical pressure
```

The strongest supported observation is:

```text
historical importance
    !=
current importance
```

Some pressures appear historically significant because they produced major repository-visible structures.

Those same pressures no longer appear to dominate investigation activity.

## Historical pressure versus active pressure

A useful evidence-backed distinction is:

### Active pressure

A relationship or explanatory gap that continues to be reconstructed repeatedly.

Indicators:

```text
repeated investigation activity

manual reconstruction persists

same question reappears

new investigations expose the same gap
```

### Historical pressure

A relationship that once generated substantial investigation effort but is now largely preserved.

Indicators:

```text
relationship preserved

manual reconstruction uncommon

follow-up investigations move elsewhere

unknowns become localized

pressure appears mostly in historical context
```

### Residual pressure

A middle state.

Indicators:

```text
central pressure addressed

secondary pressure remains

new investigations focus on boundaries,
limitations, coverage, or extensions
```

## Pressure transition indicators

Repository evidence supports several indicators that pressure is becoming historical.

### Relationship preservation

A dedicated repository-visible structure preserves the relationship.

### Reduced reconstruction frequency

Investigations stop repeatedly rebuilding the same explanation.

### Shift in inquiry focus

Follow-up work moves toward adjacent questions.

### Localization of unknowns

Unknowns become narrow and scoped.

### Transformation of pressure

The original pressure becomes less important than new pressures revealed by the solution.

Pattern:

```text
pressure
    -> implementation
    -> transformed pressure
```

rather than:

```text
pressure
    -> implementation
    -> no pressure
```

## Can historical pressure reactivate?

Current repository evidence is limited.

However, evidence does support caution against assuming:

```text
historical
    -> permanently inactive
```

Reasons:

```text
repository evolution occurs

new explanatory layers appear

new relationships become important

coverage can change
```

The strongest supported conclusion is:

```text
reactivation is plausible

not currently demonstrated strongly
```

## Historical pressure versus residual pressure

The distinction appears meaningful.

### Historical pressure

Characteristics:

```text
relationship preserved

major pressure reduced

reconstruction uncommon

future investigations rarely centered there
```

### Residual pressure

Characteristics:

```text
core pressure addressed

related pressure still active

investigations continue nearby
```

Example:

```text
reference pressure
```

may be largely historical inside supported domains while residual pressure remains outside them.

## Counterexamples

### Implementation exists and pressure remains active

Examples:

```text
observation-domain pressure

question -> answering surface

surface -> follow-up surface
```

Related investigations continue identifying these as active reconstruction pressures.

### Historically important pressure that no longer dominates investigations

Examples:

```text
selection rationale pressure

projection understanding pressure
```

These pressures appear repeatedly in historical explanations but less frequently as central investigation drivers.

## Pressure lifecycle assessment

Repository evidence partially supports:

```text
active pressure
    -> investigation
    -> implementation
    -> partially resolved
    -> historical pressure
```

However a stronger supported model is:

```text
active pressure
    -> investigation
    -> finding
    -> supported conclusion
    -> implementation
    -> reassessment
    ->
       historical pressure
       or residual pressure
       or active pressure
       or unknown
```

This accommodates the observed persistence of residual pressure.

## Distinguishing implemented, partially resolved, historical, resolved, and unknown

| State | Evidence pattern |
| --- | --- |
| Implemented | Relationship preservation exists. |
| Partially resolved | Pressure reduced but still visible. |
| Historical pressure | Pressure largely addressed and no longer central. |
| Resolved | Pressure appears substantially absent; repository evidence for complete resolution remains limited. |
| Unknown | Insufficient evidence to classify. |

The strongest distinction is:

```text
implementation
    focuses on structure

historical pressure
    focuses on reduced inquiry demand
```

## Supported conclusions

1. Repository evidence supports distinguishing active pressure, residual pressure, and historical pressure.
2. Several historically important pressures appear substantially addressed.
3. Selection rationale pressure and projection understanding pressure are among the strongest historical-pressure candidates.
4. Observation-domain pressure remains one of the strongest active pressures.
5. Question-to-answering-surface and surface-to-follow-up-surface relationships remain active reconstruction pressures.
6. Historical pressure and residual pressure are meaningfully different states.
7. Implementation alone does not determine pressure state.
8. Pressure appears to transform more often than disappear.
9. Pressure state is better described as a lifecycle than as open versus closed.

## Unsupported conclusions

The evidence does not support these stronger claims:

- Historical pressure can never reactivate.
- Every implemented relationship becomes historical pressure.
- Historical pressure implies complete resolution.
- Residual pressure implies implementation failure.
- Pressure state can be classified mechanically from current evidence.
- Inquiry graphs should automate pressure classification.
- The proposed lifecycle is universal.

## Open questions

- What threshold separates residual pressure from historical pressure?
- Which inquiry classes most commonly become historical?
- Which classes remain persistently active?
- Can historical pressure reactivate after significant repository evolution?
- Is complete resolution observable, or only substantial reduction of pressure?
- Are some inquiry classes inherently resistant to becoming historical?

## Acceptance answers

### Which inquiry pressures have become historical?

Selection rationale pressure, projection understanding pressure, portions of reference pressure, and portions of capability meaning pressure are the strongest candidates.

### Which remain active?

Observation-domain pressure, question-to-answering-surface pressure, surface-to-follow-up-surface pressure, and answer-to-next-investigation pressure remain active.

### How can repository evidence distinguish historical pressure from residual pressure?

Historical pressure shows preserved relationships, uncommon reconstruction, shifted investigation focus, and localized unknowns. Residual pressure still generates nearby investigation activity despite the original pressure being addressed.

### Can pressure reactivate?

Reactivation is plausible and not contradicted by repository evidence, but strong direct evidence remains limited.

### Is pressure state better understood as a lifecycle than as open versus closed?

Yes. Repository evidence supports a lifecycle-oriented interpretation more strongly than a binary state model.

### Would an inquiry graph be able to distinguish historically important work from currently important work using repository evidence alone?

Repository evidence suggests the distinction is meaningful and partially observable through pressure state indicators. This investigation does not evaluate whether a graph implementation could perform that classification.
