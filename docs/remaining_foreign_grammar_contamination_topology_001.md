## Positive constitutional topology governing preservation

A live witness is preservable only when current repository evidence supports this topology:

```text
constitutional kind
↓
bounded family system, only where independently evidenced
↓
exact bounded subject
↓
characterization through relevant major dimensions
↓
localities required by the bounded responsibility
↓
responsible acts, constraints, and relations
↓
current producer → artifact → consumer road
```

The available characterization dimensions are subject/identity, assertion/content, standing, source/provenance, responsibility, authority/warrant, scope/locality, and occurrence/preservation. These dimensions are not a new universal class, pipeline, registry, or schema.

## Direct examination: `project_representation_grammar_applicability(...)`

Current evidence shows an untyped parameter named `handoff`. The function dereferences `handoff.probe_request_id`, `handoff.capability_identity`, `handoff.required_input_representation`, `handoff.requested_output_representation`, and `handoff.to_json_dict()`. Tests build a `SimpleNamespace` with that shape. `ExaminationProbeRequest` exposes a lawful neighbor shape with request id, capability identity, input/output representation, constraints, provenance, and no provider/tool/operation arguments; however its field is `request_id`, not `probe_request_id`.

Classification: **mixed but separable / Unknown at exact type boundary**.

It is not currently a clean direct dependency on `ExaminationProbeRequest`, because the parameter name and required field names do not match the class. It is not enough to introduce a replacement handoff or adapter. Current evidence supports either:

1. a future direct dependency on `ExaminationProbeRequest`, if consumers can use `request_id` as the probe/request reference; or
2. a smaller independently warranted applicability input containing exactly request/probe identity, capability/demand identity, source representation, required target representation, and bounded serialization/provenance.

The current untyped `handoff` shape remains compatible with deleted `OperationalRealizationHandoff` grammar and should not be treated as preserved by intuition.
