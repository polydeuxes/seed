# Repository Visible Inquiry State Investigation

## Central finding

Repository evidence supports a distinction between:

```text
inquiry state
```

and:

```text
inquiry movement
```

The strongest supported observation is:

```text
some inquiry state is already repository-visible

but most inquiry movement remains human-interpreted
```

## What makes inquiry state repository-visible?

A concept appears more repository-visible when it has at least one of these properties:

```text
implementation-backed evidence
surface visibility
diagnostic visibility
repeatable observation
explicit unknown or boundary preservation
```

A concept appears less repository-visible when it exists mainly as prose, section structure, or human interpretation across documents.

## Visibility gradient

### Strongly repository-visible

```text
unknown
boundary
```

These recur in implementation-backed surfaces and investigations.

They are not merely documentation artifacts.

### Partially repository-visible

```text
pressure
finding
```

These recur strongly in investigations and sometimes appear through operational or diagnostic surfaces, but are not consistently represented as standalone repository-visible state.

### Mostly human-interpreted or document-visible

```text
supported conclusion
unsupported conclusion
open question
```

These are highly important inquiry artifacts, but current evidence places them mostly in investigation documents rather than implementation-backed surfaces.

## Repository-visible inquiry state today

The strongest current inquiry-state concepts are:

```text
unknown
boundary
pressure
finding
```

with strongest support for unknown and boundary.

Repository-visible examples include surfaces that preserve:

```text
unknowns
limitations
unsupported domains
read-only boundaries
mutation boundaries
capability pressure
selection evidence
reference evidence
```

This means the repository has begun accumulating inquiry-state-like concepts independent of prose.

## Human-interpreted inquiry state today

The following remain mostly human-interpreted:

```text
supported conclusion
unsupported conclusion
open question
investigation trigger
historical pressure classification
residual pressure classification
pressure transformation classification
```

These concepts recur across investigations, but humans currently infer them by reading documents and connecting claims.

## State versus movement

Inquiry state can exist without movement visibility.

Examples:

```text
unknown
boundary
pressure
```

can be visible as endpoints.

But movement such as:

```text
unknown
    -> investigation target

boundary
    -> narrowed inquiry

finding
    -> new question

conclusion
    -> next investigation

pressure
    -> transformed pressure
```

usually requires human interpretation.

Therefore:

```text
state
    !=
movement
```

## Repository-visible versus human-interpreted inquiry state

### Repository-visible inquiry state

A concept is repository-visible when repository surfaces or implementation-backed evidence expose it without requiring a human to reconstruct an investigation chain.

Examples:

```text
unknown
boundary
surface existence
implementation existence
some pressure evidence
```

### Human-interpreted inquiry state

A concept is human-interpreted when it requires reading prose, comparing investigations, or reconstructing inquiry movement.

Examples:

```text
open question
supported conclusion
unsupported conclusion
pressure transformation
historical pressure classification
```

## Counterexamples

### Inquiry-relevant but not strongly repository-visible

```text
open question
supported conclusion
unsupported conclusion
pressure transformation
```

These matter deeply for inquiry continuity, but current repository evidence keeps them mostly document-visible.

### Repository-visible without much human interpretation

```text
unknown
boundary
unsupported domain
read-only behavior
mutation boundary
```

These are frequently exposed directly by implementation-backed surfaces.

## Documentation artifact versus observation candidate

Some inquiry artifacts are mostly documentation structure:

```text
section headers
acceptance answers
evidence reviewed
```

Some increasingly behave like observation candidates:

```text
unknown
boundary
pressure
finding
```

The evidence supports this distinction, but does not support implementing inquiry observations.

## Historical perspective

Recent inquiry chains suggest:

```text
repository-visible inquiry state
```

is emerging independently from:

```text
human-interpreted inquiry movement
```

Unknowns and boundaries are the strongest examples.

Pressure and finding are weaker but plausible emerging examples.

Supported conclusions, unsupported conclusions, and open questions remain mostly document-backed.

## Supported conclusions

1. Some inquiry-state concepts are already repository-visible.
2. Unknown and boundary are the strongest repository-visible inquiry-state concepts.
3. Pressure and finding are partially repository-visible.
4. Supported conclusion, unsupported conclusion, and open question remain mostly human-interpreted or document-visible.
5. Inquiry state can exist without inquiry movement visibility.
6. Repository-visible state mostly preserves endpoints.
7. Human interpretation currently reconstructs movement between endpoints.
8. The repository has begun accumulating inquiry-state concepts independent of prose, but only partially.

## Unsupported conclusions

- Inquiry state is fully repository-visible today.
- Inquiry movement is repository-visible today.
- Inquiry artifacts should become implementation-backed observations.
- Inquiry graphs should be implemented.
- Supported conclusions are already implementation-backed inquiry state.
- Open questions are already repository-visible without prose.
- The visibility gradient is complete or canonical.

## Open questions

- Are boundaries the strongest bridge between inquiry state and implementation-backed evidence?
- Which inquiry-state concepts remain invisible because they lack repeatable surface evidence?
- Can movement visibility exist without implementing inquiry graphs?

## Acceptance answers

### What inquiry-state concepts are already repository-visible?

Unknown and boundary are strongest. Pressure and finding are partially visible.

### What inquiry-state concepts remain human interpreted?

Supported conclusion, unsupported conclusion, open question, investigation trigger, pressure transformation, historical pressure classification, and residual pressure classification remain mostly human interpreted.

### What distinguishes repository-visible inquiry state from inquiry movement?

Repository-visible inquiry state exposes endpoints. Inquiry movement links endpoints across time or investigations.

### Can inquiry state exist without movement visibility?

Yes. Unknowns, boundaries, and some pressures can be visible while their movement into later inquiry remains human interpreted.

### Has the repository begun to accumulate inquiry-state concepts independent of prose?

Partially yes. Unknowns and boundaries are strongest; pressure and finding are weaker emerging candidates.
