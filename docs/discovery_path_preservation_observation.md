# Discovery Path Preservation Observation

## Purpose

This document investigates whether Seed currently preserves how understanding
changed as well as it preserves what became understood.

It asks whether recent documentation contains evidence for a distinct
preservation concern centered on challenge sequences, critique sequences,
assumption exposure, contradiction-driven shifts, compression removal, and
understanding transitions.

This is documentation observation and critique only. It is not a reconciliation,
not a vocabulary authority, not governance, not workflow, not a schema, not a
runtime design, not an implementation plan, and not an ontology promotion.
Repository authority wins over this document.

## Central Question

Recent Seed documentation preserves findings, audits, observations, and
frontiers. The question is whether those artifacts preserve this other shape
well enough:

```text
accepted understanding
    -> challenge
    -> contradiction or pressure
    -> hidden compression exposed
    -> distinction appears
    -> understanding changes
```

The investigation does not assume that a new category is warranted. It tests
whether repository evidence shows a recurring preservation gap.

## Method

The investigation inspected repository documents that recently recorded or
routed major conceptual shifts, including:

- claim-centric architectural work;
- observation/evidence/change/event and derivation-related inquiry;
- persistence and continuity frontiers;
- current-work-position and active-edge frontiers;
- documentation-lineage observation;
- architectural-findings preservation.

For each example, the investigation attempted to distinguish:

- final finding;
- discovery path;
- challenge applied;
- assumption exposed;
- boundary clarified.

Absence of an explicit discovery path was treated as evidence of a preservation
risk, not proof that the path never existed.

## Existing Artifact Boundaries Under Test

### Audit

Audits in this repository usually evaluate repository state, implementation
state, documentation state, or a scoped boundary against evidence. They can
contain critique, but their preserving force is usually the evaluated condition
and finding.

### Observation

Observation documents preserve repository evidence without promoting the result
to canonical architecture. They are well suited to visible artifact patterns,
including documentation lineage, but they do not necessarily preserve the
challenge that made the pattern visible.

### Frontier

Frontiers preserve unresolved inquiry. They are strong at keeping a question
available without making it implementation-ready. They can record the pressure
that motivated the question, but their main object is the unresolved edge rather
than the transition by which a participant's understanding changed.

### Critique

Critique, as observed in recent repository work, pressure-tests findings,
assumptions, observations, categories, and compressions. It can expose that a
stable-looking conclusion actually hides multiple meanings. Critique is not the
same as audit because it may challenge a framing rather than evaluate a fixed
surface. It is not the same as observation because it may operate on an existing
observation and ask what the observation failed to distinguish.

This document does not define Critique as a new repository category. It records
that critique-like pressure appears repeatedly in the discovery paths below.

### Discovery Retrospective

A discovery retrospective would preserve how understanding changed: what was
accepted, what pressure was applied, what contradiction or compression appeared,
what distinction emerged, and what understanding changed. Existing documents
sometimes contain this information, but the repository does not appear to route
it as a stable artifact family.

This document does not propose creating that family. It only records that the
candidate differs from audit, observation, and frontier in the repository
evidence reviewed here.

## Example Review

### 1. Claim-centric architectural shift

Final finding:

Seed is claim-centric. Observations report, evidence preserves provenance,
claims are central knowledge primitives, facts normalize accepted claims, and
relationships normalize connection claims.

Discovery path:

The repository strongly preserves the conclusion in foundational and knowledge
representation documents. It less directly preserves the earlier transition from
fact-oriented framing toward claim-oriented framing. The visible path is
reconstructible from repeated boundary corrections: fact is not raw reality,
projection is not authority, relationship is not identity, evidence is not the
claim, and contradiction/verification/explanation qualify reasoning rather than
replace claim support.

Challenge applied:

The pressure was not merely "what is a fact?" but whether treating facts as the
primary stable object overcompressed support, uncertainty, provenance,
relationships, contradiction, and revision.

Assumption exposed:

A fact-centered frame can hide that Seed often needs to preserve propositions
before they are accepted, rejected, corroborated, contradicted, revised, or
projected.

Boundary clarified:

Claim, fact, evidence, observation, relationship, and projection became
separable. The final boundary is well preserved; the challenge sequence that
made the claim layer necessary is more scattered.

Preservation assessment:

This is a strong example of the repository preserving what became understood
better than how understanding changed.

### 2. Event without observation pressure and derivation-related inquiry

Final finding:

Knowledge change does not always require a new observation. Acquisition,
derivation, support expansion, revision, refinement, and contradiction discovery
are not identical operations. Derivation-like inquiry asks whether represented
knowledge can support additional represented knowledge.

Discovery path:

The observation/evidence/change/event reconciliation first exposed that more
observations do not necessarily mean more changes, and that changes can occur in
Seed's represented understanding. Later knowledge-change and derivation documents
made the pressure explicit: some new represented understanding comes from
reasoning over existing observations, evidence, claims, relationships,
comparisons, assumptions, and projections.

Challenge applied:

The challenge was applied to an implicit acquisition frame: if no new
observation arrived, did anything new happen? The repository answer became yes,
but only under preserved support, assumptions, caveats, and explanation.

Assumption exposed:

Observation pressure was hiding derivation pressure. Treating every knowledge
change as acquisition would compress support expansion, derivation, revision,
confidence change, and contradiction discovery into one intake-shaped event.

Boundary clarified:

Acquisition answers what entered Seed. Derivation answers what additional
represented understanding is justified by what Seed already preserves. Revision
relates later understanding to earlier understanding. Contradiction discovery is
recognition or derivation of incompatibility, not creation of the contradiction.

Preservation assessment:

The derivation frontier preserves the emergent question. The earlier challenge
sequence is partly recoverable, but only by reading across several documents.
This is a moderate preservation gap.

### 3. Persistence -> continuity

Final finding:

Persistence is not storage and not identity. Continuity asks what survived
through change without requiring strict sameness. Continuity appears weaker than
identity but stronger than resemblance.

Discovery path:

The persistence frontier pressure-tested an unexamined premise: represented
things persist. It exposed that storage, identity, inquiry movement,
relationship survival, finding survival, and working-state survival could not be
collapsed. The continuity frontier then challenged whether persistence was still
too broad or storage-shaped and reframed the central question from "does it
persist?" to "what survived?"

Challenge applied:

The challenge targeted a compressed survival term. "Persist" seemed to carry
storage durability, conceptual survival, identity, lineage, and handoff
continuation at once.

Assumption exposed:

A stored artifact can remain while represented understanding no longer persists;
conversely, a question, gap, or working position can survive through changed
wording or role. Survival could not be explained by bytes, names, or strict
identity alone.

Boundary clarified:

Persistence became a pressure about recognizable survival across time, revision,
role changes, inquiry movement, and handoff. Continuity became the narrower
question of what survived through change.

Preservation assessment:

This is one of the strongest examples of understanding transition. The final
frontiers preserve much of the path, but the repository still routes them as
separate frontier conclusions rather than as a preserved transition sequence.

### 4. Working state -> current work position

Final finding:

Current Work Position is not simply working state. It is a selected, bounded,
continuation-relevant orientation: what is active, why it is active, what
unresolved pressure it occupies, what constraints apply, and what movement is
safe.

Discovery path:

Handoff and continuation work showed that preserved information can be
insufficient for continuation. Persistence and continuity work then asked what
survives. Current-work-position inquiry shifted the focus from stored or
surviving content to resumable orientation.

Challenge applied:

The pressure was applied to the assumption that working state names the relevant
continuation object. If two participants have the same facts but cannot resume
the same work safely, the preserved thing is not just state content.

Assumption exposed:

"Working state" compressed active frontier, constraints, unresolved tensions,
validation state, selection rationale, and next safe moves. The continuation
problem required orientation, not only retained data.

Boundary clarified:

Working state can be part of current work position, but current work position
appears to include selection, boundedness, authority constraints, validation
state, and continuation relevance.

Preservation assessment:

This transition is well represented in frontier content, but the discovery path
is still distributed across handoff-lineage, persistence, continuity, and
current-work-position documents.

### 5. Documentation lineage observation critique

Final finding:

Documentation lineage and inquiry lineage overlap but are not identical.
Documentation lineage is the visible artifact path; inquiry lineage is the
question path. Creating a document does not automatically imply knowledge growth.

Discovery path:

The documentation-lineage observation initially preserved how documents generate
other documents. Its own evidence forced a distinction: artifact lineage could
be observed through dependencies, links, frontmatter, navigation updates, and
commit clusters, while inquiry lineage had to be inferred through question
movement, repeated pressure, conceptual narrowing, and redirects.

Challenge applied:

The critique pressure asks whether artifact succession is being mistaken for
understanding succession. A file can be downstream without carrying a new
inquiry; an inquiry can continue across several documents without a simple
file-to-file chain.

Assumption exposed:

"Lineage" compressed at least artifact lineage, inquiry lineage, and possibly
observation lineage. The observation document records the artifact/inquiry split
but does not fully preserve the critique sequence that exposed the compression.

Boundary clarified:

Artifact lineage, inquiry lineage, and observation lineage should not be assumed
to be the same. This document does not reconcile them; it only records the
preservation risk created when they are compressed.

Preservation assessment:

This is a strong example of critique-driven discovery. The final observation is
preserved; the critique path that made the distinction salient remains mostly
implicit.

### 6. Active edge emergence

Final finding:

Active Edge investigates what currently pulls work forward among preserved
questions, gaps, contradictions, tensions, relationships, frontiers, concerns,
risks, ambiguities, findings, and recommendations. It is exploratory and not
implementation-ready.

Discovery path:

Selection and attention work showed that many unresolved items can exist while
only some become active. Current-work-position work showed that resumable work
needs selected orientation. Active-edge work then applied pressure to the
forward-moving part of that orientation: what makes this preserved concern the
one that currently pulls work onward?

Challenge applied:

The challenge targeted the assumption that preserved frontier, current priority,
working state, or attention target fully explains work movement.

Assumption exposed:

Preserved unresolvedness is not the same as active pull. A frontier can be
recorded without being current. A gap can persist without blocking safe
movement. A contradiction can be known without becoming today's pressure.

Boundary clarified:

Active edge may be related to inquiry, attention, current work position,
relationship pressure, and frontier status, but it should not be collapsed into
any one of them without later reconciliation.

Preservation assessment:

The active-edge frontier preserves the unresolved concept. The transition from
selection/attention and current-work-position pressure into active-edge pressure
is visible but not separately preserved as a discovery path.

## Cross-Example Findings

### Discovery-path findings

The repository does preserve many discovery paths in fragments. Purpose,
method, existing-authority, pressure-test, open-question, and conclusion
sections often let a future reader reconstruct why a document exists.

However, the preservation is usually document-local. A reader can find the final
frontier or final observation faster than the sequence by which an accepted
understanding was challenged and changed.

The recurring pattern appears real in multiple examples:

```text
accepted understanding
    -> challenge
    -> contradiction or pressure
    -> hidden compression exposed
    -> distinction appears
    -> understanding changes
```

Strongest examples:

1. persistence -> continuity;
2. working state -> current work position;
3. documentation lineage -> artifact/inquiry lineage distinction;
4. acquisition/observation pressure -> derivation pressure;
5. selected unresolvedness -> active edge.

### Critique findings

Critique-like work appears as boundary pressure, pressure testing, challenge of
implicit frames, and contradiction exposure. It is especially visible when a
term seems stable until it is asked to explain a neighboring case.

Critique differs from audit in the reviewed evidence because the target is often
not a repository implementation surface but the adequacy of a framing. It differs
from observation because it frequently acts on existing observations or findings
and asks what they compress or fail to distinguish.

The strongest critique-driven discoveries are:

- fact/finding preservation pressure becoming claim-centric architecture;
- observation-only pressure becoming derivation pressure;
- persistence pressure becoming continuity pressure;
- artifact lineage pressure becoming inquiry-lineage distinction;
- preserved frontier/current-position pressure becoming active-edge inquiry.

### Compression findings

Recurring hidden compressions include:

| Compression | Distinctions exposed |
| --- | --- |
| fact as primary stable knowledge | claim, fact, evidence, observation, relationship, projection |
| knowledge change as new observation | acquisition, support expansion, derivation, revision, refinement, contradiction discovery |
| persistence as storage or identity | storage survival, conceptual survival, continuity, lineage, relationship survival |
| working state as resumable position | working state, active frontier, constraints, validation state, selection rationale, next safe moves |
| lineage as artifact succession | documentation lineage, inquiry lineage, observation lineage risk |
| unresolved item as active edge | preserved frontier, selected attention, current pull, current work position |

The strongest compression-removal examples are persistence -> continuity and
documentation lineage -> inquiry lineage. Both show a familiar repository word
becoming too compressed under pressure and then splitting into more careful
boundary language.

### Preservation-gap findings

The strongest preservation gap is not that findings are missing. The repository
is already strong at preserving conclusions, audits, observations, and
frontiers. The gap is that discovery-path reconstruction often requires
cross-reading several documents and inferring the challenge sequence.

Potentially lost when only final findings survive:

- which accepted framing was challenged;
- what pressure made the old framing insufficient;
- what assumption was exposed;
- which alternatives were considered and rejected implicitly;
- why a distinction appeared when it did;
- how a future participant should avoid re-compressing the distinction.

This matters because many recent concepts are understandable only as responses
to pressure. If the pressure is lost, the final distinction can look arbitrary,
redundant, or ontology proliferation rather than boundary preservation.

### Existing-type sufficiency finding

Existing document types can contain discovery-path material. No evidence proves
that a mandatory new document category is required. A frontier can preserve the
unresolved edge, an audit can preserve evaluated pressure, an observation can
preserve visible evidence, and a reconciliation can preserve settled boundary
reasoning.

The evidence does show, however, that none of those types has discovery-path
preservation as its primary center. If discovery-path preservation remains
important, the repository may need at least a navigable way to find such
retrospective material. This observation does not define that mechanism.

## Tensions Preserved

### Finding vs discovery path

A finding answers what the repository now understands. A discovery path answers
how the repository stopped understanding something in the old way.

### Observation vs critique

Observation records evidence. Critique pressures the interpretation or framing
of evidence and may expose that an observation is overcompressed.

### Audit vs critique

Audit evaluates a scoped state. Critique can evaluate the adequacy of a framing
before a stable audit target exists.

### Inquiry lineage vs discovery lineage

Inquiry lineage follows question movement. Discovery lineage follows
understanding transition. They overlap when a question changes understanding,
but they are not automatically identical.

### Artifact preservation vs understanding preservation

A document can be preserved while the reader loses why it mattered. Artifact
preservation is necessary but insufficient for preserving understanding
transition.

### Conclusion preservation vs transition preservation

Conclusion preservation lets a future participant cite the stable answer.
Transition preservation lets a future participant avoid undoing the distinction
by returning to the compressed premise that preceded it.

## Unresolved Observations

- Whether critique should remain a mode inside existing document types or ever
  become a routed documentation category is unresolved.
- Whether discovery retrospective is a useful name is unresolved.
- Whether discovery-path preservation belongs near architectural findings,
  documentation lineage, inquiry lineage, or frontier routing is unresolved.
- Whether all examples reviewed here are recent-work artifacts rather than a
  general repository pattern is unresolved.
- Whether artifact lineage, inquiry lineage, observation lineage, and discovery
  lineage require further distinction is unresolved.
- Whether preserving discovery paths would improve future implementation work or
  only add documentation overhead is unresolved.

## Conclusion

The repository currently preserves `what became understood` better than `how
understanding changed`.

That conclusion is not absolute: many documents preserve partial paths, and
frontier documents often explain their motivating pressure. But across the
reviewed examples, final findings are easier to locate than challenge sequences,
assumption exposure, compression removal, and understanding transitions.

A recurring preservation gap therefore appears to exist. The gap is not missing
conclusions; it is missing or weakly routed discovery-path memory.

The recurring pattern is important enough to preserve as an observed concern,
especially because recent repository strength came from critique-driven
boundary clarification. This document does not recommend runtime, schema,
workflow, governance, or ontology changes. It only records that discovery-path
preservation is a meaningful repository concern under current evidence.
