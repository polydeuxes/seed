## Current distinguishability before pass 077

The implementation could distinguish incompatible boundary values, corrupted payloads, stale event ids, and absence. It could not distinguish a newly saved bounded snapshot from a legacy row upgraded by ALTER defaults because both rehydrated as equal `ProjectionSnapshotBoundary()` values. It could distinguish in-memory defaults from SQLite rows by backend only, not by testimony origin. A manually populated row with expected strings was indistinguishable from producer-recorded testimony if values matched. A rejected row rebuilt from ledger evidence became distinguishable only by replacement timing, not by persisted origin. Lawful stop: do not infer producer occurrence from matching strings.
