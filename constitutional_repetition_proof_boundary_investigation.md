## Bounded question

Across recurring repository evidence, what constitutional boundary, if any, separates:

```text
repetition / recurrence / repeated assertion
```

from:

```text
proof / sufficient support / warranted conclusion
```

and under what conditions, if any, can repetition lawfully contribute to evidence without becoming proof?

This is exactly one bounded constitutional investigation focused only on `Repetition != Proof`. ## Repetition / recurrence analysis

Repository evidence already uses recurrence in at least three bounded ways.

First, implemented documentation recurrence is mechanical structure evidence. The app's documentation-structure recurrence counts repeated section labels, front matter keys, heading depths, code fence languages, and link target classes, while explicitly refusing prose interpretation, claim extraction, authority inference, shape inference, ontology promotion, event-ledger writes, and repository mutation.

Second, investigation prose uses `recurring repository evidence` as a constitutional method: recurrence can reveal a pattern worth investigating, especially when the recurrence appears across independent artifacts, implementation surfaces, tests, diagnostics, or bounded prior investigations.

Third, recurrence can be an architectural signal only when paired with contradiction checks, authority classification, source classification, and implementation or documentation evidence. Repository evidence explicitly warns that recurrence alone would over-merge projection/state/cache/authority, ownership/authority, implementation support/evidential support, and capability/command/execution/action.

Therefore recurrence means observed repetition of a structure, label, relationship, concern, or evidence pattern. It does not by itself mean truth, proof, authority, ontology, implementation behavior, source independence, or sufficient support.

## Repeated assertion analysis

Repetition is merely repeated assertion when the repository only shows the same or similar claim appearing again without evidence that the repetitions add independent support.

Repeated assertion includes:

- copied wording across documents without preserved provenance;
- repeated section labels without claim extraction or authority inference;
- recurring presentation vocabulary without knowledge-reachability preservation;
- repeated diagnostic visibility without evidence that the visible output supports the asserted semantic claim;
- repeated prior conclusions that cite each other without exposing source independence or original support;
- high count or high volume without an authority path, local admissibility rule, or claim-strength match.

In these cases, repetition may show that wording recurs, a documentation pattern exists, or a concern is salient enough to recur. It does not show the claim is true, independently supported, sufficiently warranted, or proven.

## Independent corroboration analysis

Independent corroboration differs from copied repetition by preserving source independence and support paths.

Repository evidence supports independent corroboration only when there is visible reason to treat supports as distinct rather than copied. Relevant indicators include:

- different source classes, such as implementation evidence, tests, diagnostics, generated architecture evidence, app behavior, bounded prior investigations, and provider-local observations;
- preserved provenance or support metadata sufficient to inspect origin;
- non-circular support paths rather than one artifact merely repeating another;
- local authority for the claim being made;
- consistency across independent already-observed streams, while preserving the boundary that agreement evidence is not truth promotion;
- counterexample and contradiction review, so recurrence does not over-merge distinct concepts;
- claim-strength matching, so independent evidence for a pattern is not promoted into proof of implementation behavior or universal truth.

Copied repetition lacks those conditions. If multiple artifacts repeat a claim but share an origin, omit provenance, cite each other circularly, or remain documentation-only while the claim concerns implementation behavior, the lawful conclusion is Unknown or weak support, not corroboration.

## Evidence-strength analysis

Recurrence can lawfully contribute to evidence under bounded conditions.

It can contribute as evidence of a pattern when:

1. the repeated thing is exactly identified, such as a section label, relation shape, phrase, diagnostic field, implementation trait, or architectural concern;
2. the surface boundary matches the claim, such as structural recurrence supporting a structural claim but not a semantic truth claim;
3. provenance or source paths make it inspectable where the claim requires provenance;
4. source independence is established or the claim is explicitly limited to recurrence within a dependent corpus;
5. contradictions and over-merge risks are checked;
6. the authority boundary permits the use;
7. confidence and claim strength remain calibrated to the evidence;
8. Unknowns are preserved where provenance, independence, currentness, or support is missing.

Under those conditions, repetition can be evidence that something recurs, that a pattern is candidate-stable, that a concern should be investigated, or that a family resemblance exists. It still does not become proof by count alone.

## Proof / warranted-conclusion analysis

Repository evidence does not support a general proof engine or a universal repetition-to-proof rule.

The repository more strongly supports `warranted conclusion` than abstract proof. A conclusion is warranted only when evidence, provenance/support, role, authority limit, negative authority, Unknown preservation, and confidence limit support lawful reliance for a bounded consumer. Claim support is a relationship, not proof or truth. Evidence is not automatic proof, truth, verification, promotion authority, ownership, mutation authority, or complete reality.

Sufficient support depends on claim form:

- A structural recurrence claim can be sufficiently supported by exact structural rows and counts from the documentation structure surface.
- A source-location claim can require drilldown or source navigation rather than mere recurrence count.
- An implementation-behavior claim requires implementation evidence, tests, diagnostics, or generated implementation-backed evidence, not documentation recurrence alone.
- A constitutional characterization can rely on recurring repository evidence and prior bounded investigations, but only with preserved negative authority, Unknowns, and confidence limits.
- A current-state or fact claim requires fact support, current facts, evidence, confidence, and stale/contradiction boundaries where applicable.

Thus evidentiary support becomes a warranted conclusion only when the support is adequate for the specific claim strength and authorized consumer. It does not become proof merely because it is repeated.

## Authority-boundary analysis

Authority is the boundary that determines what a source can lawfully support.

The documentation recurrence surface has authority to report mechanical structural recurrence. It lacks authority to interpret prose, extract claims, infer authority, infer shapes, promote ontology, write events, or mutate the repository.

Diagnostic inventory and diagnostic shape audit have authority to expose declared operational surfaces and compare static implementation shape declarations. They do not adjudicate truth. They can prove a surface is registered or statically checked only within their implementation-backed visibility contract, not prove the semantic truth of the content displayed by every surface.

Prior investigations can support constitutional characterization when they preserve their evidence, counterexamples, boundaries, Unknowns, and confidence. They do not become world research, legal analysis, implementation proof, or universal ontology merely by being repeated.

The authority boundary therefore separates repetition from proof: repetition has no lawful force beyond the source authority and claim form it can support.

## Confidence / claim-strength analysis

Confidence and claim strength prevent overclaiming from recurrence.

A high recurrence count may justify confidence that a label or structure appears often. It may not justify confidence that repeated claims are independent, current, implementation-backed, authoritative, or true unless the evidence includes provenance, independence, currentness, implementation support, and authority.

Claim-strength matching is the governing discipline:

- repeated wording supports at most a wording/visibility recurrence claim;
- repeated section labels support at most structural recurrence unless paired with content review;
- repeated documentation claims can support documentation-language or architectural-intent claims, but not implementation conformance without implementation evidence;
- repeated implementation-backed tests or diagnostics can support stronger implementation claims, but only within their tested scope;
- repeated independent observations can support candidate agreement, but agreement evidence still refuses truth promotion unless another authority supports that promotion.

## Unknown preservation

The following Unknowns must remain when repeated claims lack provenance or independence evidence:

- whether the repetitions are copied, circular, independently authored, independently observed, or derived from one source;
- whether repeated claims are current or stale;
- whether repeated claims are contradicted elsewhere;
- whether the repeated wording has the same meaning in every location;
- whether the repeated sources have authority for the claim being made;
- whether documentation recurrence corresponds to implementation behavior;
- whether a repeated presentation label is preserved repository knowledge;
- whether a repeated assertion has fact support;
- whether recurrence is evidence of a true pattern or only evidence of repeated expression;
- whether any stronger conclusion is warranted.

Unknown preservation is not failure. It is the lawful result when support, provenance, independence, authority, or claim-strength evidence is missing.

## Lawful stop

The lawful stop condition is:

```text
Stop when the available recurrence evidence supports only repetition, visibility,
structural recurrence, or candidate pattern, and no independent provenance,
source-independence evidence, local admissibility rule, authority boundary, and
claim-strength-matched support authorize a stronger conclusion.
```

At that point, the investigation may say that the claim recurs, the wording is visible, the section label repeats, the pattern is candidate-supported, or the boundary deserves bounded future inquiry. It must not say that repetition proves the claim, that repeated visibility is truth, that documentation recurrence is implementation behavior, or that volume supplies authority.

## Required answers

### 1. What does repository evidence already mean by recurrence or repetition?

Repository evidence means observed recurrence of structures, labels, wording, relationship shapes, concerns, or evidence patterns. Implemented app recurrence is narrower: mechanical document-structure recurrence with no prose interpretation, claim extraction, authority inference, shape inference, ontology promotion, event-ledger writes, or repository mutation.

### 2. When is recurrence valid evidence of a pattern?

Recurrence is valid evidence of a pattern when the repeated item is exactly identified, the surface has authority for the claimed pattern, source/provenance boundaries are visible, contradictions and over-merge risks are checked, and the conclusion remains matched to the evidence strength. It is strongest when recurrence appears across independent implementation, test, diagnostic, app, and bounded-investigation evidence rather than copied prose alone.

### 3. When is repetition merely repeated assertion?

Repetition is merely repeated assertion when the same claim appears multiple times without provenance, independence, authority, contradiction review, or claim-strength-matched support. Repeated wording, repeated visibility, repeated section labels, or repeated documentation statements are not sufficient support for truth, implementation behavior, or warranted conclusion by themselves.

### 4. What distinguishes independent corroboration from copied repetition?

Independent corroboration preserves non-circular source paths, distinguishable source classes or origins, inspectable provenance, authority for the claim, and consistency across sources that could have disagreed. Copied repetition lacks evidence that the repetitions are independent and must remain Unknown or weak recurrence evidence.

### 5. What distinguishes repeated visibility from evidentiary support?

Repeated visibility means a surface, phrase, label, or output appears more than once. Evidentiary support means the visible material is connected to a support relationship, provenance, source authority, admissibility boundary, and claim form. Visibility can expose evidence, but visibility itself is not support unless the repository surface has authority to support that claim.

### 6. What distinguishes evidentiary support from proof or warranted conclusion?

Evidentiary support is a support relationship. It is not proof or truth by itself. A warranted conclusion requires enough support for the specific claim strength plus preserved evidence, provenance/support, role, authority boundary, negative authority, Unknowns, and confidence limits. The repository does not support a general proof engine.

### 7. What role do provenance, source independence, authority, confidence, and claim-strength matching play?

Provenance makes the origin inspectable. Source independence separates corroboration from copying. Authority limits what the source can lawfully support. Confidence calibrates how strongly the repository may rely on the support. Claim-strength matching prevents a weak recurrence claim from being promoted into truth, implementation behavior, ontology, or universal proof.

### 8. What Unknowns must remain when repeated claims lack provenance or independence evidence?

Unknowns remain about copying, circularity, independence, currentness, contradiction, meaning consistency, source authority, implementation correspondence, fact support, and whether any stronger conclusion is warranted.

### 9. What lawful stop condition prevents repetition from becoming proof?

The lawful stop is reached when recurrence evidence lacks independent provenance, source-independence evidence, local admissibility, authority, and claim-strength-matched support for a stronger conclusion. The repository may report recurrence or candidate pattern and must refuse proof, truth, authority, ontology, or implementation claims.

### 10. Does this investigation strengthen the previous partial boundary?

Yes. The previous review found `Repetition != Proof` partially preserved because the repository had evidence-strength discipline but less direct treatment of `repetition` as a named boundary. This investigation strengthens that partial by separating structural recurrence, repeated assertion, independent corroboration, evidentiary support, warranted conclusion, and proof-like overclaiming under repository authority.

### 11. Does repository evidence support any implementation work yet?

No. No concrete repository failure was found. The app-visible surfaces already preserve relevant boundaries for documentation recurrence, diagnostic inventory, diagnostic shape audit, knowledge reachability, and fact support. The current need is constitutional clarification, not implementation.

- Repeated visibility differs from evidentiary support because support requires authority, provenance, admissibility, and claim-form matching.
