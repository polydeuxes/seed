# Constitutional Translation / Rendering Source-Equivalence Boundary Investigation

## Bounded question

Across recurring repository evidence, when Seed receives or produces a transformed representation of source material, what prevents that representation from being treated as equivalent to the source without separately preserved original source material, transformation provenance, language/version identity, method boundary, claim form, support path, authority boundary, confidence limits, Unknowns, and lawful stop?

This is exactly one bounded constitutional investigation of the breadcrumb:

```text
Translation / Rendering != Source Equivalence
```

It does not implement anything, create a schema, create a diagnostic, create a source-equivalence engine, score translations, or fetch public-world evidence.

It also reports diagnostic registry surfaces as consistently non-mutating unless specifically record-scoped.

## Repository meanings already present

### 1. Translation, rendering, or transformed representation

Repository evidence already uses **translation** in at least two bounded ways.

First, provider-language translation is implementation-backed work that turns provider-local language into Seed observation-facing language. Its recurring responsibilities include bounded vocabulary selection, validation/rejection of provider grammar, decoding into provider-local or normalized structures, preservation of provider identity/evidence in subjects, dimensions, values, and metadata, and Seed predicate assignment where observation construction occurs.

Second, translation-oriented views may move material between reasoning spaces or operator-facing forms. The context-preservation evidence warns that the important issue is not translation alone but whether context that would otherwise be lost remains visible.

Repository evidence also uses **rendering** for human or CLI presentation paths. Diagnostic inventory and diagnostic-shape surfaces repeatedly distinguish rendered output from recording, event-ledger writes, cluster mutation, schema authority, and generalized inference.

Repository evidence does **not** currently preserve `translation` as a reachable app-visible implementation concept in knowledge reachability for the subject `translation`; app evidence classifies it as unknown/not preserved. Therefore this investigation treats translation/rendering as bounded investigation vocabulary, not repository ontology.

### 2. Source equivalence

Repository evidence already has a broader **justified equivalence** discipline. Equivalence requires positive evidence stronger than mere boundary creation, shared state, similar wording, recurrence, or convenience. Conservative separation is the default when equivalence evidence is weak. Identity equivalence is stronger than value equivalence, state equivalence, query equivalence, or visibility equivalence.

The repository does not appear to contain a dedicated source-equivalence model for translations, OCR, transcripts, summaries, or normalized text. The supported recovery is therefore smaller: transformed representations can be access artifacts or support-path artifacts, but they are not source equivalence without independent positive equivalence evidence and preserved boundaries.

## Translation / rendering analysis

### 3. What distinguishes translation from original source?

A translation is a transformed representation across language, provider grammar, reasoning space, or presentation boundary. The original source is the source material being represented. Repository evidence distinguishes them by requiring preservation of provider/source identity, transformation boundary, claim form, support path, authority boundary, Unknowns, and confidence limits. A translation may provide access to source material, but it does not itself prove full meaning preservation, legal identity, historical identity, temporal provenance, source authority, or truth.

### 4. What distinguishes rendering from source identity?

Rendering is presentation. Source identity is a claim about the source artifact/referent. Diagnostic rendering paths show that rendered lines can expose existing fields without owning recording, event-ledger authority, cluster mutation, schema authority, or generalized semantics. Therefore a rendered form may show a source-derived value but does not become the source, the identity of the source, or authority to collapse source boundaries.

### 5. What distinguishes paraphrase or summary from source text?

A paraphrase or summary changes claim form, selection, wording, scope, and omission risk. Repository evidence on model output, narrative/history, warrant, and context-preserving translation rejects treating generated or composed text as source truth. A summary may identify a possible support path or explain bounded content, but it must preserve that it is selected/composed/derived rather than the source text itself.

### 6. What distinguishes quote or excerpt from full source context?

A quote or excerpt preserves some wording but not full context, structure, surrounding qualifications, omitted portions, temporal/source edition context, or authority boundary. The repository's source-independence and public-evidence readiness disciplines require support-path separation and packet readiness rather than isolated passage promotion. A quote/excerpt can support a bounded claim about the excerpted wording only if provenance and context limits are preserved.

### 7. What distinguishes OCR rendering from original document/image?

OCR rendering is machine-produced text derived from an image/document. It may lose layout, handwriting, marginalia, seals, pagination, columns, tables, uncertain glyphs, erasures, damage, and visual context. Repository evidence on model output and artifact intake supports treating parser/OCR-like outputs as transformed artifacts with method and confidence boundaries, not primary document identity or original-source truth.

### 8. What distinguishes transcript from original audio/video/event?

A transcript is a selected textual representation of audio, video, or an event. It may lose tone, timing, speaker uncertainty, nonverbal context, editing history, recording chain, and event completeness. It is not the original audio/video/event and cannot establish event truth, speaker identity, or legal/historical authority without preserved original material, transformation provenance, method boundary, and support path.

### 9. What distinguishes normalized or canonicalized text from source structure?

Normalization/canonicalization may make values comparable, but it can remove source order, formatting, spelling variants, punctuation, layout, encoding, section boundaries, ambiguity, and local semantics. Repository normalization evidence treats canonical forms as bounded projected or provider-facing forms, not total identity. Normalized text can support search, grouping, or compatibility, but not source-structure preservation unless the original structure remains available and the normalization method is bounded.

### 10. What distinguishes language equivalence from meaning preservation?

Language equivalence is a claim that wording in different languages corresponds under some method or context. Meaning preservation is stronger: it claims semantic, pragmatic, legal, historical, cultural, temporal, and claim-form content has survived transformation. Repository evidence does not support automatic promotion from cross-language correspondence to meaning preservation. Confidence in a translation remains confidence in a transformation, not proof that all meanings relevant to later reliance are preserved.

### 11. What distinguishes parallel wording from structural equivalence?

Parallel wording is surface similarity or aligned text. Structural equivalence requires preserved relationships, order, hierarchy, definitions, exceptions, cross-references, roles, versions, and context. The justified-equivalence evidence rejects treating shared state or matching values as identity equivalence. By analogy, parallel wording does not prove structural equivalence without positive evidence of preserved structure and source identity.

### 12. What distinguishes translation confidence from evidence strength?

Translation confidence is confidence that a transformation method produced a plausible representation. Evidence strength is confidence that a claim is supported by source material under preserved provenance, independence, authority, claim form, temporal boundary, and Unknown discipline. A high-confidence translation can still be weak evidence for a legal/historical/source claim if original source, version identity, method boundary, context, or authority is missing.

### 13. What distinguishes transformed representation from source authority?

Source authority attaches, if at all, to an authoritative source under a specific authority boundary. A transformed representation may be generated by a provider, OCR engine, translator, parser, summarizer, citation renderer, or Seed report. The model-output and authority-volume boundaries prevent generated or repeated outputs from becoming authority by volume, polish, confidence, or presentation. Transformation provenance must remain distinct from source authority.

### 14. What distinguishes transformation artifact from primary evidence?

A transformation artifact is evidence that a transformation occurred and may be secondary support for access, search, orientation, or limited content claims. Primary evidence is the original source material or source-authorized record relevant to the claim. Repository evidence supports preserving transformation artifacts as support-path components, not silently substituting them for primary evidence.

## Lawful uses and readiness

### 15. What can translation or rendering lawfully support?

Translation/rendering can lawfully support bounded access and orientation claims, such as:

- “this is the translated/rendered/paraphrased/OCR/transcribed/normalized form we have”;
- “this representation was produced by this method, from this identified source artifact, at this time, by this actor/tool/version”;
- “this representation suggests a possible claim to check against the original source”;
- “this excerpt/quote supports a narrow claim about the excerpted wording, with context limits preserved”;
- “this normalized value supports lookup, matching, or grouping under a bounded method”;
- “this translation can be used for preliminary review when lawful reliance stops before source-equivalence, truth, authority, or implementation pressure.”

Translation/rendering cannot lawfully support stronger claims unless original source material, provenance, method, identity, claim form, support path, authority boundary, confidence limits, Unknowns, and stop conditions are preserved strongly enough for that claim.

### 16. What must be preserved before a transformed representation may contribute to evidence, warrant, or lawful reliance?

Before contribution to evidence, warrant, or lawful reliance, the repository-backed minimum preservation set is:

1. original source material or a bounded explanation of its absence;
2. source identity, source type, source version/edition/language, and temporal provenance;
3. transformation provenance: actor/tool/model/parser/OCR/transcriber/translator, method, date/time, inputs, settings if relevant, and transformation chain;
4. transformation boundary: translation, rendering, paraphrase, summary, excerpt, quote, OCR, transcript, format conversion, normalization, or canonicalization;
5. claim form: what the transformed representation is being used to claim and what it is not claiming;
6. support path from transformed representation back to source material and any intermediate artifacts;
7. source-independence boundary where corroboration is claimed;
8. authority boundary separating source authority from transformer/provider/renderer authority;
9. confidence limits distinguishing transformation confidence from evidence strength;
10. Unknowns, including missing original, missing context, missing method, missing version, missing language competence, missing layout, missing speaker identity, missing edition, missing chain of custody, and missing legal authority;
11. lawful stop condition if stronger reliance is unsupported.

### 17. Lawful stop condition

The lawful stop condition is:

> If original source material, source/version/language identity, transformation provenance, method boundary, claim form, support path, authority boundary, confidence limits, Unknowns, and lawful reliance boundary are not preserved enough for the proposed claim, the transformed representation must remain only a bounded access/orientation artifact. It must not become source equivalence, source identity, meaning preservation, structure preservation, truth, source authority, warrant, lawful reliance, lawful acceptance, or implementation pressure.

## Stronger-claim readiness analysis

A transformed representation is not ready for stronger claims merely because it is readable, fluent, repeated, cited, confidence-scored, app-rendered, or operationally convenient. Stronger-claim readiness requires evidence appropriate to the stronger claim:

- source-equivalence claims require positive equivalence evidence, not similarity;
- meaning-preservation claims require method and context evidence, not language confidence alone;
- structure-preservation claims require preserved structure or reliable structural mapping;
- source-authority claims require source authority, not transformer authority;
- primary-evidence claims require original or source-authorized material;
- warrant/lawful-reliance claims require bounded evidence, provenance, authority boundary, Unknowns, and confidence calibration.

## Provenance / original-source / transformation-method / support-path analysis

The recurring repository discipline is provenance before promotion. Transformed representations may enter a support path, but the path must remain typed and inspectable:

```text
original source material
  -> transformation method and actor/tool
  -> transformed representation
  -> bounded claim form
  -> support path and authority boundary
  -> confidence/Unknowns
  -> lawful use or lawful stop
```

If the first node is missing, the absence must remain Unknown or unavailable, not silently replaced by the representation. If intermediate transformation steps are missing, the support path is incomplete. If the claim form exceeds the support path, lawful stop applies.

## Authority-boundary analysis

A translator, OCR engine, transcript provider, parser, summarizer, quote selector, citation renderer, model, or Seed report may have authority only over its own transformation artifact. It does not inherit source authority unless repository evidence separately establishes that boundary. Volume, repetition, fluent narrative, official-looking formatting, or operational convenience do not create authority.

## Confidence / claim-strength analysis

Confidence must be typed:

- transformation confidence: plausibility/quality of rendering;
- source-identification confidence: whether the correct source/version/language was used;
- support confidence: whether the representation supports the bounded claim;
- authority confidence: whether the source has the relevant authority;
- reliance confidence: whether lawful reliance is warranted now.

The investigation supports no automatic conversion between those confidence types.

## Unknown preservation

The following Unknowns must remain visible rather than collapsed:

- unknown original source availability;
- unknown source edition/version/language;
- unknown transformation actor/tool/method;
- unknown chain of transformations;
- unknown omitted context;
- unknown structural loss;
- unknown OCR/transcript errors;
- unknown legal/historical authority;
- unknown source independence;
- unknown temporal provenance;
- unknown claim-form fit;
- unknown whether stronger reliance is lawful.

## Supported conclusions

1. Repository evidence supports treating translation/rendering/transformed representations as bounded artifacts whose authority terminates at their preserved transformation boundary.
2. Repository evidence does not support automatic source equivalence for translations, renderings, paraphrases, summaries, excerpts, quotes, OCR text, transcripts, normalized text, or canonicalized text.
3. Source equivalence requires positive equivalence evidence stronger than similarity, fluency, recurrence, confidence, generated citation, or app rendering.
4. Meaning preservation, structure preservation, source identity, source authority, primary evidence, truth, warrant, lawful reliance, and lawful acceptance are stronger claims requiring separately preserved support.
5. Translation confidence is not evidence strength.
6. Rendering is not source identity.
7. Summary/paraphrase is not source text.
8. Quote/excerpt is not full context.
9. OCR/transcript/normalized/canonicalized text is not original source truth.
10. Parallel wording is not structural equivalence.
11. Missing original source, missing method, missing language/version identity, missing context, missing authority, and missing support path remain Unknowns, not negative proof and not permission to promote.
12. The lawful output when preservation is insufficient is stop, bounded orientation, or Unknown preservation, not implementation pressure.
