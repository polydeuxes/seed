## Executive answer

Lawful architectural questions in this repository repeatedly become recoverable through neighborhoods of neighboring distinctions, not through isolated distinctions alone.

The recurrence is bounded. The repository does not show one universal neighborhood topology, one constitutional neighborhood engine, or one fixed sequence that every investigation follows. It does show a repeated discipline: an interesting observation becomes a lawful architectural question only after adjacent distinctions make the observation's evidence, boundary, authority, and non-authority visible enough for the repository to answer honestly.

The smallest recurring neighborhood currently supported is:

```text
noticed observation / pressure / candidate
  != implementation-backed evidence
  != authority-bearing admission
  != boundary-local question / lawful stop
```

In shorter form:

```text
Observation
  -> Evidence
  -> Authority / non-authority
  -> Boundary-local inquiry
```

That is not a runtime pipeline. It is a methodological neighborhood: the repository repeatedly needs more than one distinction before a question can be asked lawfully.

## Neighborhood review

### 1. Bounded inquiry recovery

**Initially noticed distinction:** an operator question or possible inquiry is not automatically a lawful answer surface.

**Neighboring distinctions that became necessary:**

- Exact registered question family versus free-text question.
- Eligible surface versus diagnostic-only or not-dispatchable surface.
- Required arguments versus missing parameters.
- Surface-local answer responsibility versus broader semantic routing.
- Bounded answer versus next responsibility.

**Insufficient alone:** `interesting question != lawful investigation` is true but too small. By itself it does not identify which implemented surface may answer, whether parameters exist, whether the family is diagnostic-only, or where dispatch must stop.

**Neighbor that revealed the actual boundary:** eligibility and dispatch boundaries revealed the actual boundary. The lawful question exists only where an implemented question family maps to a bounded surface with required arguments and no diagnostic-only stop.

## Topology analysis

Recurring neighborhoods exist, but they are family-local and not identical.

### Topology A: Observation / evidence / authority / admission

Recovered in observation, evidence, fact promotion, capability candidate, candidate attachment, and artifact delta work:

```text
Observed or preserved material
  -> provenance / evidence / support
  -> authority boundary
  -> local admission or non-admission
```

This topology answers whether material may become fact, support, readiness, candidate agreement, grammar observation, or remain local.

### Topology B: Pressure / boundary / owner / preservation

Recovered in pressure, implementation readiness, read-model ownership, projection lineage, diagnostics, and completion-audit work:

```text
Pressure or repeated reconstruction
  -> hidden boundary or compressed owner
  -> bounded owner / handoff
  -> compatibility-preserving proof
  -> residual or frontier stop
```

This topology answers whether an implementation-backed recovery is possible or whether pressure remains residual/frontier.

### Topology C: Question / eligibility / surface / answer boundary

Recovered in bounded ask, question-family inventory, inquiry orientation, answer composition, and responsibility-to-inquiry audits:

```text
Possible question
  -> exact family / preserved note
  -> eligible implemented surface
  -> bounded answer shape
  -> explicit non-authority
```

This topology answers whether a question can be asked through an implemented answering surface and what the surface must refuse.

### Topology D: Representation / structural evidence / observation / relationship

Recovered in external grammar, provider translation, artifact intake, grammar observation, relationship observation, and architectural naming work:

```text
External or provider representation
  -> Seed-owned structural evidence
  -> canonical observation or local record
  -> relationship / recurrence / candidate visibility
  -> truth and ownership refusal unless admitted
```

This topology answers how expressive external material is reduced before the repository can treat it as observation, relation evidence, candidate recurrence, or fact.

### Topology E: Methodology / recurrence / earned section / lawful stop

Recovered in competency interrogation and methodology reconciliation:

```text
Repeated investigation move
  -> evidence that the move changes answers
  -> earned or provisional methodology section
  -> reliance boundary
  -> remaining unknowns
```

This topology answers whether an investigation practice itself has earned use in future investigations.

## Distinction analysis

### Single distinction != recoverable neighborhood

Supported. Single distinctions repeatedly prevent one error but do not recover enough context to ask lawfully. Examples:

- `pressure != authority` needs compression, owner, boundary, compatibility, and residual/frontier distinctions.
- `support != admission` needs candidate, verification, readiness, policy, execution, and mutation distinctions.
- `provider grammar != observation grammar` needs structural evidence, canonical observation, fact promotion, authority reduction, and counterexample distinctions.
- `interesting question != lawful investigation` needs registration, eligibility, required arguments, dispatch, and answer-boundary distinctions.

### Interesting question != lawful investigation

Strongly supported. A question becomes lawful only when a repository surface, evidence path, boundary, and stop condition are visible. The repository rejects free-text semantic routing, unknown family inference, diagnostic-only answering as dispatch, and unsupported promotion from operator language to repository truth.

### Boundary != neighborhood

Supported. A boundary is one edge; a neighborhood is the set of adjacent edges that prevent the boundary from being misused. For example, the observation/fact boundary only becomes honest when evidence, fact promotion, support, current belief, explanation, and mutation boundaries are also visible.

### Recovered participant != recovered topology

Supported. Recovering one participant, artifact, candidate, operation, or record does not recover the full topology it belongs to. Capability candidates do not recover policy or execution. Inquiry notes do not recover semantic intent. Grammar observations do not recover grammar authority. Relationship records do not recover responsibility ownership.

### Recurrent neighborhood != universal engine

Supported. Recurrence is real, but repository authority blocks promotion into a universal neighborhood engine. The neighborhoods are methodological and family-local. They guide characterization; they do not authorize runtime discovery, planning, or automatic completion.

## Negative analysis

Recurring failures appear when an investigation asks only one question.

### Universal hypotheses

Failure shape:

```text
One recurring phrase or transition appears
  -> assume universal architecture
```

Counterevidence recurs in transition families, provider grammar, bounded question discipline, and implementation readiness. The repository supports several local families, not one universal movement or engine.

### Isolated participants

Failure shape:

```text
Recovered participant
  -> assume recovered responsibility topology
```

Counterevidence recurs in capability candidate, inquiry note, observation agreement, grammar observation, and selection-path work. A participant may be visible while admission, ownership, policy, execution, mutation, and truth remain explicitly denied.

### Vocabulary without neighbors

Failure shape:

```text
Recurring vocabulary
  -> promote vocabulary into knowledge
```

Counterevidence recurs in naming, presentation vocabulary, inquiry orientation, and structural grammar work. Vocabulary may orient inquiry, but repository knowledge requires implementation-backed evidence and authority boundaries.

### Artifact without adjacent boundaries

Failure shape:

```text
Recovered artifact
  -> treat artifact as fact, support, verification, admission, or authority
```

Counterevidence recurs in artifact delta, observation/evidence/fact, capability readiness, and diagnostic boundary work. Artifacts require local admission rules and mutation boundaries.

### Pressure without owner locality

Failure shape:

```text
Operator pain or architectural pressure
  -> implement broad fix
```

Counterevidence recurs in pressure methodology and implementation readiness. Pressure or pain can orient inquiry, but implementation requires a compressed boundary, owner/corridor, preservation surface, and tests.

### Question without answer surface

Failure shape:

```text
Question is meaningful
  -> assume repository can answer it
```

Counterevidence recurs in bounded ask, question-family inventory, inquiry orientation, and answer composition. Lawfulness requires an implemented answering surface or an explicit stop.

## Locality

Neighborhood questioning currently appears **methodology-local and implementation-backed**, with operator-local input often acting as orientation.

- **Methodology-local:** the recurrence belongs to how investigations are honestly reduced. It is strongest in reports that recover evidence, authority, negative authority, boundary, neighbor contrast, unknowns, confidence, and termination.
- **Implementation-backed:** the neighborhoods are justified by implementation-backed reports, code surfaces, tests, diagnostics, event shapes, read-only boundaries, and negative authority statements. The characterization is not based on elegance alone.
- **Operator-local:** operator testimony, pain, and discussion can make an observation or pressure visible, but they do not finish the neighborhood. They orient inquiry until repository evidence can answer.
- **Not currently runtime-local:** no reviewed evidence supports automatic runtime neighborhood interrogation, planning, completion, or topology discovery.
- **Still unknown constitutionally:** recurrence is strong enough for characterization, but not strong enough to declare neighborhood questioning a constitutional competency.

## Smallest truthful answer

The smallest recurring neighborhood of distinctions that repeatedly transforms an interesting observation into a lawful architectural question is:

```text
An observation, pressure, candidate, vocabulary, or artifact becomes lawfully question-bearing only when neighboring distinctions expose:

1. what evidence backs it,
2. what authority it does not have,
3. what local boundary or owner can answer it,
4. what admission, mutation, implementation, or runtime claims must stop.
```

Compressed:

```text
Observation
  + Evidence
  + Non-authority
  + Local answer boundary
  + Lawful stop
```

This neighborhood recurs across bounded inquiry, competency interrogation, implementation readiness, pressure recovery, transition families, artifact delta work, external grammar recovery, naming recurrence, inquiry orientation, question responsibility, and capability admission work.

It does not recur as one elegant topology. It recurs as a family-local discipline of refusing to ask a broad architectural question until adjacent evidence, authority, admission, and boundary distinctions have become visible together.
