# Bounded Inquiry Frontier Slice 001

This slice adds one read-only `BoundedInquiryFrontier` assembler.

It consumes one exact selected inquiry need and matching `InquiryFrontierBoundaryTestimony`, preserves selected-need identity, native inquiry-item lineage, uncertainty component, repository/world subject, selected goal, horizon, and every supplied testimony clause.

A frontier is established only when already-supplied testimony contains coherent operative clauses for all four required families:

1. included inquiry scope;
2. eligible evidence territory;
3. sufficient-resolution condition;
4. lawful stopping condition.

Any material binding conflict prevents establishment. Missing families remain explicit. Unsupported, unknown, conflicting, mixed, adjacent-family, stale, unavailable, and out-of-scope clauses remain preserved and non-operative unless they independently satisfy the coherent operative rule.

The assembler is read-only. It does not invent scope, invent evidence admission, formulate a constitutional question, open inquiry, select sources or observations, authorize access, execute, record, write the event ledger, mutate cluster state, or claim that inquiry results are known.
