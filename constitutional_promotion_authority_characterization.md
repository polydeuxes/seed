# Constitutional Promotion Authority Characterization

## Status

This is an investigation and characterization, not a governance system, runtime design, authorization mechanism, implementation recommendation, permission model, or universal promotion manager.

The question is architectural authority:

```text
Who is constitutionally permitted
to promote
an artifact
from one constitutional state
to another?
```

Repository authority wins. If this characterization conflicts with implementation evidence, implementation evidence wins.

## Executive answer

Promotion authority is a recurring constitutional artifact, but not as one universal office, actor, manager, or owner.

The repository repeatedly supports this smaller pattern:

```text
Promotion is permitted only when
family-local repository evidence
satisfies that family's boundary, ownership, and stop conditions.
```

The constitutional authority is therefore not primarily an operator, artifact, pressure item, recommendation, vocabulary term, methodology label, or implementation desire. It is an evidence-gated, boundary-local permission to move a bounded subject.

The recurring constitutional participants are:

1. **Evidence**: the only recurring mover from Null, candidate, unsupported, pressure, or architecture toward a more committed state.
2. **Family-local boundary / owner / corridor**: the local place where evidence can lawfully be interpreted for that promotion family.
3. **Explicit stop condition**: the counter-authority that prevents unsupported movement.
4. **Repository-visible preservation surface**: implementation, tests, diagnostics, reports, completion audits, or bounded handoff artifacts that make the promotion reviewable.

No universal promoter has been earned.

Short answer:

```text
Evidence authorizes movement,
but only through the local constitutional boundary
that is competent to interpret that evidence
for that promotion family.
```

## Promotion families

### 1. Null -> Knowledge / supported state

**Who authorizes movement?**

Repository-visible evidence authorizes movement. The Constitution states that Null persists until evidence justifies transition and that only evidence may move a subject from Null toward candidate, supported, bounded, verified, or trusted states.

**What evidence authorizes movement?**

Observable repository material: implementation behavior, tests, diagnostics, observations, compatibility behavior, reports, and bounded handoff artifacts. The Null discipline investigation shows this across capability verification, selection visibility, diagnostic inventory, diagnostic shape audit, and read-only authority surfaces.

**What does not possess authority?**

Observation alone, vocabulary, operator concern, projection output, diagnostic output, presentation labels, and architectural preference do not become knowledge by appearing.

**Automatic?**

No. The default is uncommitted. Unsupported evidence leaves the subject at Null, candidate, unsupported, or insufficient evidence.

**Explicit evidence required?**

Yes.

**Local or universal?**

The discipline is recurring and constitutional, but implementations are local. The repository does not show a universal Null-transition runtime pipeline or shared ownership family.

### 2. Candidate -> Admitted / verified

**Who authorizes movement?**

The local admission or verification boundary authorizes movement when it has repository evidence that satisfies its rules. Examples include capability verification relying on projected `capability_verified` facts and provider-local validators accepting provider-language shape before observation construction.

**What evidence authorizes movement?**

Existing verified facts, accepted provider-specific contract evidence, declared diagnostic shapes, tests that preserve behavior, or family-specific admission evidence.

**What does not possess authority?**

Candidate presence, format hypothesis, requested capability, executable operation metadata, inventory display, or provider vocabulary does not admit the candidate by itself.

**Automatic?**

No. Candidate status is intentionally weaker than admission.

**Explicit evidence required?**

Yes. The evidence may be local and family-specific; it need not be a universal admission token.

**Local or universal?**

Family-local. The evidence required to admit a provider-language candidate differs from the evidence required to verify a capability or admit diagnostic shape.

### 3. Working methodology -> Earned methodology

**Who authorizes movement?**

Repeated implementation-backed use across investigations authorizes methodology promotion, as interpreted through reconciliation reports.

**What evidence authorizes movement?**

Independent recurrence, prevention of recurring constitutional mistakes, material changes in understanding, evidence ledgers, authority boundaries, negative authority findings, stop conditions, and confidence calibration. The competency methodology reconciliation classifies sections as Earned, Working, or Insufficient Evidence rather than promoting the whole method automatically.

**What does not possess authority?**

A useful section name, elegant report shape, repeated prose, or local convenience does not earn methodology.

**Automatic?**

No. Some recurring sections remain working methodology, and some remain insufficient evidence.

**Explicit evidence required?**

Yes. The evidence must show the methodology repeatedly prevented mistakes or produced unique constitutional evidence.

**Local or universal?**

Mixed. The discipline can recur across investigations, but each methodological element is promoted or refused independently.

### 4. Architecture -> Implementation

**Who authorizes movement?**

Implementation readiness is authorized by converging repository evidence: a bounded subject, stable implementation-real owner or corridor, recurring boundary, implementation necessity, compatibility-preserving slice, and testable preservation surface.

**What evidence authorizes movement?**

The strongest evidence is an independently recurring boundary plus a concrete owner/corridor and a smallest behavior-preserving change that can be proved by tests, diagnostics, or shape checks.

**What does not possess authority?**

Architecture accumulation, pressure, operator excitement, elegant vocabulary, frontier status, implementation convenience, or methodology stability alone.

**Automatic?**

No. Architecture may remain characterization, frontier, typed unknown, or insufficient evidence.

**Explicit evidence required?**

Yes, and implementation promotion requires stricter evidence than methodology promotion because it changes or preserves executable surfaces.

**Local or universal?**

Local. The repository shows a recurring readiness discipline, not a universal implementation mandate.

### 5. Pressure -> Inquiry

**Who authorizes movement?**

Pressure may orient inquiry only when observable material identifies a bounded unknown or compressed owner/corridor. Classification and inquiry boundaries decide whether pressure becomes a lawful inquiry subject.

**What evidence authorizes movement?**

Concrete owners, corridors, evidence streams, report shapes, diagnostic rows/counts, measurements, completion-audit residuals, compatibility residue, or repeated manual reconstruction tied to repository evidence.

**What does not possess authority?**

Pressure itself does not command work, select implementation, recover ownership, or prove readiness. Operator pain is valid orientation evidence but not cause authority.

**Automatic?**

No. Pressure can stop as adjacent, unsupported, frontier-only, already recovered, or insufficient evidence.

**Explicit evidence required?**

Yes. A pressure label without observable repository material is not enough.

**Local or universal?**

Mixed but mostly local. Pressure is a recurring constitutional input, while pressure evidence and classification remain local or methodological rather than a universal implementation owner.

### 6. Observation -> Fact

**Who authorizes movement?**

The local fact/admission/projection boundary authorizes movement only after observation evidence satisfies repository rules for fact preservation or verification.

**What evidence authorizes movement?**

Provider-specific validation, observation model construction, recorded fact semantics, projected verified facts, or tests proving a surface preserves the distinction.

**What does not possess authority?**

Observation itself. The Constitution explicitly preserves `Observation is not fact`; projection is not verification; diagnostic output is not cluster truth.

**Automatic?**

No.

**Explicit evidence required?**

Yes.

**Local or universal?**

Family-local. Provider translation, observation import, capability verification, diagnostic facts, and cluster facts use different authority boundaries.

### 7. Evidence -> Conclusion

**Who authorizes movement?**

The bounded inquiry, classification, audit, or characterization surface authorizes the conclusion by interpreting preserved evidence within stated boundaries and stop conditions.

**What evidence authorizes movement?**

Evidence ledgers, implementation references, tests, counterexamples, recurring observations, negative observations, and explicit uncertainty classifications.

**What does not possess authority?**

Evidence accumulation alone. Uninterpreted evidence does not automatically become conclusion; a conclusion must name scope, authority, non-authority, and stop condition.

**Automatic?**

No. Evidence can authorize supported conclusion, insufficient evidence, unsupported conclusion, or lawful stop.

**Explicit evidence required?**

Yes.

**Local or universal?**

Local to the inquiry family, with recurring constitutional discipline.

### 8. Recovery -> Frontier

**Who authorizes movement?**

A legitimate recovery stop plus recurring future pressure bounded by existing recovered architecture may authorize frontier characterization.

**What evidence authorizes movement?**

Completion audits, ownership-family change, insufficient-evidence findings, repeated autonomy/review/responsibility gaps, explicit negative authority, compatibility expectations, and bounded evaluation criteria.

**What does not possess authority?**

A completed recovery does not automatically produce a frontier. A frontier does not automatically authorize implementation.

**Automatic?**

No. Recovery can simply stop.

**Explicit evidence required?**

Yes.

**Local or universal?**

Mixed. Frontier promotion is a recognizable human-applied characterization pattern, not an implemented universal workflow.

## Negative authority

The refusals share a recurring constitutional pattern:

```text
A thing may orient inquiry
without possessing authority to promote.
```

Repeated non-authorities include:

| Apparent authority | What it may do | What it may not do |
| --- | --- | --- |
| Pressure | Orient inquiry toward compressed responsibility, pain, or missing visibility. | Command recovery, implementation, work selection, or frontier promotion. |
| Recommendation | Point to a possible inspection or next command. | Create truth, priority, ownership, or implementation readiness. |
| Operator testimony | Provide observation, concern, context, and pressure. | Override repository evidence or admit claims into knowledge. |
| Vocabulary | Help describe patterns. | Stabilize repository knowledge without implementation evidence. |
| Accumulation | Increase pressure to reconcile or audit. | Substitute for boundary stability, ownership, tests, or evidence. |
| Visibility | Make something seen. | Grant authority, mutation, verification, or cluster truth. |
| Diagnostic output | Preserve read-only findings. | Become cluster facts unless an explicit boundary says so. |
| Projection | Present a derived view. | Verify facts or authorize mutation. |
| Candidate | Preserve possibility. | Become admitted or verified automatically. |
| Frontier | Preserve unresolved future pressure. | Prove implementation readiness or schedule work. |
| Methodology | Discipline inquiry. | Mutate runtime or grant architectural authority beyond its evidence. |

The shared pattern is not that these are useless. They are useful precisely because they are bounded. Their constitutional role is orientation, preservation, or discipline, not promotion authority.

## Promotion participants

Promotion authority does not belong primarily to one participant in all families.

| Participant | Characterization |
| --- | --- |
| Artifacts | Artifacts carry evidence, status, or handoff material. They rarely self-authorize promotion. |
| Responsibilities | Responsibilities/owners/corridors often provide the local boundary where promotion can be evaluated. |
| Competencies | Competencies can classify, verify, audit, or hand off inside their bounded authority. They do not become universal promoters. |
| Evidence | Evidence is the recurring movement-permitting participant. It is necessary but must be interpreted locally. |
| Operators | Operators provide observations, pressure, tasks, and context. They do not possess constitutional override authority. |
| Methodology | Methodology disciplines promotion review and prevents mistakes. It can be earned, but it does not by itself promote runtime state. |
| Implementation | Implementation is the strongest court of repository evidence and can preserve promoted boundaries. It does not automatically make every architectural phrase true. |
| Unknown | Several promotion cases remain typed unknowns; unknown must not be collapsed into an implied authority. |

Smallest supported synthesis:

```text
Evidence is the recurring mover;
local bounded responsibility is the recurring interpreter;
stop conditions are the recurring refusal authority.
```

## Locality

Promotion authority is **mixed**, with a family-local center.

### Family-local

Most actual promotion gates are family-local:

- provider-language candidates use provider-specific validation;
- capability verification uses capability-specific verified facts;
- diagnostic recording uses declared diagnostic scope and shape audit;
- implementation readiness uses owner/corridor, compatibility, and tests;
- methodology promotion uses independent recurrence across reports;
- pressure classification uses observable pressure evidence and classification boundaries.

### Boundary-local

Even inside a family, authority is often boundary-local. The current-facts timing chain distinguishes measurement, interpretation, diagnostic payload, and presentation. State-build cache-debug work distinguishes evidence streams, assembly, final evidence, report payload, compatibility report, and presentation. A downstream formatter does not own upstream evidence admission merely because it renders the result.

### Globally shared

Only the constitutional constraints are broadly shared:

- Null persists until evidence justifies transition.
- Repository authority wins.
- Authority must be explicit.
- Stop is a positive result.
- Visibility is not authority.
- Observation is not fact.
- Candidate is not verified.
- Pressure is not recovery.

These are not a global promoter. They are global constraints on local promotion.

## Constitutional pressure on promotion authority

Promotion authority itself is governed by recurring pressure from:

1. **Evidence**: no movement without support.
2. **Boundaries**: movement must occur inside the family or owner competent to interpret the evidence.
3. **Stop conditions**: insufficient evidence, unsupported domain, adjacent family, frontier-only, or unknown are lawful outcomes.
4. **Explicit ownership**: no owner or promotion authority is presumed without recurring implementation evidence.
5. **Compatibility / preservation**: implementation promotions must preserve or deliberately justify public behavior changes.
6. **Negative authority**: apparent authorities must be named and refused where unsupported.
7. **Typed unknowns**: unresolved promotion cases remain open rather than inferred.

No additional constitutional participant has been earned as a universal promoter.

## Recurring patterns

The recurring promotion-authority grammar is:

```text
subject appears
↓
subject remains uncommitted / candidate / pressure / working / architectural
↓
local evidence is gathered
↓
competent boundary interprets evidence
↓
movement is either supported, refused, deferred, or stopped
↓
artifact records evidence, authority, non-authority, locality, and stop condition
```

The recurring refusal grammar is:

```text
apparent authority appears
↓
it is useful for orientation or presentation
↓
repository evidence does not grant promotion authority
↓
it remains bounded and non-authoritative
```

## Smallest truthful answer

The smallest truthful source of constitutional promotion authority is:

```text
Repository evidence sufficient for the local boundary that owns the proposed movement.
```

Even shorter:

```text
Local evidence-bound authority.
```

This means:

- not evidence alone, because evidence must be interpreted by a competent local boundary;
- not boundary alone, because no boundary may promote without evidence;
- not operator desire, because operators orient but do not constitutionally override evidence;
- not methodology alone, because methodology disciplines but does not substitute for repository support;
- not implementation alone, because implementation is evidence and preservation surface, not a universal semantic truth-maker;
- not the Constitution document alone, because it is an orientation artifact and implementation evidence wins.

This investigation terminates at characterization.

- a promotion manager;

- a new diagnostic surface;

The repository supports recurring promotion discipline and recurring local authority boundaries. It does not support one universal constitutional promoter.

## Confidence

High confidence:

- no universal promotion authority has been earned;
- evidence is the recurring movement-permitting participant;
- promotion authority is family-local or boundary-local in actual reviewed families;
- pressure, vocabulary, operator testimony, recommendations, accumulation, and visibility repeatedly lack promotion authority by themselves.

Medium confidence:

- methodology promotion and implementation promotion share a constitutional discipline while using different final gates;
- recovery-to-frontier promotion is a recognizable human-applied pattern but not a runtime competency.

Low confidence / unknown:

- rollback, emergency promotion, concurrent promotion, and cross-family authority conflict.
