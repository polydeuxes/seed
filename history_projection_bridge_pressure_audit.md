## Constitutional testimony

Question answered only from the constitutional side:

```text
From the constitutional side,

when preserved history changes,

what becomes available

to neighboring responsibilities,

and what explicitly
does not?
```

### Sender

The sender is append-only constitutional history, characterized by recurring repository evidence as durable event-supported transition material under provenance.

It is not a speaking agent, runtime owner, scheduler, or behavior mechanism. Its constitutional role is to preserve auditable event provenance and transition input so current lawful projections can be rebuilt, inspected, challenged, and explained.

### Artifact

The artifact made available is preserved append-only event history / transition input with provenance, event identity, kind, payload, workspace, actor, causation, and authority boundary as applicable to the recorded event.

Constitutional reports also distinguish the absence of preserved event history from an empty projection: before the first preserved event, there is no event to replay, no event provenance, no event actor, no event kind, and no event payload. Once preservation occurs, the newly available artifact is preserved history, not a completed present-tense truth object.

### Constitutional responsibilities

When preserved history changes, neighboring responsibilities may receive or rely on:

- replayable transition input;
- auditable provenance-bearing event history;
- material from which a current projected condition may be rebuilt;
- material from which later explanation may be composed;
- inspectable support, conflict, confidence, unknown, and authority-boundary material when present;
- a subordinate relation for caches and views: they may be rebuilt from history, but they do not replace history.

The constitutional side supports this recurring shape:

```text
Preserved history changes
→ replay/audit input becomes available
→ current lawful projection may be rebuilt or explained under projection, warrant, reliance, and unknown-preservation discipline
```

### Boundaries

Preserved history changing does not, by itself, produce universal current reality. Repository evidence repeatedly keeps event history, projection, current belief, evidence, facts, views, context, diagnostics, and audit output separate.

The constitutional boundary is:

```text
history is projection/audit authority
not objective truth
not cached state
not current lawful condition by itself
```

The constitutional side also preserves bounded movement after Null: evidence, not desire or recurrence alone, moves a subject from Null toward a supported state. A first preserved event may make event-supported material available, but it does not dictate the universal event kind, universal startup sequence, or universal downstream responsibility.

### Explicitly non-crossing responsibilities

Preserved history explicitly does not cross into these responsibilities merely by changing:

- deciding truth;
- becoming objective reality;
- forcing all Seeds or scopes to converge;
- resolving contradictions by existence alone;
- verifying external testimony merely because it was recorded;
- owning runtime behavior;
- owning host mutation;
- owning scheduling, lifecycle, federation, or execution;
- replacing evidence, warrant, confidence, contradiction, reliance, or unknown-preservation disciplines;
- becoming the cached projection, consumer view, or diagnostic output;
- producing behavior directly.

### Constitutional-side confidence

High confidence that the constitutional side sends preserved, auditable transition input to neighboring responsibilities and explicitly refuses truth, verification, mutation, runtime, and cache/view ownership. Medium confidence that `constitutional history` is the best vocabulary, because the reports support the function while noting that implementation vocabulary remains event history and projection.

## Bridge comparison

Comparison begins only after the independent testimonies and translations above.

### Recurring bridge shape

The recurring bridge shape is visible as an availability-and-boundary seam, not as an engine:

```text
Preserved history / event boundary changes
→ changed transition input or last-event dependency boundary becomes visible
→ neighboring projection/cache/read-model responsibilities may consume that bounded evidence
→ projected/read-model/report surfaces may rebuild, hit cache, miss cache, skip, or explain status
```

In constitutional terms, this occupies the grammar position between History and Affected State. The bridge is a bounded recognition of history-derived relevance to projection, but the audit does not recover a formal Recognition framework.

### Matching responsibilities

The sides match on these responsibilities:

- history/event boundary is authoritative input for projection-facing rebuild or explanation;
- caches, snapshots, and reports remain subordinate to event/history boundaries;
- projection-facing consumers require bounded input evidence before changing output;
- provenance, event identity, dependency identity, cache status, and selected diagnostic evidence matter before projection-facing output is trusted;
- non-promotion boundaries are central: history is not truth, projection is not verification, diagnostic output is not cluster truth, and report evidence does not own mutation or execution.

### Mismatched assumptions

The sides do not support these assumptions as shared facts:

- that constitutional `Recognition` exists as a named responsibility;
- that affected-state projection is a repository-wide owner rather than a localized implementation pattern in the reviewed cache-debug path;
- that preserved history changing necessarily forces rebuild rather than allowing cache hit, cache miss, skipped projection, unavailable cache, or bounded explanation;
- that current lawful condition and implementation `State` are the same artifact;
- that projection diagnostics are the same responsibility as projection execution;
- that read-model cache discipline is constitutional history ownership.

### Unsupported crossings

Unsupported bridge crossings are:

- append-only constitutional history directly owning affected-state determination;
- preserved history becoming current truth without projection/warrant/unknown discipline;
- cache-debug projection evidence owning projection execution or ledger replay;
- diagnostic/report output mutating cluster truth;
- implementation cache identity becoming a constitutional family;
- presentation vocabulary such as `affected-state projection` becoming preserved repository knowledge beyond the implementation evidence reviewed here;
- a Recognition framework being introduced from the bridge grammar alone.

### Preserved unknowns

The audit preserves these unknowns:

- Whether the repository should name a bridge responsibility `Recognition`.
- Whether a general affected-state projection responsibility exists outside localized state-build cache-debug evidence.
- Whether all constitutional transitions and implementation projection dependencies share a single event-ledger boundary in every path.
- Whether future evidence will establish a formal bridge owner between append-only history and affected-state projection.
- Whether constitutional current lawful condition maps cleanly to implementation State across all surfaces.

### Confidence

Medium-high confidence that a bridge shape is visible as bounded history/event-boundary recognition before projection-facing consumption. Medium confidence that a named constitutional responsibility occupies the bridge, because the evidence supports the role but not a formal family or owner.
