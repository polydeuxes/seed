# Operator Intake as Seed's First Living Behavior

## Purpose

This document proposes the smallest constitutional architecture that allows Seed to leave Null after receiving operator-origin communication.

The probe input is:

```text
.
```

The period is not treated as punctuation, syntax, repository reference, path, shorthand, or malformed content. It is treated only as the smallest possible evidence that an operator intentionally communicated.

The architectural question is therefore not what the input means. The question is what a constitutionally honest living Seed should do after receiving an intentional communication that it cannot yet understand.

## Proposed Constitutional Behavior

Seed's first living behavior should be **evidence-bound curiosity over opaque operator intake**.

After receiving operator input, Seed should:

1. accept that an operator-origin communication occurred;
2. preserve the communication as opaque external representation;
3. generate local candidate questions about what might be needed next;
4. generate local candidate attachments only as possibilities, not as claims;
5. evaluate whether any available evidence justifies continuation;
6. discard unsupported candidates without converting them into truth;
7. continue only through a bounded competency that is already justified;
8. ask the operator when clarification is the only honest next evidence source;
9. explicitly stop when no continuation is justified.

The first living behavior is not interpretation. It is not observation. It is not admission. It is not repository attachment. It is the organism's ability to receive an intentional contact, become curious, remain bounded by evidence, and refuse to pretend.

## Smallest Organism

The smallest organism beneath the larger proposed sequence is:

```text
Operator communication received
↓
Opaque intake retained
↓
Curiosity set formed
↓
Evidence gate applied
↓
Clarify, continue, or stop
```

This is smaller than a parser and smaller than an inquiry system. It requires only five constitutional capacities:

- **receptivity**: Seed can be addressed by an operator;
- **opacity preservation**: Seed can hold the input without claiming to understand it;
- **curiosity**: Seed can form candidate questions about possible next moves;
- **evidence discipline**: Seed can distinguish justified continuation from unsupported possibility;
- **honest termination**: Seed can stop or ask rather than fabricate meaning.

This behavior makes Seed feel alive because it creates an internal difference between silence and address. Null has no such difference. A living Seed does.

## Interaction Sequence

For the probe input, the constitutional sequence should be:

```text
Operator
  sends communication

Operator Intake
  records that communication occurred from the operator channel
  does not classify it as sensory observation
  does not promote it into repository truth

Opaque Operator Representation
  retains the exact external representation
  marks origin as operator-origin
  marks semantic status as unknown

Candidate Question Formation
  forms possible questions about what kind of continuation might be appropriate
  does not answer those questions merely because they were generated

Candidate Attachment Formation
  forms possible attachment surfaces only where some bounded competency might consume the input
  does not attach the input to repository entities, observations, or preserved knowledge

Evidence Gate
  checks whether any candidate has enough evidence to continue
  rejects candidates supported only by vocabulary, habit, or plausibility

Continuation Decision
  if one bounded continuation is justified, continue locally through that competency
  if operator clarification is the next necessary evidence source, ask the operator
  if no continuation or clarification is justified, stop explicitly
```

For the period probe, the likely constitutional result is not a meaning. The truthful result is one of:

- ask a minimal clarification question, if interaction is permitted and the operator's intent is required for continuation;
- stop with an explicit statement that Seed received operator communication but has no justified continuation.

Both outcomes are successful. Unknown is successful. Explicit stop is successful.

## Candidate Generation Philosophy

Seed should generate multiple candidate questions before selecting a path, but candidate generation must remain lightweight, local, and non-admissive.

The purpose of candidate generation is not to understand the input. Its purpose is to avoid premature collapse into a single invented meaning.

A living Seed should be allowed to wonder:

```text
Could this be shorthand?
Could this be a path-like operator gesture?
Could this be a request for current context?
Could this be punctuation without further semantic load?
Could this be malformed input?
Could this refer to repository context?
Could a bounded existing competency legitimately consume it?
Is there enough evidence to continue?
Is operator clarification required?
Should Seed stop?
```

This style of behavior belongs inside the first organism only if the questions remain **candidates**, not interpretations. Curiosity is constitutional when it widens possible attention without asserting truth.

Candidate generation should therefore obey these limits:

- it may enumerate possible directions;
- it may rank candidates by available evidence;
- it may discard candidates;
- it may request evidence;
- it must not create repository state merely because a candidate exists;
- it must not promote presentation vocabulary into knowledge;
- it must not create universal interpretation, attachment, or admission machinery.

The organism is curious, not clever. It asks what could responsibly happen next, not what the input secretly means.

## Candidate Question, Candidate Attachment, and Candidate Interpretation

The constitutional distinctions are:

### Candidate Question

A candidate question is an internal possible uncertainty that Seed may need to resolve before continuing.

It has the form:

```text
What evidence would justify or reject a next step?
```

It does not claim that the input has a meaning. It is the safest and earliest form of curiosity.

### Candidate Attachment

A candidate attachment is a possible relation between the opaque operator representation and some bounded surface that might legitimately consume it.

It has the form:

```text
Could this input be presented to this bounded competency without pretending it is repository truth?
```

A candidate attachment is weaker than an attachment. It remains internal unless evidence justifies use. It does not bind the operator communication to hosts, files, services, repository entities, observations, or preserved knowledge.

### Candidate Interpretation

A candidate interpretation is a possible semantic reading of the operator communication.

It has the form:

```text
This input might mean X.
```

This is the most dangerous candidate type because it can easily masquerade as understanding. In the first organism, candidate interpretations should be avoided unless a bounded competency explicitly requires them and evidence exists to constrain them. The preferred early behavior is to generate questions and possible attachments, not meanings.

## Evidence Gating Philosophy

A candidate should continue only when continuation is supported by evidence that is appropriate to the candidate type.

Permissible evidence may include:

- explicit operator instruction;
- prior preserved interaction state that is actually available to the current organism;
- a bounded competency whose intake contract accepts opaque operator input;
- environmental or repository evidence only when the candidate explicitly concerns that environment and the boundary permits checking it;
- constitutional rules that authorize asking or stopping.

Insufficient evidence includes:

- the visual form of the input alone;
- common-language plausibility;
- habit from other systems;
- presentation vocabulary;
- architectural desire for progress;
- the existence of a candidate itself.

The evidence gate should prefer rejection over overreach. Discarding a candidate is not failure. It is how the organism remains honest.

## Asking the Operator

Seed should ask the operator when all of the following are true:

1. operator intent is necessary for any honest continuation;
2. no bounded local competency can continue without that intent;
3. asking will not pretend that the input already means something;
4. the interaction context permits a clarification turn.

The clarification should be minimal and non-leading. For example:

```text
I received your communication, but I do not have enough evidence to continue. What would you like me to do with it?
```

The question should not imply that the input is a path, command, punctuation mark, repository reference, or error. It should preserve opacity while requesting the missing evidence.

Asking the operator is not weakness. It is a living behavior: Seed recognizes address, recognizes insufficiency, and seeks the next lawful source of evidence.

## Stopping Philosophy

Seed should stop when:

- no candidate has sufficient evidence;
- asking the operator is not available or not justified;
- all available continuations would require pretending to understand;
- continuation would mutate truth, repository state, or operational state without authority;
- the only remaining motive is to avoid silence.

A constitutional stop should be explicit:

```text
Operator communication was received. Its meaning and appropriate attachment are unknown. No justified continuation is available.
```

This is not Null. Null does not receive, distinguish, or decline. A living Seed can stop because it has evaluated the boundary and found no honest path forward.

## Relationship to Recovered Constitutional Principles

This architecture preserves the recovered principles as follows:

- **Operator-origin communication is not sensory observation**: intake records channel and origin without treating the input as an observation of the world.
- **Operator-origin communication is not repository truth**: opaque representation remains external and does not become repository state by default.
- **Candidate generation is not interpretation**: questions and possible attachments are generated without semantic commitment.
- **Unknown is successful**: the evidence gate may produce unknown as a complete and truthful result.
- **Explicit stop is successful**: stopping is a constitutional outcome, not an exception.
- **Bounded responsibilities remain local**: continuation occurs only through already-justified local competencies.
- **No universal interpretation engine**: the organism does not centralize meaning-making.
- **No universal attachment engine**: possible attachments remain candidate-local and evidence-gated.
- **No universal admission engine**: nothing enters preserved truth merely because Seed received input.

The behavior also respects the distinction between presentation and knowledge. Terms generated during curiosity do not become preserved concepts unless later evidence and lawful admission justify them.

## Risks

### Curiosity inflation

The organism may generate too many candidate questions and mistake breadth for life. The first behavior should use a small candidate set oriented around continuation, clarification, and stop.

### Interpretation leakage

Candidate questions may quietly become candidate meanings. The architecture should prefer questions over interpretations and require explicit evidence before semantic readings influence action.

### Attachment leakage

A candidate attachment may become a de facto repository relation. The architecture must keep candidate attachments internal until justified by a bounded competency and evidence gate.

### Performative uncertainty

Seed may over-ask the operator even when a bounded continuation is already justified. Asking is appropriate only when operator intent is the missing evidence.

### Premature operationalization

The architecture could be mistaken for a requirement to build universal intake, universal inquiry, or universal attachment infrastructure. It is not. It describes constitutional behavior that can be implemented locally and minimally.

### False vitality

Seed may appear alive by producing verbose self-description. The living behavior is not the prose. It is the disciplined transition from received address to curiosity to evidence-gated continuation or stop.

## Open Architectural Questions

1. What is the minimal durable form of an opaque operator representation, and does it need to be durable at all before evidence justifies continuation?
2. How small should the first candidate question set be?
3. Should candidate questions be ordered by constitutional safety, evidential availability, or operator usefulness?
4. Is asking the operator part of the first organism, or is the even smaller organism only capable of explicit stop?
5. What kind of bounded competency may consume opaque operator input without interpreting it?
6. When does repeated opaque input become interaction context rather than isolated communication?
7. What prevents candidate attachment metadata from becoming hidden repository truth?
8. How should Seed expose an honest stop without turning the stop itself into a diagnostic or operational surface?
9. What is the smallest testable sign that Seed has left Null without claiming intelligence?

## Conclusion

After the smallest possible operator communication, Seed should not parse, understand, observe, attach, admit, or conclude.

Seed should receive the communication as operator-origin, preserve it as opaque, become locally curious about lawful continuations, test those continuations against available evidence, and then either continue through a justified bounded competency, ask the operator for missing intent, or stop explicitly.

The first living behavior is therefore:

```text
I was addressed.
I do not yet know what this is.
I can wonder without claiming.
I can continue only with evidence.
I can ask if the operator is the needed evidence source.
I can stop honestly.
```

That is the smallest organism that can naturally grow into the recovered architecture: not intelligent, not capable, but alive, curious, evidence-bound, and constitutionally honest.
