# Constitutional Neighborhood Characterization Methodology

## Selected neighborhood

The selected survey target is the recently recovered grammar-transformation neighborhood, treated only as an observed neighborhood and not as a promoted architectural family.

The bounded neighborhood is the chain in which substrate or local records are consumed by an implementation-local owner, transformed into a bounded artifact grammar, and handed to a neighboring competency that asks only questions legal within that artifact grammar.

Most implementation-backed center of the neighborhood:

```text
Structure Observation boundary
↓
substrate adapters / observation record producers
↓
Observation Agreement candidate records
↓
Grammar Observation recurring-shape records
↓
neighboring question, orientation, answer, lineage, and audit consumers
```

This report does not claim a universal grammar-transformation architecture. The neighborhood is used as the survey target because it exposes multiple touching constitutional surfaces: upstream substrate authority, local transformation ownership, downstream eligibility, horizontal neighbor topology, vertical law/artifact relations, temporal transition boundaries, and counterexamples.

## Upstream characterization

### Lawful substrate

Lawful upstream substrate is local and owner-specific:

- Structure Observation lawfully names only substrate-independent structural extraction, evidence preservation, read-only status, and non-interpretation.
- Repository Artifact Observation lawfully consumes caller-provided Python source text plus a source path.
- Documentation Structure lawfully consumes Markdown documentation files and documentation-local structure.
- Relationship Observation lawfully consumes explicit documentation navigation metadata or caller-provided Python source text, depending on adapter.
- Observation Agreement does not consume original substrate. Its lawful upstream material is already-observed documentation relation records, repository artifact facts, and relationship facts.
- Grammar Observation does not consume original substrate or lower-level records. Its lawful upstream material is `ObservationAgreementRecord` instances.

### Lawful adapter

A lawful adapter is an implementation-local owner that consumes a substrate or upstream record grammar and emits a bounded artifact without acquiring downstream promotion authority.

Recovered adapter examples:

- `RepositoryArtifactObservationAdapter` transforms supplied Python text into module, class, function, method, and import `RepositoryArtifactFact` records.
- Documentation Structure transforms Markdown files into documentation structure, recurrence, membership, and relation records while preserving read-only/non-interpretive boundaries.
- Relationship Observation transforms explicit metadata or Python syntax into `RelationshipFact` records.
- Observation Agreement transforms already-supplied observation records into candidate agreement records.
- Grammar Observation transforms candidate agreement records into recurring relation-shape records.

### Adapter authority

Adapter authority includes only the local transformation it implements:

- read-only structural extraction;
- evidence preservation;
- local parse or record-construction authority where explicitly implemented;
- provenance preservation;
- compatibility preservation for existing public surfaces;
- emitting bounded records for downstream consumption.

Adapter authority does not include global grammar ownership, responsibility recovery, architectural promotion, runtime execution, event-ledger mutation, repository mutation, or cluster mutation unless another implementation surface explicitly owns those acts.

### Explicit negative authority

The upstream boundary is defined as much by refusal as by output.

Negative authority recovered in implementation includes:

- Structure Observation does not interpret content, own substrate parsing, own grammar, recover responsibility, own lexicon, write event ledgers, mutate repositories, or mutate clusters.
- Repository Artifact Observation never reads files, scans repositories, imports modules, uses LLMs, reconciles claims, or integrates with runtime/tool execution.
- Documentation Structure refuses prose interpretation, claim extraction, authority inference, shape inference, ontology promotion, event writes, and repository mutation.
- Relationship Observation refuses behavior, call, route, boundary, ownership, runtime reachability, graph reconciliation, runtime execution, and repository traversal claims.
- Observation Agreement refuses substrate parsing, semantic inference, grammar ownership, lexicon ownership, responsibility recovery, promotion, truth, ledgers, and mutation.
- Grammar Observation refuses raw substrate consumption, semantic meaning, responsibility/family recovery, lexicon ownership, capability promotion, architectural truth, ledgers, and mutation.

### Constitutional condition entering the neighborhood

The neighborhood begins lawfully only when a bounded owner has either:

1. a substrate it is explicitly authorized to observe; or
2. an upstream artifact grammar it is explicitly authorized to consume.

The entering condition is therefore not “possible meaning exists.” It is “lawful observed material exists for this owner.” If the material is unavailable, invalid, unsupported, or outside owner authority, the lawful condition is no output, empty output, diagnostic uncertainty, or refusal rather than promotion.

## Transformation characterization

### Consumed grammar

Consumed grammars are local:

| Owner | Consumed grammar |
| --- | --- |
| Structure Observation | Boundary conditions for structural extraction, not substrate text itself. |
| Repository Artifact Observation | Caller-provided Python source text and source path. |
| Documentation Structure | Markdown documentation structure and explicit metadata. |
| Relationship Observation | Explicit documentation metadata or caller-provided Python source text. |
| Observation Agreement | Supplied documentation relation records, repository artifact facts, and relationship facts. |
| Grammar Observation | `ObservationAgreementRecord` instances. |
| Question Surface Inventory | Exact `question_family` strings and static maps. |
| Inquiry Orientation | Preserved `InquiryNoteRecord` plus projected `State` read models. |

### Emitted grammar

Emitted grammars are likewise bounded:

| Owner | Emitted grammar |
| --- | --- |
| Repository Artifact Observation | `RepositoryArtifactFact` records for modules, classes, functions, methods, and imports. |
| Documentation Structure | Documentation structure outputs, recurrence/membership records, and documentation architectural relation records. |
| Relationship Observation | `RelationshipFact` records. |
| Observation Agreement | `ObservationAgreementRecord` candidate agreements with supporting evidence, participating streams, provenance, and non-promotion boundary. |
| Grammar Observation | `GrammarObservationRecord` recurring relation shapes with supporting agreements, provenance, recurrence evidence, and non-promotion boundary. |
| Question Surface Inventory | Eligibility, selection, dispatch request, and dispatch result records. |
| Inquiry Orientation | `InquiryOrientationView` over related material, uncertainty, and authority boundary. |

### Preserved properties

The transformation surface preserves:

- identity appropriate to the owner: source path, symbol, relation subject/object, stream, question family, note id, support id, diagnostic/surface name;
- provenance: line references, source paths, fact paths, stream provenance, support strings, source-navigation matches;
- evidence: supporting records and recurrence evidence;
- authority boundaries: read-only, non-promotion, non-mutation, diagnostic-only, not-dispatchable, no semantic interpretation;
- compatibility: existing public shapes and wrapper behavior where present;
- unknowns and limitations rather than filling gaps by inference.

### Intentionally discarded properties

Transformations intentionally discard or refuse:

- raw substrate access by downstream consumers that do not own it;
- semantic meaning not represented by the local artifact;
- inferred ownership, capability, family identity, or responsibility;
- architectural truth from agreement or recurrence alone;
- operator intent from lexical overlap alone;
- behavior, execution, or mutation authority from diagnostic or observation evidence;
- presentation vocabulary as preserved knowledge unless another evidence surface supports it.

### Supporting evidence preserved

Observation Agreement preserves supporting evidence records with stream, provenance, and evidence text. Grammar Observation preserves supporting agreement records, flattened provenance, recurrence evidence, and the observed relation shape. Inquiry Orientation preserves support strings explaining lexical overlap against projected fact supports and source navigation. Question Surface Inventory preserves question family, bounded status, required args, permitted flag, selected dispatch surface, and reason.

### Unknown preservation

Unknown preservation appears in multiple forms:

- unsupported question families become `not_dispatchable` instead of invented work;
- diagnostic-only question families remain diagnostic-only rather than answer surfaces;
- invalid Python returns no relationship facts or a parse-failure-limited module fact, depending on adapter;
- orientation with no lexical matches states absence of deterministic related material without proving unrelatedness;
- candidate agreements and grammar observations retain non-promotion boundaries rather than becoming truth.

### Local ownership

No recovered transformation owner owns the whole chain. Each owner owns only its local consumption/emission boundary:

- Structure Observation owns the shared structural-observation boundary.
- Substrate adapters own parsing and record construction.
- Observation Agreement owns candidate grouping over supplied records.
- Grammar Observation owns recurring syntactic shape observation over candidate agreements.
- Question Surface Inventory owns exact family-to-status/dispatch metadata.
- Inquiry Orientation owns lexical orientation over preserved notes and projected read models.

## Downstream characterization

### Neighboring consumers

Observable downstream consumers include:

- Observation Agreement consuming documentation relation records, repository artifact facts, and relationship facts.
- Grammar Observation consuming Observation Agreement records.
- Bounded question selection and dispatch consuming exact question-family eligibility artifacts.
- Inquiry Orientation consuming preserved inquiry notes and projected state read models.
- Answer and audit surfaces consuming evaluated material, support, boundary, limitation, lineage, and unknown payloads.
- Cross-neighborhood analysis consuming the artifact-handoff evidence as topology evidence, not as implementation authority.

### Newly legal questions

Once a transformed artifact exists, new local questions become legal:

- Do two or more independent observation streams contain exactly matching evidence text?
- Which streams and provenance support a candidate agreement?
- Do at least two candidate agreements share the same syntactic relation shape?
- Which recurring shape has recurrence evidence?
- Is an exact question family eligible now, eligible with parameters, diagnostic-only, or not dispatchable?
- Which projected facts or source-navigation rows lexically overlap a preserved inquiry note?

### Newly impossible questions

The same transformation makes other questions impossible for that owner:

- Observation Agreement cannot ask whether the matched evidence is semantically true.
- Grammar Observation cannot ask what the relation terms mean.
- Inquiry Orientation cannot ask what the operator intended or what should happen next.
- Question Surface Inventory cannot infer dispatch from free-text semantics.
- Answer Composition cannot create facts by presenting a supported answer.
- Diagnostics cannot silently mutate cluster truth by being recorded.

### Downstream constitutional assumptions

Downstream consumers assume:

- upstream records are already bounded artifacts, not licenses to revisit every original substrate;
- provenance and supporting evidence remain available for traceability;
- candidate and grammar observations are not truth promotion;
- exact identity matters more than semantic resemblance where the implementation says exact identity;
- local authority is not inherited from upstream owner authority;
- no mutation follows unless a separate mutation owner is invoked.

### Conditions under which upstream must be consulted again

Upstream must be consulted again when:

- the downstream artifact lacks required provenance or support;
- a question requires raw substrate parsing owned by an adapter;
- semantic, responsibility, capability, or ownership claims are being considered;
- invalid or missing source material prevents lawful record construction;
- exact evidence equality or relation-shape recurrence is insufficient for the proposed question;
- compatibility/public surface claims need confirmation from the owner implementation;
- presentation vocabulary is proposed as knowledge.

### Neighboring bridges

Observable bridges are artifact handoffs, not universal buses:

- agreement records into grammar observation;
- question-family identity into eligibility/dispatch metadata;
- pressure classifications into audit evidence;
- evaluated support/counterexamples into answer composition;
- lineage payloads into public compatibility objects;
- diagnostic declarations into shape-audit checks.

### Converging neighborhoods

Convergence occurs where multiple upstream owners emit records that one downstream owner can compare:

- documentation relations, repository facts, and relationship facts converge at Observation Agreement;
- candidate agreements converge at Grammar Observation by relation shape;
- pressure, responsibility, support, limitation, and lineage artifacts converge at answer/audit surfaces;
- diagnostic inventory declarations and implementation specs converge at Diagnostic Shape Audit.

### Diverging neighborhoods

Divergence occurs after a bounded artifact is emitted:

- repository artifact facts can support self-model alignment or agreement but do not become runtime artifacts;
- documentation structure can feed diagnostics, recurrence, membership, relationship evidence, or agreement without becoming prose truth;
- question-family metadata can support bounded dispatch or orientation but not arbitrary planning;
- candidate agreement can feed grammar observation or characterization but not architectural truth.

### Competing owners

Potentially competing owners remain separated by implementation boundary:

- Structure Observation owns shared read-only boundary, while adapters own parsing.
- Relationship Observation owns relationship facts but not graph reconciliation or repository traversal.
- Observation Agreement owns candidate agreement, not grammar, truth, or responsibility.
- Grammar Observation owns recurring shape observation, not lexicon or architecture.
- Runtime/execution owners remain separate from diagnostic, orientation, pressure, and answer surfaces.

### Explicit non-neighbors

The following are explicit non-neighbors for this survey target:

- planner;
- pipeline framework;
- universal grammar framework;
- message bus;
- orchestration layer;
- runtime execution surface;
- schema/JSON/CLI/event/ledger change surface;
- architectural-family promotion authority.

They are excluded because no implementation evidence reviewed grants them as touching owners of this neighborhood.

## Vertical topology

### Above: constitutional principles constraining the neighborhood

Above the neighborhood are constitutional principles rather than executable pipeline stages:

- repository authority wins over candidate vocabulary;
- lawful facts, permissions, prohibitions, identities, boundaries, and invariants constrain behavior;
- evidence insufficiency blocks commitment;
- diagnostic-only findings do not become organism truth;
- observations may be candidate interpretations without promotion;
- unknowns remain lawful when evidence is insufficient;
- presentation vocabulary is not automatically knowledge.

The constitutional layer constrains the neighborhood by prohibiting semantic promotion, mutation, behavior scripting, and ownership collapse without implementation evidence.

### Below: implementation artifacts realizing the neighborhood

Below the neighborhood are concrete implementation artifacts:

- dataclasses and boundary dictionaries;
- adapter methods using `ast.parse` or documentation parsers;
- record types such as `RepositoryArtifactFact`, `RelationshipFact`, `ObservationAgreementRecord`, `GrammarObservationRecord`, `BoundedWorkEligibilityResult`, and `InquiryOrientationView`;
- exact matching and recurrence algorithms;
- test suites covering structure observation, repository observation, observation agreement, grammar observation, bounded questions, inquiry orientation, diagnostic inventory, and diagnostic shape audit.

### Lawful unknowns in either direction

Above remains lawfully unknown whether the recurring pressure has earned a universal architectural family. It has not.

Below remains lawfully unknown whether every implementation surface touching this pressure has been exhaustively enumerated. The current survey reviewed the specified evidence and adjacent surfaces; it did not prove global repository exhaustiveness.

## Temporal topology

### Constitutional condition before transformation

Before transformation, the lawful condition is one of:

- substrate exists for an authorized adapter;
- upstream record grammar exists for an authorized consumer;
- preserved note or projected read model exists for orientation;
- exact question family exists for eligibility evaluation;
- evidence is absent, invalid, insufficient, or outside owner authority.

### Constitutional condition after transformation

After transformation, the lawful condition is a bounded artifact with preserved evidence and explicit authority boundary:

- structural fact;
- relationship fact;
- candidate agreement;
- grammar observation;
- eligibility/selection/dispatch record;
- orientation view;
- answer/audit/lineage payload.

The artifact may permit a downstream local question, but it does not carry upstream owner authority or architectural truth.

### Lawful transitions

Lawful transitions include:

- authorized substrate to adapter-owned structural records;
- supplied observation records to candidate agreement records;
- candidate agreement records to recurring-shape records;
- exact question-family identity to eligibility status and selected dispatch metadata;
- preserved note plus projected state to read-only orientation view;
- evaluated evidence to bounded answer composition;
- diagnostic declaration to shape-audit consistency check.

### Impossible transitions

Impossible transitions include:

- candidate agreement directly to architectural truth;
- grammar observation directly to responsibility family, semantic meaning, lexicon authority, or capability promotion;
- orientation lexical overlap directly to operator intent, command, requirement, recommendation, or next safe move;
- diagnostic recording directly to cluster mutation;
- presentation vocabulary directly to preserved knowledge;
- unsupported question family directly to executable work.

### Preserved continuity

Continuity is preserved through identity, provenance, support, recurrence evidence, boundary text, non-promotion markers, and compatibility shapes. The continuity is evidentiary, not ontological: downstream consumers can reconstruct why an artifact exists without acquiring authority to say it is architectural truth.

## Counterexamples

### Direct substrate consumption

Direct substrate consumption exists and is lawful at adapter/owner boundaries. Repository Artifact Observation consumes supplied Python text. Documentation Structure consumes Markdown. Relationship Observation consumes supplied Python text for import/definition adapters. These are lawful specializations, not contradictions, because they are upstream owners rather than downstream bypasses of Grammar Observation.

Classification: lawful specialization.

### Neighboring bypasses

Inquiry Orientation directly tokenizes `InquiryNoteRecord.raw_note` rather than requiring a separate note-grammar producer. It is constrained because the note remains preserved prose, not a fact, claim, command, or intent source.

Runtime/projector owners also consume events/state directly where they own that responsibility.

Classification: lawful specialization when owned locally; implementation exception only to the overstrong claim that all downstream consumers must consume immediately neighboring transformed grammar.

### Competing transformations

Relationship Observation emits `RelationshipFact` from both documentation metadata and Python syntax. It is not fully collapsed under Structure Observation in current evidence and therefore competes with any strict single hierarchy.

Classification: implementation exception to a single-owner hierarchy; not an architectural contradiction to bounded neighborhood characterization.

### Authority leakage

Observation Agreement and Grammar Observation explicitly prevent authority leakage through non-promotion boundaries and false fields for truth, promotion, mutation, grammar ownership, family recovery, and semantic interpretation. Diagnostic surfaces similarly separate record scope, event-ledger writes, and cluster mutation.

Classification: not observed as an implementation contradiction in reviewed evidence; leakage is an explicitly refused transition.

### Semantic promotion

Semantic promotion is explicitly refused by Observation Agreement, Grammar Observation, Inquiry Orientation, Relationship Observation, and Documentation Structure. Exact equality, recurring shape, and lexical overlap do not create meaning.

Classification: explicit refusal; any promotion would require a separate implemented authority.

### Ownership collapse

Ownership collapse is rejected by cross-neighborhood evidence: artifacts cross, owners do not. Structure Observation does not own adapter parsing; Relationship Observation does not own graph reconciliation; Observation Agreement does not own truth; Grammar Observation does not own responsibility; answer surfaces do not own evidence creation; runtime execution remains owner-specific.

Classification: explicit refusal; no reviewed implementation evidence requires collapse.

## Supported conclusions

1. The survey characterized more than one boundary. It recovered upstream, transformation, downstream, horizontal, vertical, temporal, integrity, counterexample, and neighboring-survey surfaces around the selected neighborhood.
2. The implementation repeatedly supports a bounded producer → bounded artifact → bounded consumer shape.
3. The shape is constitutional pressure and topology evidence, not a promoted universal architecture.
4. Upstream lawful substrate and adapter authority are local.
5. Transformations preserve identity, provenance, evidence, authority boundary, compatibility, and unknowns.
6. Transformations intentionally discard or refuse semantic meaning, ownership promotion, architectural truth, behavior, and mutation unless separately owned.
7. Downstream consumers ask newly legal questions only inside the grammar they receive.
8. Counterexamples are lawful and important: direct substrate consumption exists at adapter/owner boundaries.
9. The neighborhood can be reconstructed by future surveyors from the artifact handoffs, negative authority, counterexamples, and lawful unknowns listed here.

## Unsupported conclusions

1. Unsupported: the repository has proven a universal grammar-transformation architecture.
2. Unsupported: every competency must consume transformed grammar rather than direct substrate.
3. Unsupported: candidate agreement, relation-shape recurrence, or lexical overlap is semantic truth.
4. Unsupported: Observation Agreement or Grammar Observation owns responsibility recovery, family recovery, lexicon stabilization, or architectural promotion.
5. Unsupported: a planner, pipeline, framework, schema, CLI, JSON shape, event, or ledger surface should be added.
6. Unsupported: presentation vocabulary should be stabilized as repository knowledge.
7. Unsupported: diagnostic or audit findings silently mutate cluster truth.
