# Operator emission surface classification and split recovery 001

This recovery report is repository testimony from implementation investigation, not independent constitutional authority. It applies the governing Book grammar recovered through PRs 1890 and 1891: View formation is not View emission; emission is not the emitted assertion; assertion is not warrant; warrant is not lawful reliance; lawful reliance is not consumer uptake.

## Recovered minimum topology

The examined family currently compresses eight distinct assertion kinds:

1. observation-record visibility;
2. attributed-testimony visibility, currently only partial/implicit and not a separate standing emission;
3. normalized or fact-shaped artifact preservation;
4. projected support;
5. bounded Fact standing, absent as a proven public View here;
6. projection-local current selection;
7. present-facing applicability, absent as a proven public View here;
8. selection/support explanation, distinct from Fact-warrant or applicability explanation.

The smallest complete surface family needed for the recovered evidence is:

- projected observation-record inventory diagnostic;
- projected fact-support inventory diagnostic;
- projected-support diagnostic for a subject and predicate;
- current-selection diagnostic for a subject and predicate;
- selection-explanation diagnostic;
- separate standing-bearing Fact View, currently absent;
- separate current-applicability View, currently absent;
- verification/capability-verification standing surfaces where verification-specific implementation supplies a warrant.

## First implemented split

The first coherent split is the `--current-facts` arity boundary. The no-argument path is now explicitly a projected fact-support inventory diagnostic. The `SUBJECT PREDICATE` path is now explicitly a current-selection diagnostic. They shared a flag name but emitted materially different assertions and objects of reliance. This split does not decompose `State` and does not change fact production, admission, expiry, conflict, confidence, or verification algorithms.

## Absences and blockers

No examined implementation proves a true standing-bearing FactView. No examined implementation proves a true present-facing current-applicability View. Remaining work is blocked where `State.facts`, `State.fact_supports`, and current-fact getters compress artifact preservation, support projection, representative selection, and standing language without a separate warrant-producing owner.
