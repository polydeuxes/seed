## Scope and method

This is a report-only canonical cross-examination of Seed's bootstrap-capability, external-grammar, and structured-communication model against the current canonical Book after PR 1903. It does not amend canonical chapters, the concordance, unresolved questions, production code, tests, CLI behavior, persistence, projections, or selectors.

Implementation is used only as witness to present behavior. Current implementation names do not establish constitutional authority.

## Executive finding

The Book already supports the central negative boundaries:

```text
mechanism observed != capability established
capability established != currently reachable
currently reachable != authorized
authorized != selected
selected != realized
external grammar != constitutional grammar
manual claim != observed behavior
View or rendering != stronger standing
communication standing != receipt, understanding, reliance, or authority grant
```

The candidate bootstrap model is broadly compatible with current Book grammar, but not yet canonical as a named bootstrap doctrine. The Book supports capability as bounded standing; external grammar as attributed testimony; and communication as bounded handoff preserving evidence, limits, Unknowns, conflicts, authority scope, and stopping conditions. The Book does not clearly resolve the first-realization problem: what Seed must already possess before it can examine any environment-supplied candidate at all.

Therefore the lawful conclusion is:

- Seed does not constitutionally require one privileged bootstrap mechanism on current Book evidence.
- Seed does not constitutionally require a shell on current Book evidence.
- The environment may supply candidate mechanisms or material.
- Operator-supplied artifacts or testimony can provide first candidate material, but not established capability by identity.
- Manuals and specifications can be external grammar testimony, not behavioral capability proof by themselves.
- Behavioral examination is necessary for verified capability standing, scoped to the claim being verified; one test does not yield complete competence.
- A Bash dungeon is not constitutionally necessary, but can be useful as optional bounded behavioral testimony or safe curriculum if separately authorized.
- Prometheus is best treated as one current pre-recovered implementation capability/adaptor witness, not the constitutional sensory model.
- Seed does not need prose generation for constitutionally sufficient assistance requests if a structured representation preserves the necessary request standing and limits.
- Natural-language rendering can remain an optional View, operator-owned rendering, or later external representation capability.

## Minimum constitutional bootstrap

### What Seed must possess constitutionally

Current Book grammar warrants a minimal constitutional bootstrap in terms of responsibilities and preserved distinctions, not a single primitive operational mechanism. Seed must be able, at the constitutional level, to preserve and distinguish:

- testimony from established fact;
- evidence from admission and reliance;
- movement from constraint;
- capability from mechanism;
- reachability from authority;
- authority from selection;
- selection from realization;
- external grammar from Seed-native constitutional grammar;
- representation from meaning and standing;
- refusal, Unknown, conflict, insufficiency, and stopping from failure;
- communication standing from operator-facing prose or action permission.

This is constitutional grammar. It does not prove that Seed already has a shell, HTTP client, JSON parser, provider adapter, universal tool system, or prose generator.

### What may remain environment-supplied

Current Book grammar permits different environments to expose different initial candidates without changing Seed's constitutional grammar, as long as the crossing preserves source, scope, provenance or provenance Unknown, authority limits, uncertainty, refusal conditions, and standing. Candidate supplied boundaries may include:

- shell process;
- filesystem access;
- HTTP client;
- language runtime;
- local RPC;
- restricted host API;
- operator-mediated handoff;
- pretranslated material;
- manuals, specifications, examples, or help output.

The Book does not require one universal invocation form. The narrower lawful requirement is a bounded handoff whose input, output, authority, scope, provenance, representation grammar, result standing, Unknowns, and refusal/non-performance conditions can be preserved.

### The unresolved first-realization edge

The Book does not fully answer what must exist before Seed can examine its first environment-supplied capability. It permits operator expression and external material as testimony after bounded interpretation, and it permits external grammar to become addressable without assimilation. It also permits capability candidates from catalog declarations and behavioral observations. But it does not yet specify whether the very first realization boundary is:

- Seed-owned constitutional road;
- environment-supplied mechanism;
- operator-mediated handoff;
- pretranslated testimony;
- or an Unknown boundary outside Seed's current competence.

The safest classification is **Needs clarification**. Bootstrap capability is best understood as a combination: a constitutional road for preserving and examining candidate standing, plus one or more realization boundaries that may be supplied by the environment or operator. It is not an innate mechanism by identity.

## Shell and bounded process composition

The prior characterization of bounded process composition can be kept only in narrow standing:

```text
one bounded process request
→ environment-local realization
→ stdout material
→ stderr material
→ exit standing
```

This is not canonical proof of Bash competence. It is compatible with the Book as one possible mechanism realization, one externally supplied operational grammar, and one handoff boundary. It may support bounded capability standing only after evidence preserves invocation grammar, process boundary, input, output, exit standing, authority, scope, representation limits, and behavior.

Direct answer: **Seed does not constitutionally require a shell** on current Book evidence.

If a shell is supplied, the narrowest lawful standing is:

- testimony that a candidate environment-local process mechanism may exist;
- possible external grammar material, such as shell syntax, help output, or manual claims;
- possible reachability evidence if an authorized discovery or bounded test supports it;
- possible behavioral evidence for scoped requests.

It does not establish:

- Bash competence;
- installed command meaning;
- host authority;
- authority to run commands;
- semantic truth of exit zero;
- understanding of stdout grammar;
- general process composition capability;
- future selection or realization.

## External grammar, manuals, guides, and examples

The Book already governs external grammar strongly. External material may become addressable, attributed, translated, cited, compared, or consumed within a bounded scope without becoming constitutional grammar. A lawful movement can be:

```text
external material received
→ representation recognized
→ declared grammar examined
→ candidate grammar standing
→ behavioral examination when authorized
→ bounded capability standing
```

Direct answer: **A manual cannot establish behavioral capability standing by itself.**

What a manual or specification can lawfully establish, when provenance and scope are preserved:

- declared syntax;
- declared vocabulary;
- declared representation;
- declared behavior;
- declared limitations;
- candidate examples;
- possible examination questions;
- candidate expectations for behavioral tests;
- source-attributed claims about authority, version, or compatibility.

A small orientation guide has no stronger constitutional standing by identity. It can still be useful as an external witness that helps navigate vocabulary, select manual sections for examination, or suggest bounded behavioral questions. It does not authorize behavior or establish truth. Its provenance, source role, scope, and authority must remain visible.

Examples and help output also remain external material. An example succeeding can support a scoped behavior claim, but cannot establish complete grammar competence. Help output can declare locally exposed vocabulary, but not prove behavior, authority, or global availability.

## Behavioral examination

The Book supports behavioral examination as the road from declaration to verified bounded capability standing, but only with preserved constraints:

```text
manual claim
→ candidate behavioral expectation
→ authorized bounded experiment
→ result testimony
→ comparison
→ supported, contradicted, Unknown, conflict, or scoped capability standing
```

Behavioral examination contributes observed support, contradiction, dependency findings, representation findings, reachability findings, authority gaps, environment-local limits, and test-construction uncertainty. It is not execution authority by identity.

A behavioral failure may implicate grammar, mechanism, representation, authority, scope, environment, input construction, dependency, time, or provider condition. One failure is not capability absence. One success is not complete competence.

## Bash dungeon

A Bash dungeon or similar exercise environment is not constitutionally necessary on current Book evidence. It can remain useful as:

- optional behavioral testimony;
- safe experimental opportunity;
- curriculum;
- sandbox;
- supplied test environment;
- environment-local variation witness.

The distinction is:

```text
manual → what Bash declares
sandbox/dungeon → bounded opportunities to examine behavior
```

If Seed can consume Bash manuals and local help, a dungeon is still not required by the Book. But if the goal is behaviorally supported competence rather than declared grammar standing, some behavioral examination remains necessary; the dungeon is one optional way to supply it.

## Prometheus as fossilized recovered capability

The current Prometheus path is best characterized as one concrete already-recovered sensory capability/adaptor witness, not the constitutional sensing process. Present implementation assumes much of the recovery work in advance:

- HTTP-like acquisition;
- JSON-like representation;
- Prometheus envelope grammar;
- query vocabulary;
- metric semantics;
- predicate translation;
- acquisition and filter scope;
- treatment of no returned observations.

The Book supports treating the adapter as current implementation testimony, a supplied implementation capability, a reusable local witness, or a test fixture. It does not support treating it as Seed-native knowledge by identity.

Preserved distinctions:

```text
pre-recovered adapter != Seed-native knowledge
adapter present != adapter selected
adapter selected != authority granted
adapter returned no observations != condition absent
acquisition failed != negative observation
malformed material != no observations
```

## Inquiry-targeted acquisition

The Book supports constraining sensing by one bounded evidence demand while allowing provider acquisition to return one signal, many signals, a full snapshot, or provider-determined material, as long as the scope and admission distinctions remain visible:

```text
Seed question != provider query
provider query != acquisition scope
acquisition scope != returned material
returned material != selected material
selected material != interpreted testimony
```

Provider limitations may require broad acquisition without making the original constitutional question broad. Full acquisition is not full semantic need. Material acquired is not all material admitted.

## Structured communication and enum language

The Book does not require natural-language generation for constitutionally sufficient assistance requests. Communication and handoff require preservation of identity, subject, evidence/support basis, authority scope, uncertainty, Unknowns, conflicts, refusal conditions, neighboring-inquiry boundaries, and lawful stopping limits. These can be preserved in structured fields before any prose renderer exists.

A possible minimum structured request language may carry:

- movement state;
- boundary;
- standing;
- attempted capability;
- material received;
- required evidence;
- required authority;
- requested assistance kind;
- scope;
- Unknowns;
- conflicts;
- next lawful movement.

These are not a prescribed schema. They are examples of preserved constitutional content.

A lawful relation is:

```text
structured enum/state request
→ operator-facing View
→ optional natural-language renderer
```

The renderer does not create authority, scope, standing, Unknowns, or requested movement. It only renders an already bounded request.

### Enum cautions

Enums and structured standing can serve as Seed's first communication language if they preserve semantic minimums:

- the subject and boundary of the request;
- current standing and its evidence;
- Unknown, conflict, refusal, unsupported, blocked, unrecognized, and not-yet-examined as distinguishable states;
- authority requested versus authority already held;
- scope and applicability;
- references to source material, evidence, and preserved limits;
- extensibility for novel Unknowns and unrecognized distinctions.

The danger is ontology freeze:

```text
enum convenience → premature compression → lost distinction → false standing
```

Closed enums are useful for bounded state vocabularies, but they must not become a universal ontology. Some fields must remain open vocabulary, reference-bearing, or extensible when the Book has not fixed the distinction.

## Natural language as optional representation capability

English grammar, dictionaries, thesauri, style guides, and examples can be external grammar materials supporting a later movement from structured standing to English expression. Natural-language competence is not shown by current Book evidence to be required constitutional competence. It may be:

- optional representation capability;
- operator-owned rendering;
- external grammar consumed later;
- a View over structured standing;
- or Unknown in environments that require prose-only operator channels.

Preserved distinctions:

```text
English fluency != constitutional understanding
well-formed prose != warranted standing
natural-language request != authority grant
translation != semantic strengthening
```

## Failed sensing and assistance-request states

Failed sensing should not collapse into one `probe_failed` state. Depending on boundary and evidence, the following may be testimony, constraint results, gap findings, diagnostic states, assistance-request inputs, or different subjects at different boundaries:

- acquisition failed;
- material received;
- representation unrecognized;
- representation recognized but malformed;
- provider grammar unrecognized;
- provider envelope rejected;
- sample decoding failed;
- semantic interpretation unavailable;
- Observation emission refused;
- Observation emitted but insufficient.

The Book's negative-finding, Unknown, refusal, non-performance, evidence, and communication grammar requires these distinctions to remain visible when they condition lawful reliance or assistance.

## Direct answers

### Does Seed constitutionally require one privileged bootstrap mechanism?

No on current Book evidence. The Book requires preserved constitutional distinctions and lawful crossings, not one privileged mechanism.

### Does Seed constitutionally require a shell?

No on current Book evidence. A shell may be an environment-supplied candidate, but it is not a constitutional requirement.

### Can an operator-supplied artifact or testimony provide the first candidate capability?

Yes, as candidate testimony or external material, if source, scope, provenance/Unknown, representation, and authority limits are preserved. It does not establish capability by identity.

### Is bootstrap capability better understood as a mechanism, a standing, a constitutional road, or a combination?

A combination. The constitutional part is a road for preserving and examining candidate material and standing. The realization part may be a supplied mechanism or operator-mediated boundary. Capability itself is bounded standing, not the mechanism.

### Can manuals and specifications be treated as external grammar testimony?

Yes. They can provide attributed declared grammar, vocabulary, behavior, limitations, and examination expectations.

### Can manuals establish behavioral capability without examination?

No. They can establish declared claims as testimony, not verified behavior.

### Is some form of behavioral examination necessary for verified capability standing?

Yes, for verified capability standing scoped to behavioral claims. The required examination depends on the claim, authority, environment, and risk. One test may support one bounded fragment; it does not establish complete competence.

### Is a Bash dungeon constitutionally necessary?

No on current Book evidence.

### Can a Bash dungeon remain useful even when manuals are available?

Yes. It can provide safe, bounded behavioral opportunities and curriculum-style evidence distinct from declared manual grammar.

### Is Prometheus currently one pre-recovered implementation capability rather than the constitutional sensory model?

Yes. Current Prometheus behavior is implementation testimony for a specific adapter path, not proof that Seed constitutionally knows Prometheus, HTTP, JSON, provider predicates, or acquisition semantics natively.

### Does Seed need prose to request operator assistance?

No on current Book evidence, provided the structured request preserves identity, standing, scope, evidence, Unknowns, conflicts, authority need, and lawful next movement.

### Can structured enums and references constitute a minimum assistance-request language?

Yes, if they remain sufficiently expressive and extensible. Closed enums must not compress novel Unknowns or constitutional distinctions.

### Can natural-language rendering remain an optional View or external capability?

Yes. Rendering can be presentation standing over a structured request or result. It does not strengthen authority or warrant.

### What dangers arise from treating enums as a closed universal ontology?

Closed enums can freeze ontology prematurely, collapse Unknown into unsupported or failure, hide conflicts, erase provenance, confuse authority requested with authority held, and make presentation convenience look like constitutional law.

### Which parts of the proposed model are already supported by the Book?

Supported: preserving testimony/evidence/standing/warrant; distinguishing movement from constraint; distinguishing capability from mechanism; distinguishing reachability from authority; preserving Unknown/refusal; examining attributed external grammar; treating capability as bounded standing; communicating bounded requests and views without strengthening result standing.

### Which parts require canonical clarification before implementation removal begins?

Clarification is needed for the first-realization edge, the minimum structured assistance-request vocabulary, the threshold for verified capability standing from behavioral examination, the acquisition/admission distinction for broad provider snapshots, and failed-sensing outcome taxonomy.

## Supported, contradicted, incomplete, and Unknown portions of the bootstrap hypothesis

Supported:

- Minimal Seed preserves testimony, evidence, standing, warrant, Unknown, refusal, and authority limits.
- Capability is distinct from mechanism, reachability, authority, selection, and realization.
- External grammar can be examined without assimilation.
- Bounded handoff can carry requests, references, and insufficiency.
- Different environments may supply different candidates, if boundaries are preserved.

Compatible but not canonical adoption:

- Environment may expose shell, filesystem, HTTP, language runtime, RPC, restricted API, operator handoff, manuals, or none.
- Recovery may proceed through declared grammar, behavioral support, reachability, authority, representation compatibility, and bounded applicability.
- Structured enum/state communication may precede natural language.
- A dungeon/sandbox can help behavioral learning.

Needs clarification:

- What must exist before the first candidate capability can be examined.
- Whether operator-mediated handoff can serve as the first realization boundary in all environments.
- What minimum structured request fields are constitutionally sufficient.
- How much behavioral support establishes what level of capability standing.
- How failed sensing states should be compactly classified without creating a universal probe ontology.

Contradicted:

- Manual alone establishes behavioral capability.
- Shell is constitutionally required.
- Prometheus adapter is the constitutional sensory model.
- Exit zero proves semantic success.
- Structured rendering or prose creates authority.

Implementation-only:

- Current hard-coded Prometheus query/envelope/predicate translation assumptions.
- Current JSON observation import/export grammar.
- Current concrete observation source classes and collection service mechanics.
- Current CLI filters and full/partial provider sweep behavior.

Unknown:

- Whether current Book can name a specific first bootstrap road without amendment.
- Whether any existing Seed-local producer can form and emit operator assistance requests from structured insufficiency standing.
- Whether prose-only operator channels would make natural-language rendering operationally required in a specific environment while remaining non-constitutional generally.

## Three canonical questions before removing hard-coded assumptions

1. What is the Book-authorized first-realization boundary by which Seed may receive or examine an initial candidate capability when no shell, HTTP client, provider adapter, prose generator, or universal tool system is assumed?
2. What minimum preserved fields make a structured assistance request constitutionally sufficient, and which fields must remain open, reference-bearing, or extensible to avoid enum-based ontology freeze?
3. What scoped behavioral-examination warrant is sufficient to move from declared external grammar testimony to bounded capability standing without promoting a manual, example, adapter, or provider response into Seed-native knowledge?
