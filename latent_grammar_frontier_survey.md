# Latent Grammar Frontier Survey

## Bounded question

```text
When a single external grammar
lawfully admits multiple
supported interpretations,

what constitutional capability,
if any,

is responsible for preserving,
bounding,
or resolving those
interpretations?
```

Repository authority wins.

## Answer

Recurring repository evidence **does not yet support a new constitutional capability named or equivalent to `latent grammar`**.

The pressure is real, but the currently supported anatomy is smaller and already distributed across previously recovered constitutional responsibilities:

```text
external representation is admitted only as bounded evidence;
constitutional translation preserves warranted content under authority boundary;
ambiguity may be represented as candidate interpretations, equal-rank support,
or typed Unknown;
resolution occurs only when additional evidence creates an unambiguous supported transition;
otherwise the lawful result is preservation, boundary, or stop.
```

The repository therefore supports an adjacent constitutional **pressure**:

```text
lawful interpretation-set preservation before or during translation
```

but not an independently recovered constitutional capability, owner, participant, or implementation family.

The bounded answer is:

```text
Previously recovered constitutional anatomy explains the pressure:
external-grammar structural recovery, constitutional translation under warrant,
inquiry orientation, bounded lawful reliance, ambiguity/unknown preservation,
evidence admission, and stop discipline.

No recurring implementation evidence reviewed here requires expanding that anatomy
into a new capability.
```

## Evidence reviewed

This survey reviewed repository-local implementation and constitutional artifacts involving external grammar, translation, evidence admission, ambiguity, unknowns, inquiry orientation, cross-domain translation, and characterization methodology.

Primary implementation evidence:

- `seed_runtime/inquiry_orientation.py` preserves raw operator prose as a read-only inquiry note and explicitly refuses fact, intent, ownership, command, authorization, plan, recommendation, or semantic-interpretation authority.
- `seed_runtime/state.py` preserves equal-rank entity-type ambiguity rather than choosing arbitrarily, and refuses to select a best support when the strongest support is tied.
- `seed_runtime/state_summary_views.py` surfaces storage-topology ambiguities as projection-only interpretation pressure with candidate interpretations, observable evidence, materiality, and explicit negative authority.
- `seed_runtime/input_inspector.py` performs bounded format detection with `unknown` as a lawful result rather than forcing file content into a known grammar.
- `seed_runtime/typed_unknowns.py` preserves typed Unknown records without resolving them, while projecting the existing public unknown shape.

Primary constitutional characterization evidence:

- `constitution.md`
- `external_grammar_structural_recovery_characterization.md`
- `constitutional_translation_invariant_preservation_characterization.md`
- related recovery and inquiry-characterization artifacts discovered through repository search and used only where they reinforced recurring implementation-backed patterns.

## Recurring implementation evidence

### 1. Operator prose is preserved without semantic promotion

`seed_runtime/inquiry_orientation.py` defines inquiry orientation as a minimized read-only probe. It stores operator prose outside the event ledger and states that rendering must not mutate state, append events, call providers, execute tools, create facts, goals, tool needs, decisions, proposals, or plans.

The authority boundary is explicit: the note is preserved operator prose, not fact, claim, goal, tool need, requirement, capability, decision, proposal, plan, authorization, command, or runtime instruction. Matches are deterministic lexical overlaps only and do not assert importance, ownership, intent, concern, recommended action, or next safe move.

This directly answers part of the motivating pressure. A single external prose grammar may carry many possible communicative interpretations, but the recovered implementation does not choose among them. It preserves the prose as evidence for the probe, admits only lexical overlap as related-material support, and bounds the result as orientation.

Supported distinction:

```text
external operator grammar
!= operator intent
!= command
!= repository fact
!= architecture
```

### 2. Ambiguity can be intentionally represented as candidate interpretations

`seed_runtime/state_summary_views.py` builds `storage_topology_ambiguities` from already-projected filesystem measurements, mount visibility groups, and shared-storage candidates. The implementation explicitly says these ambiguities do not create facts, ownership, storage identity, topology truth, issues, or operator clarification requests.

The same surface records:

- reasons;
- candidate interpretations;
- observable evidence;
- materiality;
- materiality basis;
- boundary.

This is strong implementation evidence that the repository can preserve multiple plausible interpretations as visibility without resolving them into truth. However, this evidence is domain-local and projection-only. It does not prove a universal grammar capability.

Supported distinction:

```text
candidate interpretation set
!= resolved fact
!= ownership
!= identity
!= topology truth
```

### 3. Equal-rank support is preserved rather than collapsed

`seed_runtime/state.py` returns the strongest supported entity types while preserving equal-rank ambiguity. If multiple entity-type assertions have the same strongest rank, the implementation returns all tied types. Separately, `_select_unambiguous_best_support(...)` returns `None` when the top support is tied.

This shows a recurring constitutional behavior outside operator prose: when support is lawful but not uniquely resolving, the system preserves non-resolution instead of selecting one candidate for convenience.

Supported distinction:

```text
supported candidate
!= uniquely selected answer
```

### 4. Unknown format is a lawful result of bounded inspection

`seed_runtime/input_inspector.py` inspects file bytes without executing content or resolving embedded references. It detects JSON, YAML, INI, or `unknown`. If content cannot be safely classified, it emits `unknown_format` rather than forcing a grammar.

This matters because the frontier pressure asks whether a single external grammar may admit multiple interpretations. Current implementation evidence first requires lawful admission: the system may stop at unknown format before interpretation even begins.

Supported distinction:

```text
available external artifact
!= admitted grammar
!= interpreted constitutional content
```

### 5. Typed Unknowns preserve unresolved investigation state

`seed_runtime/typed_unknowns.py` creates immutable `TypedUnknownRecord` values from unresolved investigation state. The record has an unknown type, area, and reason, and projects to the existing public unknowns shape.

This is evidence that unresolved interpretation pressure can be preserved without resolution and without inventing new public truth.

Supported distinction:

```text
preserved Unknown
!= resolved interpretation
!= hidden failure
```

## Recurring constitutional evidence

### External grammar structural recovery

`external_grammar_structural_recovery_characterization.md` already recovers a broad distinction between external representations and Seed-owned bounded structures. It reviews operator prose, dpkg status text, Prometheus JSON, Python source/AST, documentation metadata, and repository artifact extraction.

Its smallest recurring answer is that external representation is accepted, family-local parsing or matching occurs, a Seed-owned bounded structure is emitted, representation-specific details may be discarded or retained only as metadata/support, and negative authority boundaries prevent the representation from becoming architecture.

This covers much of the latent-grammar pressure without requiring a new owner. The repository already has a recurring constitutional discipline for external grammar, but it is family-local, not universal.

### Constitutional translation invariant

`constitutional_translation_invariant_preservation_characterization.md` states that faithful constitutional translation does not preserve grammar, representation, meaning, or organization. It preserves bounded warranted content under provenance, role, authority limit, negative authority, preserved unknowns, locality/dimensions where supported, and confidence or evidence-strength calibration where supported.

This directly constrains the present investigation. If multiple interpretations exist before translation, translation is not authorized to preserve all external grammar or assumed meaning. It may preserve only warranted content and preserved unknowns under boundary.

### Constitution-level discipline

`constitution.md` states that Null persists until evidence justifies transition, observation precedes inquiry, evidence moves Null, repository authority wins, no owner exists without recurring implementation evidence, inquiry is bounded work, authority must be explicit, stop is positive, frontiers preserve pressure recovery cannot yet relieve, and presentation vocabulary is not authority.

This forbids converting the phrase `latent grammar` into a capability merely because the phrase organizes pressure. The constitutional result must remain unknown or frontier-only unless recurring evidence demonstrates ownership.

## Recovered recurring pressures

The reviewed evidence supports these pressures:

1. **External grammar is not self-authorizing.** Raw prose, provider syntax, source syntax, file bytes, and projection labels require bounded admission before constitutional reliance.
2. **Multiple interpretations can be visible without being true.** Storage-topology ambiguity records preserve candidate interpretations while refusing facts, ownership, and identity.
3. **Support can be non-selective.** Equal-rank entity-type support and tied fact support produce ambiguity or no unambiguous selection.
4. **Translation needs boundary more than meaning theory.** Constitutional translation preserves warrant under boundary, not inferred semantic fullness.
5. **Unknown is a valid terminal or intermediate state.** Unknown format and typed Unknown records preserve unresolved states without promotion.
6. **Resolution requires additional evidence.** No reviewed evidence supports resolving ambiguity by elegance, operator intuition, vocabulary, analogy, or human-like interpretation.
7. **Frontier pressure remains.** The repository has recurring pressure around interpretation-set preservation, but no recovered universal implementation owner.

## Unsupported candidate ideas

The following candidate ideas are **not supported** by recurring repository evidence reviewed here:

- A new constitutional participant named `latent grammar`.
- A universal grammar engine that owns interpretation sets across prose, audio, intonation, roleplay, sarcasm, fiction framing, provider syntax, source syntax, and operational artifacts.
- A natural-language semantics layer responsible for pronoun attachment, implication, omitted premise recovery, sarcasm detection, or fiction/reality disambiguation.
- A requirement that constitutional translation select exactly one interpretation whenever several lawful interpretations exist.
- A requirement that constitutional translation preserve every possible interpretation of an external grammar.
- A shared implementation owner for all ambiguity, unknown, evidence-admission, inquiry-orientation, and provider-translation behavior.
- Treating operator communicative pressure as repository truth without implementation evidence.
- Treating presentation vocabulary as preserved constitutional knowledge.

## Confidence

Confidence: **medium-high** for the negative answer that no new capability is currently supported.

Reasons:

- Multiple implementation paths already preserve ambiguity, unknowns, or bounded external representations without needing a new owner.
- Existing constitutional artifacts explicitly warn against promoting grammar, representation, meaning, presentation vocabulary, or frontier pressure into knowledge.
- The reviewed implementation evidence is recurring across several domains.

Confidence is not high because:

- This is a frontier survey, not a full implementation recovery campaign.
- Some motivating examples, especially audio/intonation/sarcasm/roleplay, have no reviewed implementation-local surfaces.
- The repository may contain additional narrow artifacts not exhaustively characterized here.
