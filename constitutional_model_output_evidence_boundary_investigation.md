# Constitutional Model Output / Evidence Boundary Investigation

Repository authority wins.

## Bounded question

Across recurring repository evidence, when Seed receives, produces, or uses a model-generated or automated interpretation artifact, what prevents that output from being treated as evidence, observation, truth, authority, proof, warrant, or implementation pressure without separately preserved source material, provenance, claim form, support path, confidence limits, authority boundary, Unknowns, and lawful stop?

This is exactly one bounded constitutional investigation. It focuses only on:

```text
Model Output != Evidence
```

It does not implement anything, add a diagnostic surface, create a model-output schema, create a trust/scoring engine, fetch public-world evidence, evaluate any public-world technology, perform legal analysis of any real case, or promote model-output vocabulary into repository ontology.

## Model-output analysis

### 1. What repository evidence already means by model output, if anything

Repository evidence does not currently support a mature constitutional or implementation-local definition of `model output` as a repository concept. The app-visible source-navigation query for model-output vocabulary was unknown/unsupported, and the reasoning path exposed no derivation evidence for `model output evidence`.

The lawful recovered meaning is therefore smaller than the prompt vocabulary: model output is best treated here as an **external or generated interpretation artifact candidate** until separately preserved source material, provenance, claim form, support path, authority boundary, confidence limits, Unknowns, and lawful stop are available.

This investigation does not create a model-output ontology. It uses `model output`, `generated output`, `classifier output`, `parser output`, `OCR output`, `transcript output`, `embedding match`, `summary`, `candidate interpretation`, and `model confidence` as bounded investigation vocabulary only.

## Evidence analysis

### 2. What repository evidence already means by evidence

Repository evidence supports evidence as preserved, provenance-bearing support material or event/state material that can be inspected through bounded support paths. Evidence is not mere fluency, repetition, selected text, generated explanation, source volume, confidence number, or authority label.

Projection authority evidence distinguishes fact-backed, derived, diagnostic, and presentation surfaces. Observed/current facts, evidence, observations, and event-derived state collections may be repository knowledge when projected from ledger events, while derived rows such as fact support, source navigation, evidence graph, and confidence rows remain traceable transformations rather than independent truth.

Observation evidence also matters: observation is a source-attributed report of what was observed, bounded by provenance, source, scope, vantage point, time, confidence, subject-as-reported, and promotion decisions. Observation may support evidence formation, but observation is not automatically truth or fact.

## Required distinctions

### 3. Model output / source material distinction

Source material is the preserved or inspectable material from which a claim can be supported: a source record, event, file, observation, evidence item, fact, source fact, or bounded repository artifact with provenance and authority boundary. Model output is a generated or interpreted artifact about source material. Without preserved source material and a support path, it can at most identify a possible place to look.

### 4. Model output / observation distinction

Observation is source-attributed acquisition: something reported by a source from a vantage point at a time, with scope and provenance. Model output may report what a model generated, but that is only an observation of the model-output artifact itself, not an observation of the underlying world, source event, legal authority, implementation behavior, or repository fact.

### 5. Model output / fact-support distinction

Fact support preserves supporting fact IDs, evidence IDs, provenance, source types, confidence inputs, and grouping dimensions for a current claim. Model output that says a fact is supported is not fact support. It becomes useful to fact support only if it points to separately preserved supporting facts/evidence and the support path remains inspectable.

### 6. Generated summary / underlying-evidence distinction

A summary is presentation or interpretation. It may compress, select, omit, reorder, or misstate source material. The underlying evidence is the preserved material and support path that can be independently inspected. A generated summary may help navigate evidence, but it cannot replace the evidence it summarizes.

### 7. OCR-transcript / original-source distinction

OCR text and transcript extraction are derived representations. They may be useful access layers, but they are not identical to the original image, audio, video, document, event, or speaker act. Before OCR/transcript output contributes to evidence, Seed would need to preserve the original source or source reference, extraction method, provenance, time, scope, confidence/error limits where available, and Unknowns around illegible, inaudible, omitted, or ambiguous material.

### 8. Parser-extraction / source-truth distinction

Parser extraction is a deterministic or automated transformation of source material into structured fields. It can preserve an extracted claim candidate, but parser success is not source truth. The parser may encode matching rules, truncation, stale grammar, field loss, or interpretation errors. Lawful reliance requires the source material, parser boundary, extracted claim form, support path, and confidence/limitation evidence to remain inspectable.

### 9. Classifier-label / claim-form or source-role distinction

A classifier label is an assigned category. A claim form states what kind of assertion is being made and what support/authority is required. A source role states how a source may be used within a bounded provenance/authority discipline. A classifier may suggest a claim form or source role candidate, but it does not grant that role. Source-role and claim-form promotion require repository evidence, authority boundary, and Unknown preservation.

### 10. Embedding-match / support distinction

An embedding match or semantic-similarity result is retrieval/navigation pressure. It says that a representation was near another representation under an embedding method. It does not prove relevance, support, independence, contradiction, authority, truth, or warrant. Support requires a preserved source path and a claim-specific relation between the source material and the claim.

### 11. Model-confidence / evidence-strength distinction

Model confidence is a property or output of a model or extraction process. Evidence strength is a repository-supported assessment bounded by source material, provenance, support path, authority, corroboration/independence where relevant, contradiction, freshness, and claim form. Model confidence may be recorded as metadata about the generated artifact, but it cannot substitute for evidence strength.

### 12. Generated-citation or selected-passage / source-authority distinction

A generated citation or selected passage is a pointer or presentation candidate. It is not source authority unless the cited source actually exists, the passage is preserved or verifiable, the citation correctly identifies the source, and the source has authority for the claim form. Generated citations are especially bounded: they may help locate source material, but they do not create authority.

### 13. Seed-report / primary-evidence distinction

Another Seed report is repository evidence of a prior bounded investigation, audit, characterization, or conclusion. It is not automatically primary evidence for the external world, implementation behavior, legal authority, or source event it discusses. It can support continuity, boundary discipline, and repository-history claims; stronger claims require the underlying primary evidence or implementation evidence to remain available.

### 14. Model-derived absence / evidence-of-absence distinction

Model-derived absence is a generated failure-to-find or no-match interpretation. The absence boundary already prevents failed search, not found, missing support, or non-exposure from becoming falsehood, disproof, global absence, no authority, no event, no harm, or no implementation issue. Model-derived absence is even weaker unless the reviewed corpus, query, model/extractor behavior, time window, coverage, authority boundary, and Unknowns are preserved. It may support only a bounded statement such as: no match was generated by this model/process over this scoped material under these limits.

## Lawful uses of model output

### 15. What can model output lawfully support?

Current repository evidence supports only bounded, non-promotional uses:

- investigation pressure;
- candidate source navigation;
- candidate interpretation to be checked against source material;
- candidate claim-form or source-role suggestion;
- candidate extraction requiring source verification;
- summary for orientation when underlying evidence remains preserved;
- triage or queueing pressure when not treated as proof;
- metadata about the generated artifact itself, such as that a model produced a label, confidence, selected passage, summary, or no-match result.

Model output can lawfully support a claim about itself: `this model/process produced this output under these conditions`, if the output, process identity, input/context, time, and provenance are preserved. It does not thereby support the truth of the generated content.

## Stronger-claim readiness analysis

### 16. What must be preserved before model output may contribute to evidence, warrant, or lawful reliance?

Before model output can contribute to evidence, warrant, or lawful reliance, repository evidence indicates at least the following must be preserved:

- underlying source material or a durable source reference;
- provenance of the source material;
- provenance of the generated artifact;
- input/context given to the model or automated process, to the extent needed for auditability;
- model/process identity and version where relevant;
- extraction, selection, classification, matching, or summarization method boundary;
- claim form;
- source role and authority boundary;
- support path from source material to claim;
- confidence limits and evidence-strength limits;
- known errors, omissions, truncations, transformations, and ambiguity;
- Unknowns, including unavailable source material, missing provenance, unsupported authority, unclear claim form, uncertain extraction, and incomplete coverage;
- lawful stop / non-promotion boundary.

Preserving these conditions still would not automatically make the output evidence. It would allow the output to be considered as part of a bounded support path whose authority remains proportional to the preserved evidence.

## Provenance / source-material / support-path analysis

The repository's recurring support pattern is provenance-bounded support. Source navigation uses projected source facts/support and refuses to inspect files or claim semantic relevance. Reasoning path and selection path expose lineage and Unknowns but do not record facts or mutate cluster state. Projection authority evidence distinguishes primary fact/evidence rows, deterministic derived rows, diagnostics, and presentation.

Therefore model output cannot bypass the chain. The lawful path is not:

```text
model says X -> evidence that X
```

The bounded path is:

```text
source material + provenance + claim form + method boundary + support path + authority limit + Unknowns -> possible bounded support
```

Model output may appear inside that path only as a candidate interpretation or processing artifact, not as the source material itself.

## Authority-boundary analysis

Authority does not arise from generated fluency, source volume, selected passages, or model confidence. Repository warrant evidence requires evidence-bound lawful reliance under preserved authority boundary. Legal-authority boundary evidence similarly refuses to treat analysis artifacts, source-of-law lineage, or non-authoritative material as adjudicative authority or constitutional warrant.

For model output, the authority boundary is strict: a model may have processing utility, but it has no automatic authority to create evidence, legal authority, source authority, truth, proof, warrant, implementation ownership, or implementation pressure.

## Confidence / claim-strength analysis

Claim strength must track evidence strength. Model confidence may be high while source support is absent, provenance is missing, authority is unsupported, or the claim form is wrong. Conversely, a low model confidence value may still point to source material that can be independently inspected. Repository evidence supports confidence/evidence-strength calibration, not confidence-number promotion.

The strongest lawful claim about model confidence is usually about the generated artifact: the model/process reported confidence `C`. A stronger claim about the world or repository requires evidence outside the model confidence score.

## Unknown preservation

Unknown preservation is the constitutional brake. When model output lacks source material, provenance, claim form, support path, authority boundary, confidence limits, or coverage, the result must remain Unknown, unsupported, incomplete, candidate, or investigation pressure. Typed Unknown evidence supports preserving unresolved conditions without resolving them through implication.

Relevant Unknowns include:

- missing original source;
- unclear source role;
- missing source provenance;
- missing model/process provenance;
- unverified citation;
- unverified OCR/transcript/parser extraction;
- semantic match without support relation;
- model confidence without evidence-strength basis;
- summary without underlying evidence;
- no-match output without corpus/coverage authority;
- another Seed report without primary evidence;
- implementation pressure without concrete failing command, behavior, visibility gap, owner, or test evidence.

## Lawful stop

### 17. What lawful stop condition prevents model output from becoming evidence, truth, proof, authority, or implementation pressure?

The stop condition is:

> When generated or automated output lacks separately preserved source material, provenance, claim form, support path, confidence/evidence-strength limits, authority boundary, Unknown preservation, and a repository-supported reliance boundary, stop at candidate interpretation or investigation pressure. Do not promote the output into evidence, observation of the underlying matter, source fact, truth, proof, authority, warrant, lawful reliance, or implementation pressure.

This stop applies equally to LLM summaries, classifier labels, embedding matches, OCR/transcript/parser outputs, generated explanations/citations, model confidence scores, selected passages, another Seed report, automated clustering/entity resolution, and source-role suggestions.

## Supported conclusions

1. Repository evidence does not yet define a mature model-output evidence model.
2. Model output is not evidence by default.
3. Generated output can be a processing artifact, orientation aid, or candidate interpretation.
4. Generated output can support claims about the generated artifact itself if its provenance is preserved.
5. Generated summaries do not replace underlying evidence.
6. OCR and transcript text are derived representations, not identical to original sources or events.
7. Parser extraction is not source truth.
8. Classifier labels do not create claim form or source role authority.
9. Embedding similarity is navigation pressure, not support.
10. Model confidence is not evidence strength.
11. Generated citations and selected passages are not source authority.
12. Another Seed report is repository evidence of prior bounded work, not automatically primary evidence.
13. Model-derived absence is not evidence of absence except as a bounded no-match/process result under preserved corpus and coverage limits.
14. Warrant requires evidence-bound lawful reliance under preserved authority boundary.
15. Unknown preservation and lawful stop prevent model output from becoming truth, proof, authority, warrant, or implementation pressure.
