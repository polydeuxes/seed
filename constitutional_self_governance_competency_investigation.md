# Constitutional Self-Governance Competency Investigation

## Selected constitutional question

Can Seed eventually become constitutionally competent to determine whether one of his own competencies may lawfully proceed, or are current Competency Interrogations only external Codex development methodology?

Bounded answer: repository evidence supports a recurring constitutional pressure around **Competency Self-Governance**, but it does **not** support a runtime, framework, planner, review engine, or generic competency engine. The supported pressure is narrower:

> When an organism is about to let a bounded capability proceed, it must be able to determine whether the capability has identity, evidence, jurisdiction, authority, refusal boundaries, unknown preservation, and lawful stop conditions sufficient for the proposed movement.

Today, full Competency Interrogation remains Codex development methodology. However, several responsibilities repeatedly exercised by the interrogations are already implementation-backed as Seed physiological capabilities or neighboring surfaces: question eligibility, evidence interpretation, responsibility evaluation, observation agreement, orientation, pressure visibility, and authority-bounded evaluation.

Repository authority wins.

## Recurring constitutional pressures

### 1. Lawful continuation requires evidence-gated movement

Multiple interrogations ask what evidence permits movement and what evidence causes lawful stop. This recurs in implemented surfaces:

- question-family eligibility permits bounded work only for exact implementation-backed families;
- inquiry orientation permits related-material visibility but not authorization;
- observation agreement permits candidate agreement but not truth promotion;
- pressure audit permits pressure visibility but not planning;
- authority slices permit reachability classification but not provider acquisition or observation execution.

This supports the pressure: **a capability may not proceed merely because a question exists or a pressure is visible.**

### 2. Competency identity must be bounded before trust

The interrogations repeatedly separate worker, competency, and bounded responsibility. Implementation-backed examples do the same: Documentation Structure is a Markdown structure observer, not a prose interpreter; Repository Artifact Observation is a supplied-text structural adapter, not a repository scanner; Question Surface Inventory is an inventory/eligibility surface, not a router or recommender.

This supports the pressure: **Seed must know what capability is being invoked before deciding whether it may proceed.**

### 3. Authority must be explicit and negative authority must be preserved

The recurring interrogation question “what would be an unlawful expansion?” is implementation-backed by many surfaces that explicitly deny ownership of adjacent responsibilities. Observation Agreement refuses promotion to truth. Inquiry Orientation refuses authorization and next-move authority. Question Surface Inventory refuses unknown-family dispatch and diagnostic-only inquiry answers.

This supports the pressure: **competency self-governance is at least partly the ability to say what the competency must not do.**

### 4. Unknown preservation is a lawful result

Unknowns are preserved in question-family definitions for unknown families, inquiry orientation when no deterministic related material is found, authority slices when available authority is unknown, pressure audit counts unknown diagnostic shape rows, and responsibility evaluation investigations classify incomplete or blocked states without promotion.

This supports the pressure: **not knowing whether a competency may proceed must remain a lawful stop, not a failure to be hidden.**

### 5. Pressure is not command authority

Pressure Audit and pressure visibility investigations repeatedly preserve pressure as visibility. A pressure may recommend an inspection command, but it does not plan, mutate, record, or execute. This matters for self-governance because a competency can feel pressure to proceed while still lacking authority.

This supports the pressure: **self-governance must distinguish “there is pressure” from “this capability is authorized to act.”**

## External methodology

The following responsibilities are currently external Codex development methodology and are not implementation-backed as Seed runtime physiology:

1. **Repository evolution judgment.** Deciding what the repository should recover next, which slices to implement, or how to evolve architectural vocabulary remains external.
2. **Architectural recovery by reading many prose artifacts.** Codex can synthesize investigations across documents, but Seed does not have a generic implementation-backed architecture recovery engine.
3. **Competency Interrogation report writing.** Producing the full markdown interrogation matrix, with selected competency, neighboring boundaries, constitutional reliance, supported conclusions, and next work, remains external methodology.
4. **Slice planning and implementation review.** Selecting implementation slices, reviewing pull requests, deciding test adequacy for new features, and judging repository direction remain outside current Seed physiology.
5. **Promotion of methodology into repository authority.** The repository repeatedly cautions that presentation vocabulary and methodology prose do not become knowledge without implementation evidence.

These external responsibilities do not reject Competency Self-Governance as a pressure. They reject overclaiming that Seed already owns the full Codex interrogation process.

## Candidate internal physiology

The following responsibilities appear recoverable as internal constitutional physiology because implementation evidence already supports narrower versions of them:

1. **Question eligibility.** Exact question-family eligibility can determine whether bounded work may execute now, requires parameters, is diagnostic-only, or is not dispatchable.
2. **Evidence sufficiency classification.** Responsibility Evaluation investigations and authority slices support classifications such as supported, unsupported, ambiguous, incomplete, blocked, candidate, or outside authority.
3. **Authority-bounded reachability.** Service/container/listener authority surfaces evaluate whether required observations are reachable under declared authority without acquiring providers or executing observations.
4. **Orientation without command.** Inquiry Orientation can expose related material and uncertainty without turning prose into command, plan, ownership, or recommendation.
5. **Pressure visibility without planning.** Pressure Audit can rank pressure from existing surfaces without selecting work.
6. **Agreement without promotion.** Observation Agreement can compare independent observation streams and preserve candidate agreement without truth promotion.
7. **Lawful stop behavior.** Multiple surfaces return unknown, empty, blocked, diagnostic-only, or not-dispatchable states rather than forcing movement.

The smallest truthful internal candidate is not “Seed interrogates competencies like Codex.” It is:

> Seed may eventually possess bounded physiological checks that decide whether a named capability is eligible, sufficiently evidenced, within authority, and required to stop before acting.

## Preconditions for eventual constitutional self-governance

If constitutional self-governance is eventually recoverable, the following implementation evidence would need to exist first. These are not designs; they are evidence requirements recovered from existing patterns.

1. **Stable capability identity evidence.** A self-governance surface would need implementation-backed capability identity, not prose labels alone.
2. **Question eligibility evidence.** It would need exact eligibility rules comparable to `bounded_work_eligibility_for_question_family`, including non-dispatchable and diagnostic-only stops.
3. **Evidence sufficiency evidence.** It would need local sufficiency criteria for the specific capability being governed, not generic confidence language.
4. **Authority and negative authority evidence.** It would need declared authority boundaries and explicit refusals comparable to read-only, no-provider-acquisition, no-execution, no-ledger-write, no-mutation, and no-promotion declarations.
5. **Unknown preservation evidence.** It would need explicit unknown/blockage statuses that are rendered and tested.
6. **Boundary to behavior.** It would need to stop before behavior selection unless behavior authority is separately implemented.
7. **Diagnostic inventory/shape audit evidence if exposed operationally.** Any new diagnostic/CLI/recordable surface would need inventory registration, shape audit specs, and tests under the repository visibility contract.
8. **No silent truth promotion.** Diagnostic or candidate governance findings would need non-cluster-mutating boundaries and likely diagnostic-run-scoped recording if recording existed.

## Neighboring competencies

### Question Eligibility

Question Eligibility answers: “Is this exact question family permitted to dispatch bounded work now?” It is narrower than Competency Self-Governance. It can gate invocation, but it does not fully evaluate whether the selected competency has sufficient evidence, constitutional reliance, neighboring boundaries, or lawful termination.

### Evidence Interpretation

Evidence Interpretation answers: “What does this evidence mean inside provenance, shape, authority, and non-promotion boundaries?” It is necessary but insufficient. Competency Self-Governance would also ask whether the interpreted evidence is enough for this capability to proceed.

### Responsibility Evaluation

Responsibility Evaluation is the closest neighbor. It answers: “Is this responsibility-like claim supported, unsupported, ambiguous, incomplete, blocked, candidate, or outside authority?” Competency Self-Governance would consume this pattern but apply it reflexively to a capability’s permission to continue. It must not collapse into generic assignment, promotion, execution, or review.

### Observation Agreement

Observation Agreement answers: “Do independent supplied observation streams agree on an exact evidence pattern?” It can strengthen evidence for self-governance, but candidate agreement remains candidate-only and not architectural truth.

### Orientation

Orientation answers: “What related material is visible for this bounded concern?” It helps locate relevant context but explicitly does not authorize action, infer intent, select next work, or create a plan.

### Pressure Visibility

Pressure Visibility answers: “What pressure is visible from existing surfaces?” It can expose why governance is needed, but pressure is not authority and cannot decide that a competency may proceed.

## Counterexamples and rejecting evidence

The repository contains strong counterexamples against treating Competency Self-Governance as already implemented or against internalizing all of Competency Interrogation.

1. **Full interrogation is prose methodology.** Competency Interrogations are markdown reports produced by Codex. There is no implementation surface that generates or consumes the full interrogation matrix as runtime physiology.
2. **Repository evolution remains external.** Selecting next investigations, implementation slices, and architectural recovery paths is not implemented as Seed behavior.
3. **Architectural recovery is not generic runtime inference.** Observation Agreement and Structure Observation deliberately refuse semantic inference, responsibility recovery, grammar ownership, lexicon ownership, and truth promotion.
4. **Question inventory is not a router or recommender.** It maps exact question families to existing surfaces but does not semantically interpret arbitrary prose or decide work priority.
5. **Pressure Audit is not a planner.** It ranks pressure from existing signals and recommends inspection commands, but it does not plan, record, mutate, or execute.
6. **Orientation is not authorization.** Inquiry notes do not become facts, commands, plans, ownership, recommendations, or next moves.
7. **Diagnostic findings are not cluster truth.** Repository instructions and diagnostic surfaces preserve record/mutation boundaries; self-governance cannot silently promote diagnostic-only findings into organism truth.

These counterexamples do not prevent meaningful internal constitutional self-governance. They constrain it to bounded physiological checks and reject a full internal Codex replacement.

## Lawful unknowns

Repository evidence does **not** currently support answering in full:

```text
How does Seed determine that he is constitutionally competent to act?
```

What remains unknown:

1. There is no implementation-backed general self-governance surface for competencies.
2. There is no capability registry with tested constitutional authority, refusal behavior, evidence sufficiency, and lawful stop for every competency.
3. There is no implementation-backed mechanism that binds a proposed action to a named competency and asks whether that competency may proceed.
4. There is no tested handoff from self-governance result to behavior selection.
5. There is no evidence that Seed can conduct architectural recovery, repository evolution, PR review, or methodology reconciliation internally.
6. There is no proof that “Competency Self-Governance” is a stable repository term; it is only a candidate pressure name for this investigation.

Lawful answer: Seed does not yet determine constitutional competence to act in a general way. The repository does support several local preconditions and recurring pressures that make eventual bounded self-governance plausible.

## Supported conclusions

1. A recurring constitutional pressure exists: a capability must determine whether it is lawfully allowed to proceed under evidence, authority, jurisdiction, unknown, and stop constraints.
2. Current Competency Interrogations are external Codex methodology, but they repeatedly interrogate questions that are also constitutional and partially implementation-backed.
3. The repository supports internal physiological neighbors: question eligibility, evidence sufficiency, authority reachability, orientation, pressure visibility, observation agreement, unknown preservation, and lawful stops.
4. The smallest recoverable internal physiology is not a competency engine; it is bounded eligibility/sufficiency/authority/stop checking for named capabilities.
5. Counterexamples strongly reject internalizing repository evolution, architectural synthesis, slice planning, PR evaluation, and full report production as Seed runtime physiology.
