# Architectural Pressure Backlog

This is a pressure ledger, not a roadmap and not a TODO list.

Each entry preserves future pressure without requiring immediate action. Repository authority always wins: an item only advances when implementation evidence shows that the pressure is real, belongs to a recoverable neighborhood, and has earned the smallest lawful next step.

## Maturity vocabulary

- **Observation:** Preserved pressure or external observation; not yet Seed implementation pressure.
- **Investigation:** Worth evidence-gathering before an audit or slice.
- **Audit:** Worth inventorying against repository evidence and boundaries.
- **Slice:** Already points toward bounded implementation work, but still requires normal slice discipline.
- **Ready:** Has enough implementation-backed evidence and boundaries to begin the next bounded change.

## Guiding principle

Treat every entry as pressure, not obligation.

The repository should determine:

- whether the pressure is real;
- whether it belongs to an existing neighborhood;
- whether it deserves investigation;
- whether it matures into an audit;
- whether it ultimately earns a slice.

No item should advance merely because it appears on this list.
