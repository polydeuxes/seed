# Competency Self-Governance Characterization

Repository authority wins.

## Bounded question

Does the repository support one recurring constitutional competency, or are multiple neighboring districts being mistaken for one?

This investigation does not recover implementation, recommend a slice, recommend a campaign, introduce governance, introduce planners, introduce schedulers, introduce orchestration, or infer runtime decision machinery.

## Answer

The repository supports **one recurring constitutional competency at the characterization level**, but not under the name "self-governance" as governance, runtime autonomy, or implementation ownership.

The actually recurring competency is:

> **Lawful bounded-continuation judgment**: the constitutional competency that determines whether a bounded inquiry, capability characterization, responsibility evaluation, question family, or pressure claim has enough identity, evidence, authority, boundary discipline, negative-authority refusal, unknown preservation, reliance limits, stop conditions, and calibrated confidence to make another lawful move.

It is one district only at the constitutional-characterization level. It is several neighboring districts at the implementation level. Repository evidence does not support a single owner, API, schema, runtime object, lifecycle, registry, planner, scheduler, or governance mechanism.

When Seed asks, "May I lawfully continue?", the recurring constitutional competency actually answering is not governance over Seed. It is the evidence-bound judgment of whether the present bounded subject has any repository-supported lawful movement remaining without crossing an evidence, authority, boundary, unknown, reliance, negative-authority, or confidence limit.

## Identity

The recurring constitutional competency is **lawful bounded-continuation judgment**.

It answers:

> Does this bounded subject still have another repository-supported lawful move, or has it reached the point where continuation would cross constitutional limits?

It is not the same as governance. It is not a planner. It is not a scheduler. It is not orchestration. It is not implementation selection. It is the recurring constitutional inquiry that evaluates whether movement remains lawful for the present bounded subject.

## Inputs

The competency consumes only recovered constitutional conditions:

1. **Bounded subject identity**: a capability, inquiry, responsibility candidate, question family, pressure claim, audit family, or characterization target whose smallest supported identity can be stated.
2. **Repository evidence**: implementation evidence, tests, diagnostics, inventory rows, completed investigations, audits, bridge inspections, completion audits, or methodology evidence that can distinguish observed, absent, preserved, movement-permitting, and stop-causing facts.
3. **Granted authority boundary**: what the subject may lawfully answer, consume, expose, rely on, or hand off.
4. **Negative authority**: what the subject appears to authorize but does not actually authorize.
5. **Boundary and neighborhood distinction**: separation from adjacent owners, families, public outputs, presentation surfaces, compatibility wrappers, report composition, implementation owners, and neighboring districts.
6. **Unknown preservation**: unsupported conclusions, unresolved states, insufficiency, or unsupported domains preserved rather than inferred through.
7. **Reliance limits**: what Seed may rely on and what it must not rely on.
8. **Stop conditions**: evidence boundary reached, authority boundary reached, neighboring responsibility separated enough, negative authority named, unknown preserved, reliance bounded, confidence calibrated, or no further same-boundary movement remaining.
9. **Confidence calibration**: certainty bounded by recurring evidence rather than by elegance, recurrence alone, or operator pressure.

## Outputs

The competency produces constitutional consequences, not implementation consequences:

1. **Continue within the bounded subject** when repository evidence still supports another lawful same-boundary move.
2. **Stop / lawful termination** when continuing would cross evidence, authority, boundary, unknown, reliance, negative-authority, or confidence limits.
3. **Insufficient evidence / unsupported** when evidence cannot support the proposed conclusion or movement.
4. **Adjacent pressure preserved** when pressure is real but belongs to a neighboring family or district.
5. **Unknowns preserved** when conclusions remain unsupported.
6. **Authority boundary stated** so visibility, agreement, verification, inventory presence, pressure, or output appearance are not mistaken for permission.
7. **Confidence stated** so bounded findings are not presented as complete recovery.

The output is a bounded constitutional characterization or evaluation. It is not a plan, slice, campaign, runtime decision, cluster mutation, governance act, or implementation recovery.

## Negative authority

The recurring competency explicitly refuses to:

- create authority from visibility, verification, inventory presence, agreement, pressure, output form, recommendation-like phrasing, recurrence, or usefulness;
- convert pressure visibility into implementation recovery;
- convert observation agreement into truth;
- convert typed unknowns into facts, architecture, plans, authority, or actions;
- promote presentation vocabulary into repository knowledge without implementation evidence;
- infer through missing evidence;
- absorb adjacent districts into the current subject;
- assign ownership, select slices, rank campaigns, or recommend implementation;
- mutate cluster truth or event-ledger meaning;
- become a universal question abstraction, universal inquiry ontology, governance framework, planner, scheduler, orchestration engine, runtime self-awareness mechanism, capability registry, lifecycle object, or autonomous work-selection system.

## Neighborhood relationship

### Supported answer

The competency is **one constitutional district and several neighboring implementation/evidence districts**.

At the constitutional-characterization level, the same question recurs across many artifacts: whether a bounded subject may lawfully continue. The repeated conditions are stable enough to support one district: bounded identity, evidence, authority, boundary discipline, negative authority, unknown preservation, reliance, stop, and confidence.

At the implementation level, the repository evidence splits this recurrence across neighboring districts:

- Competency Interrogation Grammar supplies the constitutional grammar.
- Constitutional Inquiry State supplies bounded constitutional exhaustion and lawful termination.
- Question Eligibility / Bounded Ask supplies exact family admission and bounded dispatch eligibility.
- Responsibility Evaluation supplies read-only evaluation criteria for proposed responsibility recovery.
- Observation Agreement supplies agreement-without-truth-promotion discipline.
- Typed Unknown Preservation supplies unresolved-state preservation.
- Pressure Visibility supplies read-only pressure classification and authority-limit exposure.
- Family Completion Audits supply natural stop, adjacent-pressure preservation, and no-same-family-continuation discipline.

Therefore, the repository supports one recurring constitutional inquiry, but not one recovered implementation family or universal mechanism.

### Rejected answer: one implemented self-governance district

Unsupported. The repository repeatedly refuses a single owner, API, schema, runtime object, lifecycle, registry, diagnostic surface, self-observer, planner, scheduler, routing layer, or governance mechanism for this recurrence.

### Rejected answer: merely projection

Unsupported as a complete answer. The recurrence is not only projection from adjacent vocabulary, because multiple independent districts repeatedly recover the same constitutional question and stop logic. However, the name **Competency Self-Governance** is partly projective and potentially misleading if read as governance or runtime self-rule. The evidence supports lawful bounded-continuation judgment, not governance.

## Confidence

**High confidence** that a recurring constitutional competency exists: the evidence repeatedly asks whether a bounded subject has another lawful move or must stop.

**High confidence** that the supported competency is not governance, planning, scheduling, orchestration, implementation recovery, or runtime autonomy.

**Medium confidence** in the label **lawful bounded-continuation judgment**, because it is a characterization over recurring evidence rather than a stabilized repository term.

**Low confidence** that `Competency Self-Governance` is a safe final name, because it risks implying governance and self-rule that the repository evidence explicitly does not grant.

**Low confidence** that the competency has any implementation ownership today.

When Seed asks,

what is actually answering is:

Competency self-governance characterization complete.
