### Projection evidence

Question Projection and Capability Projection provide the immediate reason the pipeline stops before additional ownership recovery.

Question Projection is characterized as a deterministic selection-adjacent projection over the already recovered `BoundedConstitutionalQuestion`. Every projected question field is already owned by the bounded-question artifact, including inquiry provenance, bounded question text or identity, constitutional intent, scope status, uncertainty notes, and read-only / non-mutating flags. The characterization refuses mutation, independent persistence, constitutional authority, view selection, composition, runtime reasoning, planning, and evidence recovery.

Capability Projection is characterized as a deterministic projection over `ConstitutionalReadModelContract`, `ReadModelViewRegistration`, and immutable constitutional view artifacts. Its fields are already owned by contracts, registrations, and the Process, Governance, and Fidelity views. Independent capability ownership would duplicate constitutional contribution knowledge and risk stale parallel authority.

Together, these projection characterizations show that implementation pressure adjacent to Selection can collapse into deterministic projections when all projected fields already have owners. At that point, recovery does not continue merely because another representation is useful.

### Irreducible-boundary evidence

The Constitutional View Selection Characterization provides the complementary boundary condition. Selection is not itself reducible to a projection because the selected view set is not already owned upstream. Selection is the first boundary that produces a new implementation-local selected-view artifact from the deterministic inputs.

The same characterization refuses further decomposition inside Selection. No reviewed evidence supports a separate matching algorithm owner, ranking owner, heuristic owner, semantic-reasoning owner, planner, orchestrator, evidence-discovery owner, or authority owner. Selection remains irreducible because the evidence supports exactly one new implementation-local artifact, `SelectedConstitutionalViews`, and no lawful intermediate ownership boundary.

This matters for Lawful Architectural Stopping because the discipline is not "always stop at projections." The evidence supports a more bounded rule:

1. recover ownership when repository evidence exposes a new implementation-local owner;
2. characterize projections when representation pressure is fully derivable from existing owners;
3. stop architectural decomposition when newly discovered work is only deterministic production, deterministic projection, public surface, diagnostic surface, or integration wiring.

## Recurrence analysis

### 1. Has this stopping pattern appeared elsewhere in repository recovery?

Yes.

The Constitutional Recurrence Survey: Implementation Completion compared two independent implementation recovery campaigns: Inquiry Orientation and Reasoning Path. Those campaigns had different domains, different local code shapes, different numbers of slices, and different recovered boundaries. Both nevertheless reached the same completion pattern:

- a compressed public-facing builder or composer initially owned multiple local responsibilities;
- each slice recovered exactly one adjacent implementation-local owner;
- recovered owners were narrow producers, helpers, or artifacts;
- public compatibility shapes remained unchanged;
- recovery stopped when no additional implementation-local producer, artifact, helper, or consumer remained exposed;
- any further slice would merely wrap terminal orchestration or compatibility handoff.

That survey answered directly that both campaigns terminated for the same constitutional reason: repository evidence no longer exposed another implementation-local ownership boundary. It further classified the stopping condition as constitutional recurrence, not an implementation-specific outcome.

### 2. Is the stopping pattern specific to the constitutional pipeline?

No.

The pipeline demonstrates the pattern in a new topology, but the repository already contains recurrence outside the pipeline. Inquiry Orientation and Reasoning Path independently reached lawful completion when remaining work was terminal sequencing, payload assembly, or compatibility handoff rather than unrecovered ownership compression.

The pipeline's remaining categories are not identical vocabulary to those campaigns. They are deterministic producers, deterministic projections, public surfaces, diagnostic surfaces, and integration wiring. But the constitutional structure is the same: after ownership and projection boundaries are explicit, remaining implementation can realize, expose, and wire the architecture without creating another constitutional owner.

### 3. Does repository evidence support Lawful Architectural Stopping as a recurring constitutional recovery discipline?

Yes.

Repository evidence supports Lawful Architectural Stopping as the architectural-level form of the previously surveyed completion condition. The recurrence survey established that implementation-local ownership recovery stops when no additional owner is exposed and only terminal orchestration or compatibility handoff remains. The pipeline audit extends the same discipline to an architecture that includes deterministic projections and public/diagnostic/integration exposure: once all remaining pressure classifies outside ownership recovery, architectural decomposition lawfully stops and implementation realization may begin.

This does not authorize shortcut stopping. Future campaigns may rely on the discipline only as an evidence requirement. They must show from repository evidence that remaining work is not ownership recovery.
