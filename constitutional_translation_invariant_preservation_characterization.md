# Constitutional Translation Invariant Preservation Characterization

## Bounded question

```text
According to recurring repository evidence,
what is preserved during faithful constitutional translation?

Does constitutional translation preserve grammar,
representation,
meaning,
organization,
or some other recurring constitutional invariant?
```

Repository authority wins.

## Answer

Recurring repository evidence does **not** support grammar, representation, meaning, or organization as the thing preserved by faithful constitutional translation.

The recurring preserved invariant is narrower and more constitutional:

```text
bounded warranted content under provenance, role, authority limit,
negative authority, preserved unknowns, locality/dimensions where supported,
and confidence or evidence-strength calibration where supported.
```

In the implementation-facing observation paths, this often appears as selected subject identity, selected predicate/relation/value, evidence or metadata, dimensions/locality, confidence, and non-promotion boundaries. In broader constitutional investigations, the same movement appears as evidence-bound content that remains lawful for a downstream consumer while refusing unsupported promotion.

Faithful constitutional translation therefore preserves **constitutional warrant under boundary**, not external grammar, not exact representation, not assumed meaning, and not organization.

## Recurring repository evidence

### Provider language translation

`provider_language_translation_investigation.md` characterizes provider-language translation as bounded work that selects provider vocabulary, validates or decodes provider grammar, shapes identity, assigns Seed predicates, and constructs canonical observations or another bounded record. It explicitly identifies `Observation` construction as the point where provider-specific samples, rows, status fields, AST nodes, git porcelain lines, local files, and platform APIs become Seed observation grammar.

The same investigation shows that provider grammars differ materially:

- dpkg status text becomes `PackageRecord` and then package observations.
- Python source text/AST becomes `RelationshipFact` and then relationship observations.
- Prometheus HTTP API JSON vector samples are decoded inline into observations.
- systemd JSON rows become provider-shaped dictionaries and then observations.
- local-host values mix direct observation construction with delegated dpkg/systemd translation.
- git command output becomes `RepositoryObservation`, not canonical `Observation`.

This is strong evidence against grammar or representation as the preserved invariant. The provider forms change, the intermediate artifacts vary, and some paths do not enter canonical observation grammar at all. What recurs is selected bounded content with evidence, identity, predicate/value assignment where applicable, metadata, dimensions, and authority boundary.

### Provider translation responsibilities

`provider_language_translation_responsibility_characterization.md` reaches the same responsibility shape from implementation evidence. It states that Provider Language Translation owns turning provider-local language into Seed observation-facing language through bounded vocabulary selection, grammar validation/rejection, decoding, identity/evidence preservation, predicate assignment, and observation construction where paths are compressed.

It also explicitly refuses a stronger ownership recovery. Predicate assignment and identity shaping recur, but they are provider-specific and not uniformly separated across providers. This preserves the bounded invariant while refusing an unsupported organizational conclusion.

### Constitutional translation invariant investigation

`constitutional_translation_invariant_investigation.md` already answers the broader constitutional residue question. It identifies the surviving substance as evidence-bound, authority-reduced, provenance-preserving, role-bounded constitutional content that remains lawful for its downstream consumer and refuses unsupported promotion.

It also states that lawful translation may change external wording, provider syntax, parser-specific grammar, field names, record shape, presentation format, private handoff artifact, diagnostic/table rendering, JSON arrangement where compatible, candidate phrasing, local implementation vocabulary, compression level, and whether the output is canonical `Observation` or another bounded record.

The same investigation states that lawful translation must not change the evidence actually supporting the claim, selected subject or local identity at supported resolution, artifact role, provenance/support, authority boundary, negative authority, unresolved unknowns, confidence/evidence-strength calibration where carried, locality/dimensions where supported, and the content's lawful status as presentation/orientation/support/admission/diagnostic/observation/evidence/fact/projection/candidate/verification/readiness/agreement/inquiry result.

This is the strongest repository authority for the present bounded answer.

### External grammar structural recovery

`external_grammar_structural_recovery_characterization.md` reviews implementation-backed paths and finds multiple external grammars feeding bounded Seed structures: raw operator prose, dpkg status text, Prometheus JSON vectors, git stdout, Python source/AST, repository artifact text, systemd JSON stdout, and local-host stdlib/file values.

Its recurring movement is not grammar preservation. Dpkg text becomes `PackageRecord`; Python AST becomes `RelationshipFact` or `RepositoryArtifactFact`; Prometheus remains compressed until `Observation`; systemd remains provider-shaped dictionaries until `Observation`; git becomes `RepositoryObservation` outside canonical observation ingestion.

This supports the same invariant: external grammar may be structurally recovered enough to select warranted identity, relation/value, evidence, boundary metadata, dimensions, and downstream lawful role. It does not prove preservation of external grammar.

### External artifact intake versus provider language translation

`external_artifact_intake_vs_provider_language_translation_investigation.md` distinguishes artifact intake from provider translation. Intake handles safe acquisition, bounded decoding, language detection, and parser choice or fixed parser routing. Provider translation handles provider-local grammar and conversion into bounded Seed outputs.

The distinction matters for this characterization because external artifacts do not become constitutional content merely by being acquired or decoded. Faithful translation requires a supported boundary where selected content can lawfully move forward with provenance and authority limits. Acquisition form and decoded representation are therefore not the preserved invariant.

### Constitutional grammar and recovery discipline

The constitutional grammar recovery artifacts treat grammar as a recovery discipline and local inquiry tool, not as a universal constitutional language. They repeatedly preserve evidence boundaries, bounded questions, unsupported conclusions, and lawful stops. That recurrence supports constitutional warrant under boundary, while the explicit warnings in repository instructions forbid promoting presentation vocabulary or local grammar into preserved knowledge without implementation evidence.

### Plato's Cave constitutional translation

`plato_cave_constitutional_translation_characterization.md` characterizes translation across cave/projection boundaries as constitutional discipline rather than ontology recovery. Its relevance here is negative and bounded: projection, presentation, and external-facing forms are not automatically the preserved thing. A lawful translation must keep the support boundary visible and must not promote shadows, presentation, or orientation into more authority than evidence grants.

### Provider-local observations entering constitutional reasoning

Provider-local observations become usable by constitutional reasoning only after bounded translation or admission: provider samples, rows, stdout, local files, and source syntax must be selected, decoded or rejected, shaped into identity/relation/value evidence, and carried with provenance, confidence/metadata/dimensions where supported, and authority limits. Repeatedly, malformed, unsupported, overbroad, or non-selected provider content is not preserved.

## Recovered translation movement

The recurring lawful movement is:

```text
external/provider/presentation form
-> bounded vocabulary or surface selection
-> grammar validation, decoding, or rejection
-> supported identity/relation/value/inquiry-condition selection
-> provenance/evidence/metadata/dimension preservation where supported
-> authority reduction and negative-authority preservation
-> downstream bounded output or lawful stop
```

This movement is not always separated into the same number of implementation steps. Some providers have intermediate records. Some providers are compressed. Some outputs are canonical observations. Some are non-observation diagnostic or repository-state records. Some material lawfully stops as orientation, candidate, unknown, or unsupported content.

## Candidate preserved invariants

### Grammar

**Unsupported as the preserved invariant.**

Provider grammar repeatedly changes. Dpkg text, Python AST, Prometheus JSON vectors, systemd JSON rows, git porcelain output, local files/APIs, Markdown/YAML/Python artifact text, and operator prose do not share a preserved grammar. Current evidence supports grammar validation and bounded grammar recovery, not grammar preservation.

### Representation

**Unsupported as the preserved invariant.**

Representations vary across `PackageRecord`, `RelationshipFact`, `RepositoryArtifactFact`, `RepositoryObservation`, canonical `Observation`, provider dictionaries, diagnostic rows, orientation notes, eligibility artifacts, and constitutional investigation artifacts. Some paths have no intermediate record. Some paths do not enter canonical observation grammar. Representation can change when constitutional warrant remains stable.

### Meaning

**Unsupported as a claimed preserved invariant in this characterization.**

Repository evidence supports selected warranted content and authority boundary. It does not require defining meaning, importing semantic theory, or assuming meaning preservation. Where repository artifacts use terms such as value, relation, claim, orientation, or fact, those terms are bounded by implementation and authority evidence. This characterization therefore does not claim that "meaning" is preserved.

### Organization

**Unsupported as the preserved invariant.**

Implementation organization varies. Translation may be separated into parser/record/emitter boundaries or compressed inside a provider. The evidence supports provider-local responsibilities and constitutional boundary discipline, not a required shared translation organization or universal owner.

### Constitutional warrant under boundary

**Supported as the recurring preserved invariant.**

Across the reviewed evidence, faithful constitutional translation preserves the warranted constitutional content that remains lawful for the downstream consumer:

- selected subject or local identity at the supported resolution;
- selected relation, predicate, value, or inquiry condition;
- provenance, support, source, or evidence metadata;
- role of the artifact/content in its owning family;
- authority boundary and granted authority;
- negative authority and refusal of unsupported promotion;
- unknowns not resolved by evidence;
- confidence or evidence-strength calibration where the surface carries it;
- locality or dimensions where those are part of the support;
- read-only/non-mutating boundaries where declared.

## What is permitted to change

Faithful constitutional translation may change:

- external grammar;
- provider syntax;
- parser-specific fields;
- provider-native representation;
- intermediate record shape;
- output format;
- rendering or table/JSON arrangement where compatibility permits;
- private handoff artifact;
- local implementation vocabulary;
- compression level between decoding and output construction;
- whether the lawful output is an `Observation`, a non-observation record, a diagnostic surface, an orientation artifact, an eligibility artifact, a characterization, or a lawful stop.

## What must remain stable

Faithful constitutional translation must keep stable:

- the evidence that actually supports the translated content;
- the selected identity/relation/value/inquiry condition at the supported resolution;
- provenance/support sufficient for downstream lawful reliance;
- the content's role and owning boundary;
- authority limits and negative authority;
- preserved unknowns;
- confidence/evidence-strength limits where applicable;
- locality/dimensions where applicable;
- read-only/non-mutating status where applicable;
- refusal to promote presentation, external wording, provider excess, candidate phrasing, or malformed/unsupported material into repository truth.

## Unsupported candidates

The reviewed evidence does not support any of the following conclusions:

- constitutional grammar is a universal language;
- all faithful translations preserve external grammar;
- all faithful translations preserve representation;
- all faithful translations preserve meaning as an unsupported extra claim;
- all faithful translations preserve organization;
- every provider should use intermediate records;
- every translated result should become a canonical `Observation`;
- every external artifact can contribute constitutional content;
- provider-local vocabulary should become repository knowledge;
- recurring presentation vocabulary is preserved knowledge without implementation evidence;
- a generic translation engine, semantic framework, or universal translation owner is currently required.

## Preserved unknowns

These remain unknown or unsupported:

- whether the supported invariant has a final stable repository name;
- whether future implementation should expose a shared translation audit or owner;
- whether additional modalities would follow the same movement;
- whether future provider families will separate identity shaping, predicate assignment, and observation construction more uniformly;
- whether all constitutional inquiry artifacts leave enough evidence for later machine checking;
- whether the invariant should become an operational diagnostic surface.

## Confidence

**High confidence** that recurring repository evidence supports preservation of bounded warranted constitutional content under provenance, role, authority limit, negative authority, preserved unknowns, and confidence/locality limits where applicable.

**High confidence** that grammar and representation are permitted to change during faithful constitutional translation.

**Medium-high confidence** that organization is not the preserved invariant, because recurring provider responsibilities exist but implementation organization remains uneven and provider-specific.

**Medium confidence** that "meaning" should remain unsupported rather than redefined, because the repository evidence is sufficient to answer in terms of warranted content and boundary without importing semantic frameworks.

## Final bounded answer

```text
What survives faithful constitutional translation
is not grammar,
not representation,
not assumed meaning,
and not organization.

What survives is bounded warranted constitutional content:
the supported identity, relation/value/inquiry condition,
with provenance, role, authority limit, negative authority,
preserved unknowns, and confidence/locality limits where supported.
```

Repository authority wins.
