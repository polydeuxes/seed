# Inquiry Need Projection Slice 001

This slice adds one read-only `InquiryNeedProjection` for explicit component-bounded repository/world uncertainty testimony.

The projection consumes testimony only when it names one exact selected goal, one exact bounded advancement horizon, one bounded uncertainty component, one repository/world subject, one owning stage, and evidence already present in the horizon snapshot set. It preserves inquiry standing separately from evidence freshness and evidence availability.

It refuses to infer inquiry need from generic Unknowns, observations, unsupported facts, stale or unavailable evidence, absent artifacts, or mixed unresolved material. Clarification, authority, and operational-realization components remain outside this inquiry classification.

The projection does not open inquiry, select questions, authorize observations, select actions, judge sufficiency, execute, record, write the event ledger, or mutate cluster state.
