# Typed Unknown Characterization

## Bounded question

## Recovery basis

This characterization treats candidate Unknown names as hypotheses only. It does not introduce a framework, implementation plan, slice recommendation, or campaign authorization.

## Recovered Unknowns

### 1. Preserved Unknown

**Identity — constitutional state preserved:** A known absence, limitation, ambiguity, or unresolved state is kept visible rather than converted into a guessed fact, inferred authority, implementation claim, or cluster truth.

**Repository evidence:** Boundary preservation work names unknown preservation as conservative retention of an unknown instead of unsupported inference, mutation, promotion, or resolution. It also concludes that unknowns can exist without visible boundaries and that not every boundary preserves an unknown. Inquiry artifact work identifies unknowns as stable continuity-bearing investigation artifacts. Runtime question recovery says answer surfaces preserve unknowns, unsupported targets, non-selected candidates, limitations, and boundaries rather than filling gaps with durable claims.

**Meaning — exactly what is not yet known:** The repository knows that something is not resolved, but the missing fact, authority, relation, or next move has not been established by implementation or repository evidence.

**Lawful next inquiry:** Continue inspecting the bounded surface that exposed the unknown and ask what evidence, if any, can lawfully resolve it without crossing the surface boundary.

**Resolution conditions:** Evidence that directly establishes the missing fact, authority, relation, or state inside the relevant bounded surface. If no such evidence exists, the unknown remains preserved.

**Constitutional status:** Preserved Unknown.

**Confidence:** High.

**Preserved unknowns:** Whether every preserved unknown belongs to a more specific recurring subtype remains unknown; repository evidence rejects universal compression.
