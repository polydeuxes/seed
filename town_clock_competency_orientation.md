# Town Clock Competency Orientation

Repository authority wins.

## Purpose

This artifact answers one bounded orientation question:

```text
What organism orientation
sits behind the Eye frontier,
where the Event Ledger acts
as the town clock
and bounded competencies
respond only to relevant
constitutional change?
```

The answer is orientation only:

```text
Seed may be oriented as a town of bounded constitutional citizens.

The Event Ledger is the town clock because each ledger tick marks recorded
constitutional change.

Each bounded competency remains locally responsible. It does not act because
clock time passed. It asks whether the newly recorded constitutional change is
relevant to its own bounded responsibility. If relevant, lawful movement may
become possible. If not relevant, lawful inactivity is the correct posture.
```

This does not introduce implementation, runtime behavior, public behavior, or
new repository authority.

Repository authority wins.

## Town Clock Orientation

The Event Ledger may be oriented as the town clock only in a constitutional
sense.

A town clock tick is not wall-clock time.

A town clock tick is not a timer.

A town clock tick is not a polling cadence.

A town clock tick is not a scheduler cycle.

A town clock tick means:

```text
A constitutionally relevant event has been recorded in append-only history.
```

Repository evidence already separates append-only event history from cached
projection, current lawful condition, verification, objective truth, runtime
behavior, and mutation authority. The ledger is projection and audit authority:
it preserves auditable transition input so current lawful projections can be
rebuilt and challenged.

In this orientation, the town hears the clock only because constitutional
history changed, not because time elapsed.

Repository authority wins.

## Competency / Citizen Orientation

A competency may be oriented as a citizen only if the metaphor remains bounded.

A citizen is not a universal actor.

A citizen is not an autonomous agent.

A citizen is not a planner.

A citizen is not an execution engine.

A citizen is a bounded local responsibility that can honestly say:

* what it owns;
* what evidence it can observe;
* what provenance it preserves;
* where its authority begins;
* where its authority ends;
* what movement it may support;
* what movement it must refuse;
* what unknowns remain;
* when lawful stop or lawful inactivity is required.

Competency interrogation evidence already treats competencies this way: a
competency is inspected through identity, authority, evidence, boundaries,
artifact handoff, unknowns, locality, continuity, and auditability. That
supports the citizen metaphor only as bounded local responsibility, not as a
new universal organism runtime.

Repository authority wins.

## Event-Ledger Tick Meaning

Each event-ledger tick means constitutional change has entered recorded history.

That change may be small. It may be an observation, evidence-bearing event,
fact-supporting event, diagnostic record, authorization event, or other lawful
recorded transition already supported by implementation and repository
boundary.

The tick does not mean:

* truth has been decided;
* facts have been verified;
* current lawful condition has automatically changed;
* every projection is current;
* every competency must act;
* host, cluster, or repository mutation is authorized;
* the organism has taken an autonomous step.

The tick means only that append-only history now contains one more recorded
transition input under provenance.

Projection, evidence, warrant, reliance, current lawful condition, and unknown
preservation determine what that tick may lawfully support.

Repository authority wins.

## Relevance-Check Discipline

After a town-clock tick, a bounded competency's lawful question is not:

```text
What should I do now?
```

The lawful orientation question is narrower:

```text
Is this newly recorded constitutional change relevant to my bounded
responsibility, authority, evidence, provenance, and stop conditions?
```

If the answer is unsupported, authority-blocked, out of scope, confidence-limited,
unknown, or irrelevant, the competency does not move.

If the answer is supported and locally relevant, the competency may perform only
bounded work already lawful for that competency.

This is orientation only. It does not define an implementation algorithm,
dispatch protocol, subscription model, dependency graph, queue, worker loop, or
notification mechanism.

Repository authority wins.

## Lawful Inactivity

Lawful inactivity is a positive constitutional posture.

A bounded citizen may remain inactive because:

* the tick concerns another responsibility;
* the subject is not bounded enough;
* evidence is insufficient;
* provenance is missing;
* authority is absent;
* negative authority blocks movement;
* confidence limits prevent reliance;
* the condition remains unknown;
* the current lawful condition is lawful stop;
* no local boundary crossing is supported.

Inactivity is not failure when movement would be unsupported.

A town of bounded competencies becomes more lawful by refusing irrelevant or
unsupported movement, not by maximizing activity.

Repository authority wins.

---

## Lawful Movement

Lawful movement remains local and evidence-gated.

When a tick is relevant, movement may occur only through already supported
constitutional disciplines:

* evidence remains attached to provenance;
* projection remains subordinate to event history;
* current lawful condition is not collapsed into raw history;
* warrant governs bounded reliance;
* reliance stays inside authority boundaries;
* responsibility movement preserves boundary, locality, unknowns, confidence,
  and lawful stop;
* observation and evidence do not silently become facts;
* diagnostics and visibility do not silently become cluster truth.

The citizen metaphor does not add new movement authority. It only preserves the
orientation that each competency responds, if at all, from its own bounded place.

Repository authority wins.
