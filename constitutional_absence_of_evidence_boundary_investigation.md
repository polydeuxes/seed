# Constitutional Absence of Evidence Boundary Investigation

## Bounded question

Across recurring repository evidence, when Seed cannot find, preserve, expose, retrieve, or derive support for a claim within a bounded review surface, what prevents Seed from treating that absence as proof that the claim is false, impossible, unsupported everywhere, contradicted, legally absent, historically absent, operationally absent, or nonexistent?

This investigation is exactly one bounded constitutional investigation of `Absence of Evidence != Evidence of Absence`. It is not world research, not legal advice, not public-world fact finding, and not implementation.

## App-visible evidence used

The app was used as bounded repository visibility, not as an oracle:

```text
python scripts/seed_local.py --diagnostic-inventory --json
python scripts/seed_local.py --diagnostic-shape-audit --json
python scripts/seed_local.py --knowledge-reachability-audit --knowledge-reachability-audit-limit 30
python scripts/seed_local.py --reasoning-path pressure capability --json
python scripts/seed_local.py --selection-path current_focus --json
python scripts/seed_local.py --source-navigation "absence evidence unknown support authority"
```

Observed app-visible boundaries:

- Diagnostic inventory reports `knowledge_reachability`, `reasoning_path`, `selection_path`, `diagnostic_inventory`, and `diagnostic_shape_audit` as non-recording, non-mutating surfaces.
- Knowledge reachability rendered all 30 evaluated sample candidates with first loss `not_preserved` under a truncated, limited audit. That is a bounded preservation loss report, not a global nonexistence finding.
- Source navigation reported the query status as `unknown` / `unsupported`, zero matching support facts, and explicit non-claims: no truth, runtime behavior, reachability, ownership authority, semantic relevance, or file parsing claims.
- Reasoning path and selection path exposed read-only boundaries and empty or bounded unknown lists for the selected query path; they did not turn absent derivation evidence into truth or disproof.

## Absence-of-evidence analysis

Repository evidence already gives several smaller, implementation-backed meanings for absence of preserved evidence:

1. **Reachability loss**: a candidate may fail at `Preserved`, `Projected`, `Read Model`, `Inquiry Orientation`, or `Rendered`. The app-visible reachability audit can say where a sampled candidate first disappeared, but its bounded and truncated shape prevents global absence claims.
2. **Unsupported projected fact subset**: unsupported facts are projected facts with no linked evidence. That is a concrete subset of missing evidence, not a universal missing-evidence taxonomy.
3. **No matching source-navigation support**: source navigation can report zero projected source facts for a query. Its implementation expressly limits the result to existing projected source facts/support and refuses truth, behavior, ownership, reachability, and semantic relevance claims.
4. **Unknown / insufficient / unsupported output**: Unknown is repeatedly preserved as a bounded non-promotion or lawful stop when derivation, selection, or support evidence is absent.

Therefore, absence of evidence currently means, at most: **within a stated surface, corpus, projection state, query form, and time of review, Seed did not expose or preserve support of the requested kind**. It does not mean false, nonexistent, globally unsupported, legally absent, historically absent, operationally absent, or harmless.

## Evidence-of-absence analysis

Repository evidence does not show a general evidence-of-absence model. It supports only bounded negative claims when the claim form itself is scoped to the reviewed corpus/surface/time window and the review preserves enough scope metadata to make the negative statement auditable.

Evidence of absence can be lawful only as a **bounded absence claim**, such as:

```text
No matching projected source-navigation facts were found for query Q in the current projected source-fact support surface.
```

That is not equivalent to:

```text
Q does not exist anywhere in the repository or world.
```

## Required distinctions

### 1. What repository evidence already means by absence of preserved evidence

It means a bounded support path did not carry the candidate to the relevant preservation/projection/view stage, or a surface found no matching records of the type it is allowed to inspect. It is loss or non-exposure inside a bounded mechanism.

### 2. What repository evidence already means by evidence of absence

Only a scoped negative result is supported: a particular bounded corpus/surface/query/time window did not expose matching evidence. A generalized evidence-of-absence doctrine is unsupported.

### 3. `not found` vs. `does not exist`

`Not found` is search-surface-local and depends on query form, index contents, projection freshness, corpus boundary, permissions, parser behavior, and output limits. `Does not exist` is an ontological/global claim requiring authority over all relevant corpora, time windows, source classes, and claim forms. Repository evidence supports the former far more readily than the latter.

### 4. `not preserved` vs. `false`

`Not preserved` means Seed lacks a preserved support path at the reviewed stage. `False` would require contradiction, disproof, or truth-adjudication authority. Repository evidence explicitly treats Unknown and missing evidence as not false/disproof.

### 5. `not exposed by this surface` vs. `unsupported everywhere`

A surface can expose only what its implementation, registry declaration, and source corpus allow. Source navigation, for example, uses projected source facts and does not inspect files during lookup. Therefore one surface's non-exposure is not evidence that all other surfaces, files, ledgers, histories, public sources, or future observations lack support.

### 6. Failed search vs. disproof

A failed search is an operational result of query, corpus, index, parser, and permissions. Disproof requires an evidentiary conflict or authoritative negative showing matching the claim form. Repository surfaces preserve unknown/unsupported rather than inventing disproof.

### 7. Missing contradiction vs. no contradiction

Missing contradiction means no contradiction was surfaced by the reviewed mechanism. No contradiction exists would require adequate coverage over relevant facts, claims, interpretations, time, and contradiction rules. The repository distinguishes conflicts and contradictions as integrity signals, not truth arbitration.

### 8. Missing authority vs. no authority exists

Missing authority means no authority was preserved or exposed in the reviewed path. No authority exists would require a complete authority search over the relevant authority domain. Repository authority-boundary evidence repeatedly warns that exposure of authority records is not itself authority and missing authority blocks promotion rather than proving global absence.

### 9. Missing harm evidence vs. no harm occurred

Missing harm evidence means the bounded surface did not preserve or expose harm support. No harm occurred is a historical/world claim. No reviewed app-visible surface has authority to decide that historical absence.

### 10. Missing event evidence vs. no event occurred

Missing event evidence means no matching event was found in the reviewed event/projection surface or the surface did not expose such evidence. No event occurred would require complete historical authority over the relevant event ledger, source streams, retention, time window, and external history. Projection-only surfaces are especially insufficient because historical source of truth remains the append-only EventLedger.

### 11. Missing implementation evidence vs. no implementation issue exists

Missing implementation evidence means current review did not locate implementation-backed support for an issue. It does not prove there is no issue. Implementation issue claims require concrete failing command, behavior, visibility gap, code path, test, or diagnostic evidence. Without that, lawful stop is investigation pressure, not implementation pressure.

### 12. Role of scope, corpus, coverage, time, authority, claim form, support path, confidence, and Unknown preservation

- **Search scope** states what was queried and how.
- **Corpus boundary** states what material was eligible for search.
- **Source coverage** states which source classes were included or excluded.
- **Temporal boundary** states when the search or projection was valid and whether history/as-of coverage exists.
- **Authority boundary** states what the surface may claim and what it must not claim.
- **Claim form** distinguishes scoped negative observations from global nonexistence claims.
- **Support path** preserves how evidence would travel from observation/source to evidence/fact/view/conclusion.
- **Confidence / evidence strength** limits how strongly the result may be stated.
- **Unknown preservation** keeps unresolved or insufficient conditions from being promoted into falsehood, disproof, recommendation, or implementation work.

### 13. When can absence become evidence for a bounded absence claim?

Only when the absence claim is itself bounded to the reviewed corpus/surface/time window and the review preserves query, scope, corpus, source coverage, temporal boundary, authority boundary, support path, and limitations. The lawful form is `no evidence was found in this bounded surface`, not `the thing does not exist`.

### 14. What must be preserved before saying `no evidence was found in this bounded corpus/surface/time window`?

Seed must preserve at least:

- exact query or claim form;
- searched corpus/surface;
- source classes included/excluded;
- projection/index/version or time of review;
- relevant limits/truncation/timeouts;
- matching rules or parser/extractor behavior;
- authority boundary and non-claims;
- result count and support path inspected;
- Unknowns, errors, and skipped material.

### 15. What must be preserved before stronger claims like `X is absent` or `X did not occur`?

A stronger absence claim would require repository evidence far beyond current app-visible surfaces: complete relevant corpus authority, complete temporal coverage, retention guarantees, reliable negative-query semantics, source coverage sufficient for the claim, contradiction/event/harm/authority rules appropriate to the claim type, and an authority boundary permitting that conclusion. Current evidence does not establish those conditions generally.

### 16. Lawful stop condition

The stop condition is: when support is absent only within a bounded review surface, preserve Unknown / unsupported / insufficient evidence and stop at the bounded negative observation. Do not promote absence into falsehood, disproof, global nonexistence, no contradiction, no authority, no harm, no event, or no implementation issue.

### 17. Does this investigation create implementation pressure?

No. It identifies boundary discipline already supported by repository evidence. It does not find a concrete failing command, behavior, diagnostic invisibility gap, or implementation owner requiring a slice.

### 18. Does repository evidence support implementation work yet?

No. Repository evidence supports a bounded constitutional distinction and continued investigation pressure only. It does not support an absence-detection schema, search-quality engine, truth engine, contradiction engine, legal research engine, public-world workflow, or implementation change.

## Bounded-corpus absence analysis

A bounded-corpus absence statement is lawful when its grammar remains inside the corpus:

- lawful: `No matching projected source facts were exposed by source navigation for this query.`
- unlawful overclaim: `No source facts exist anywhere.`
- lawful: `The reachability audit marked this sampled candidate first lost at not_preserved under this limit.`
- unlawful overclaim: `The candidate is not repository knowledge and cannot exist.`

The corpus must be named because the constitutional work is preventing a local non-match from becoming global ontology.

## Stronger absence-claim readiness analysis

A stronger absence claim needs stronger preservation than current surfaces generally provide. For `X did not occur`, Seed would need complete event-source authority and time-window coverage. For `no authority exists`, Seed would need complete authority-corpus coverage and source-of-law boundaries. For `no implementation issue exists`, Seed would need executable behavior checks and implementation coverage, not merely missing documentation or search hits.

## Provenance / corpus / scope / temporal-boundary analysis

The strongest repository pattern is provenance-bounded support. Evidence, fact support, source navigation, reasoning path, and selection path all depend on bounded inputs and preserve non-claims. Temporal boundary matters because current projections are not complete historical/as-of authority. A current surface may be truthful about current projection while silent about ledger history or external history.

## Authority-boundary analysis

Authority boundaries decide how far a surface may speak. Diagnostic inventory and shape audit visibility says a surface is registered and read-only; it does not make the surface semantically omniscient. Source navigation can expose source-fact support but refuses runtime, ownership, truth, and semantic relevance claims. Reasoning and selection paths can expose implemented derivation/selection lineage but do not record facts or mutate cluster state.

## Confidence / claim-strength analysis

Claim strength must track evidence strength:

- zero matches inside a bounded surface supports a low-strength scoped absence observation;
- unsupported projected facts support a concrete missing-linked-evidence subset;
- repeated source counts or packet inclusion do not prove independence, truth, authority, or absence elsewhere;
- confidence may increase only when coverage and authority increase, not merely because search returned no results.

## Unknown preservation

Unknown preservation is the main constitutional brake. The repository has an implementation-local typed Unknown owner that preserves `unknown_type`, `area`, and `reason` without resolving the finding. Reports characterize Unknown as bounded completion, non-promotion, lawful stop, inquiry non-settlement, and current uncertainty. That means missing evidence remains a preserved unresolved condition rather than becoming falsehood or disproof.

## Lawful stop

The lawful stop is reached when Seed can state the bounded negative result and its limitations, then refuses stronger promotion. The stop preserves:

- bounded negative observation;
- Unknown/unsupported/insufficient status;
- corpus and surface boundaries;
- authority non-claims;
- remaining unsupported conclusions.

## Supported conclusions

- Absence of preserved evidence is a bounded loss/non-exposure condition.
- Missing evidence is not disproof.
- Missing observation is not negative evidence and does not prove absence of an entity, relation, fact, or capability.
- Unknown is not false, failed, unavailable, or disproof.
- Source navigation non-matches are limited to projected source facts/support and carry explicit non-claims.
- Reachability `not_preserved` is a stage-local finding, especially when output is limited/truncated.
- Bounded absence claims are possible only when their scope, corpus, query, time window, authority, support path, and Unknowns are preserved.

## Unsupported conclusions

- Failed search proves falsehood.
- Not preserved proves nonexistence.
- One surface's non-exposure proves unsupported everywhere.
- Missing contradiction proves no contradiction exists.
- Missing authority proves no authority exists.
- Missing harm evidence proves no harm occurred.
- Missing event evidence proves no event occurred.
- Missing implementation evidence proves no implementation issue exists.
- Current repository evidence supports a general absence-evidence model or truth engine.

## Implementation recommendation

No implementation work is recommended. This investigation found no concrete repository failure requiring implementation. The supported outcome is boundary discipline: treat absence as investigation pressure only, preserve bounded Unknowns/unsupported results, and stop before converting local non-exposure into global absence, falsity, disproof, or implementation pressure.
