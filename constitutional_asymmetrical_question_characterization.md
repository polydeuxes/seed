# Constitutional Asymmetrical Question Characterization

Repository authority wins.

## Bounded question

```text
According to recurring
repository evidence,

what makes a constitutional
question asymmetrical?

Does a constitutional
investigation impose
structure upon reality,

or does it allow reality
to reveal its own lawful
organization?
```

This is exactly one bounded Constitutional Investigation Characterization. It does not introduce an investigative framework, methodology engine, universal questioning system, or new definition of repository authority.

## App visibility used

The app was used only for repository visibility:

```text
python scripts/seed_local.py --documentation-structure --recurrence --summary-only --limit 20
```

The app reported 577 documents, 11,209 distinct section-label entries, 1,157 recurring section-label entries, recurring front matter keys, recurring heading depths, recurring code fence language classes, 167 internal docs links, 3 external links, and 0 broken local docs links.

Its boundary remained read-only structural recurrence only: no prose interpretation, claim extraction, authority inference, shape inference, ontology promotion, event-ledger writes, or repository mutation.

Therefore app visibility supports that recurring document structure exists. It does not itself prove asymmetry, constitutional authority, investigation truth, or a question framework.

## Short answer

Recurring repository evidence supports the orientation:

```text
A good constitutional question
is asymmetrical.

It does not force structure
onto reality.

It allows reality
to reveal its own
lawful organization.
```

The support is not because the repository has a universal questioning method. It is because recurring investigations begin from a bounded question or candidate model, then submit that model to repository-visible evidence. When evidence contradicts the investigator's expectation, the supported result changes, narrows, splits, stops, or preserves unknowns rather than preserving the initial model.

In repository terms, a constitutional question is asymmetrical when the question is allowed to lose to evidence.

## What makes the question asymmetrical

A constitutional question is asymmetrical when it has these recurring properties:

1. **It is bounded before it is answered.** The investigation asks one answerable question and excludes adjacent ambitions.
2. **It treats its own vocabulary as suspect.** Repeated wording, attractive concepts, prompt nouns, and presentation labels are not allowed to become repository knowledge without evidence.
3. **It allows implementation and repository artifacts to choose the shape of the answer.** The answer may be narrower, differently named, differently ordered, split into multiple families, or stopped as insufficient evidence.
4. **It preserves contradiction as evidence, not failure.** Counterexamples and unsupported conclusions are kept visible.
5. **It terminates at the lawful support boundary.** The result may be supported, candidate, frontier, Null, unknown, or insufficient implementation evidence.

That asymmetry is not a new method. It is the recurring constitutional posture already stated by the repository: Null persists until evidence justifies transition; observation precedes inquiry; only evidence moves Null; repository-visible implementation, tests, reports, diagnostics, observations, and compatibility behavior are the court of evidence; and ownership requires recurring implementation evidence.

## Recurring repository evidence

### Constitutional evidence

The Constitution states that it is an orientation artifact and that implementation evidence wins if conflict appears. It gives the central discipline as movement from Null/unknown through observation, bounded unknown, inquiry, evidence, supported transition or explicit stop, and bounded handoff artifact.

The same document says inquiry does not begin from global purpose, planner intent, architectural preference, or framework completion. It begins when an observation creates a bounded unknown. It also says unsupported evidence must leave the subject uncommitted, and that insufficient implementation evidence is a valid successful result.

This is the constitutional basis for asymmetry: the investigator's model does not govern the repository; evidence governs movement.

### Investigation grammar evidence

`constitutional_investigation_grammar_characterization.md` reviewed recurring investigations and found that constitutional grammar is a repository investigation discipline, not a prompt/runtime mechanism, planning mechanism, scheduling mechanism, registry mechanism, canonical specification, or implementation requirement.

Its inspected cases repeatedly show the same non-imposing pattern:

- Competency interrogation asks whether a proposed sequence is earned rather than assuming it; it rejects the exact ordered sequence as mandatory and refuses an interrogation engine.
- Bounded inquiry recovery reduces broad architectural possibility into one evidence-backed bounded question or stops it as unsupported.
- Survey warrant recurrence recognizes recurrence without converting survey behavior into taxonomy, framework, or implementation plan.
- Inquiry competency district limits the result to district resolution and refuses a framework or ownership recovery.

This supports asymmetry because the question does not install its proposed shape. It asks whether the repository has already earned that shape.

### Investigation viewpoint evidence

`constitutional_investigation_viewpoint_characterization.md` characterizes recurring investigation viewpoints without promoting operator names, a canonical taxonomy, mandatory sequence, planner, scheduler, registry, or resolution engine.

It identifies recurring postures that preserve asymmetry:

- Recurrence-recognition separates observed recurrence from authorized constitutional claims and refuses ontology promotion from repeated wording.

This supports the statement because a good constitutional question is not symmetrical negotiation between model and evidence. Evidence has priority.

This contradicts a model that would impose one universal subject or dependency vocabulary. Repository evidence reveals local identities with different validation, routing, evidence-selection, output, and authority boundaries.

### Provider-to-observation evidence

`provider_grammar_to_observation_grammar_investigation.md` supports recurring transitions from provider grammar into selected, lower-authority Seed observation grammar in strong cases, but it also preserves counterexamples. Prometheus and systemd do not prove a universal three-stage grammar shift: Prometheus constructs canonical observations directly from validated provider payloads, and systemd uses provider-shaped helper dictionaries before observations.

This is asymmetry in practice. The proposed shape is not forced across all sources. The recovered structure is mixed: some adapters show intermediate structural records, some do not.

### Roadmap evidence

`seed_competency_roadmap_v2.md` revises a prior roadmap because recent investigations sharpened the boundary. It replaces or refines prior terms: Authority Interpretation becomes Evidence Interpretation, Surface/Question Routing folds into Inquiry Navigation, Bounded Responsibility Understanding becomes Bounded Work Recovery, and Bounded Answer Explanation becomes Bounded Answer Composition.

The roadmap states that Seed should not begin from architectural nouns, ownership labels, or review ambitions. It should acquire competencies in the order required to ask an evidence-bearing question, find authority-bearing evidence, interpret what evidence can and cannot support, recover bounded work only where implementation evidence permits, evaluate responsibility claims, and compose a bounded answer.

This is direct evidence of mental-model correction. The prior roadmap was preserved, but repository evidence changed the sequence and vocabulary.

## Recurring investigation movement

The examined movement is supported, with one repository-specific refinement.

The prompt's candidate movement was:

```text
Mental model
↓
Question
↓
Repository evidence
↓
Recovered structure
↓
Mental model corrected
```

Recurring repository evidence supports this as a descriptive movement when the starting point is a candidate model, phrase, roadmap, or expectation. The repository-specific version is:

```text
Observation or candidate model
↓
bounded question
↓
repository-visible evidence
↓
supported structure, split structure, explicit stop, or preserved unknown
↓
prior model corrected, narrowed, rejected, or left uncommitted
```

The refinement matters. Some investigations do not begin with a strong mental model; they begin from observation, pressure, or a bounded unknown. Also, the recovered result is not always structure. Sometimes the lawful result is insufficient implementation evidence, a frontier, a Null condition, a candidate-only status, or preserved unknowns.

## Recovered constitutional invariants

1. **Evidence priority.** Repository-visible evidence outranks attractive concepts, repeated vocabulary, and investigator expectation.
2. **Boundedness.** A constitutional question succeeds by answering one bounded question, not by expanding into a framework.
3. **Non-promotion.** Presentation labels, roadmap nouns, and repeated terms do not become knowledge without implementation or repository evidence.
4. **Contradiction preservation.** Counterexamples are retained and may split or narrow the conclusion.
5. **Locality.** Recovered organization is often local to surfaces, adapters, families, or artifacts rather than global.
6. **Stop legitimacy.** Insufficient evidence, unsupported conclusion, frontier, and unknown are lawful outcomes.
7. **Authority separation.** Visibility is not authority; diagnostic output is not cluster truth; projection is not verification; answer is not mutation; frontier is not implementation readiness.
8. **Correction over confirmation.** A good constitutional investigation is willing to have its initial formulation corrected by repository authority.

## Unsupported explanations

The reviewed evidence does not support concluding that:

- Seed has a universal constitutional questioning system.
- Asymmetry is an implemented engine, planner, dispatcher, scheduler, or runtime method.
- Every investigation must follow an exact ordered section sequence.
- Every good question begins from a mental model rather than observation or bounded unknown.
- All recurring vocabulary names stable constitutional knowledge.
- The repository always reveals a positive structure; sometimes it reveals insufficiency, Null, frontier, or unknown.
- The statement `a good constitutional question is asymmetrical` authorizes future conclusions without evidence.

## Preserved unknowns

- Whether future investigations will stabilize a more precise term than `asymmetrical question` remains unknown.
- Whether asymmetry should become any implementation-visible diagnostic, inventory row, audit surface, or question family remains unknown and unsupported here.
- Whether all historical investigations fit the movement remains unknown; this characterization reviewed recurring evidence, not every artifact exhaustively.
- Whether repository-neutral review will eventually require additional questioning disciplines remains unknown.
- Whether a future implementation can operationalize any portion of this posture without violating evidence priority remains unknown.

## Confidence

**High** that recurring repository evidence supports the constitutional orientation that a good constitutional question is asymmetrical in the limited sense recovered here: it asks a bounded question and lets repository evidence correct, split, narrow, reject, or stop the investigator's model.

**Medium** that the exact phrase `A good constitutional question is asymmetrical` should be reused as repository vocabulary. The evidence supports the orientation, but not a canonical vocabulary system.

**Low** that this should become implementation behavior, a diagnostic surface, or a universal methodology. No reviewed evidence supports that promotion.
