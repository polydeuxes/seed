## Recovered role of Null

### What Null owns

Repository evidence supports Null as the default constitutional commitment posture for subjects that have not yet been moved by evidence. The Constitution says a subject remains Null until observable evidence justifies movement and that this applies to claims, facts, observations, ownership, pressure, inquiry, classification, authority, compatibility, and recovery. It also says no competency bypasses Null: notes, provider responses, reports, audit rows, code corridors, compatibility residue, and operator concerns may orient work, but do not become repository truth merely by appearing.

Null therefore owns the following constitutional responsibilities:

1. **Default non-commitment.** A subject starts uncommitted with respect to repository truth, ownership, authority, classification, and recovery.
2. **Evidence gate.** Movement out of Null is lawful only through repository-visible evidence.
3. **Stop legitimacy.** Insufficient implementation evidence is a valid successful result, not a failure to continue.
4. **Boundary preservation.** Null prevents notes, pressure, reports, observations, candidates, and presentation vocabulary from becoming knowledge, authority, implementation, or runtime behavior without an implementing boundary.
5. **Current lawful condition.** `constitutional_state_reconciliation.md` distinguishes the timeless rule, `Only evidence may move Null`, from current evaluations such as `Subject X is currently Null / unknown / uncommitted` or `evidence is insufficient for this claim`.

This is more than absence. Absence would be no subject and no lawful posture. Null is a named, recurring, authority-bearing posture that preserves a subject from unsupported movement.

### Null distinguished from absence

Null is not simple absence because the repository can identify a subject as uncommitted and preserve a lawful stop. The Constitution treats Null as a status that persists until evidence justifies transition, and the state reconciliation treats `subject is Null` as a current lawful condition under a timeless rule. Absence has no bounded subject to evaluate; Null has a subject whose movement is constrained.

### Null distinguished from startup and initialization

Repository evidence does not identify Null with process startup, runtime boot, activation, or initialization. The Constitution explicitly says it is not runtime behavior and does not create a new implementation family. The physiology architecture likewise treats behavior as the visible outcome of physiology under constitutional constraint, not as startup orchestration. Null-first evidence begins from a minimal input probe and preserved inquiry note, not from a runtime lifecycle.

### Null distinguished from unknown

The repository often pairs `Null / unknown / uncommitted`, but it also preserves distinctions:

- `unknown` can be an output classification in many implemented surfaces, such as unknown question families, unknown diagnostic surfaces, unknown entity types, or unavailable/unknown authority.
- `Null` is the constitutional commitment posture that prevents unsupported promotion.
- `constitutional_state_reconciliation.md` treats `Null / unknown / uncommitted` as overlapping condition language while warning that timeless rules and current status statements should not be collapsed.

Thus unknown is often an implementation-visible result or data status; Null is the constitutional non-promotion posture governing whether movement has been earned.

### Null distinguished from unavailable capability

Unavailable or unknown capability is a bounded implementation result, not equivalent to Null. Capability-related investigations show candidates, verification support, inventory status, and promotion readiness without permission or execution authority. Service/listener authority surfaces also distinguish unavailable local authority from unknown external provider authority. Null is broader: it governs whether claims about capability, authority, ownership, and promotion may move at all.

### Null as constitutional condition

Implementation evidence supports Null as a recurring constitutional condition. The strongest support is not a single module named `Null`, but repeated repository behavior:

- Constitution: subjects remain Null until evidence moves them.
- Null-first transition audit: the first supported movement for `.` is raw inquiry-note preservation only.
- Transition-family reports: orientation, eligibility, evidence, fact/admission, implementation, and methodology transitions have different authority effects.
- State reconciliation: Null can be a current lawful condition under constitutional law.
- Conservation and promotion reports: candidate, support, readiness, visibility, and pressure do not automatically become authority.

## Physiological readiness

Repository evidence supports recovered physiological competencies only as bounded competencies or characterization-level physiology, not as a completed runtime organism. The competencies that appear necessary before a constitutional organism could leave Null are the ones the repository repeatedly requires for lawful movement from uncommitted condition to behavior:

1. **Observation / preservation.** Observation precedes inquiry and creates orientation, not truth. Null-first evidence shows raw note preservation can be the first supported movement without becoming Observation, Evidence, or Fact.
2. **Bounded unknown formation.** Inquiry begins when observation creates a bounded unknown; unsupported targets remain Null or unknown rather than being inflated.
3. **Question eligibility.** Question surfaces require exact family lookup, eligibility/status checks, required arguments, dispatch mapping, and local stopping for unknown, diagnostic-only, or non-dispatchable surfaces.
4. **Evidence interpretation / evidence gating.** Evidence is the only legitimate mover of Null, but evidence interpretation remains boundary-local; support does not automatically become promotion.
5. **Observation Agreement.** Observation Agreement can preserve exact textual agreement between independent supplied observation streams, with provenance, but cannot decide truth, meaning, architecture, grammar, responsibility, or repository knowledge.
6. **Pressure visibility.** Pressure may orient inquiry but does not command work, recover ownership, prove implementation readiness, or select priority.
7. **Responsibility evaluation.** Responsibility work requires owner, inputs, outputs, exclusions, handoff, authority, and stop evidence; it is not supplied by agreement-looking text alone.
8. **Conservation / non-promotion.** Candidate, verified, readiness-supported, visible, pressured, compatible, and frontier states remain distinct from authority-bearing admission.
9. **Stopping pressure / lawful stop.** The Constitution and physiology reports repeatedly treat stop conditions as necessary protection against unsupported transition.
10. **Artifact handoff.** Competencies exchange bounded artifacts, not shared hidden state. Handoffs must expose observation, evidence, conclusion, authority, and stop condition.

These are necessary-looking only because the repository repeatedly uses them as movement guards. The repository has not proven they are sufficient for leaving Null as a living organism.

## External surveyor responsibilities

The repository preserves several responsibilities outside the organism. These responsibilities are performed by surveyors, Codex sessions, architectural recovery, competency interrogation, and slice methodology rather than by Seed runtime physiology.

### Surveyors

Surveyors presently own external recovery observation: reading the repository, selecting implementation evidence, comparing reports, identifying missing boundaries, and writing characterization artifacts. Their outputs orient later work but do not themselves become authority unless implementation evidence supports them.

### Codex

Codex presently acts as an implementation worker and recovery author. The Constitution permits Codex, assistants, operators, tools, diagnostics, and Seed competencies to exchange bounded artifacts rather than hidden shared state. Codex reports can become next-inquiry prompts, but they are not runtime behavior, autonomous selection, or repository truth by authorship alone.

### Architectural recovery

Architectural recovery owns bounded characterization of owners, responsibilities, authority boundaries, compatibility residues, and stopping points. It can identify pressure and recovered structure. It does not itself create runtime families, planners, schedulers, activation logic, or implementation authority.

### Competency interrogation

Competency interrogation owns the method for asking what a competency is, what it can observe, where it begins and ends, what authority it appears to possess but lacks, and what unknowns remain. The methodology reconciliation explicitly says no runtime physiology has been earned by the methodology itself.

### Slice methodology

Slice methodology owns bounded implementation recovery when evidence supports a smallest behavior-preserving change and tests can preserve it. It remains outside the organism because it is a repository-development practice: selecting slices, applying patches, preserving diagnostics, and proving behavior with tests. It is not an autonomous runtime planner.

## Organism responsibilities already suitable for direct runtime physiology

Repository evidence supports the following as candidate organism physiology, but only in the bounded sense of capacities that transform lawful conditions into response tendencies. This characterization does not claim they are implemented as a runtime physiology subsystem.

- **Receptivity / preservation.** Lawful signals can be preserved without promotion.
- **Observation orientation.** Observation creates orientation and possible bounded unknowns without creating truth.
- **Candidate question formation.** Physiology reports treat candidate-question formation as a physiological capability when uncertainty exists, while preserving discard and stop gates.
- **Question eligibility gating.** Implemented question-surface inventory evidence supports exact family/status/argument gating before dispatch.
- **Evidence gating.** Evidence can support movement, while insufficient evidence stops movement.
- **Agreement preservation.** Observation Agreement can preserve exact candidate agreement without semantic promotion.
- **Pressure sensing.** Pressure Audit can expose pressure and inspection references without priority, command, or planning authority.
- **Responsibility discrimination.** Responsibility evaluation can ask for owner/input/output/exclusion/handoff/test evidence before assigning responsibility.
- **Commitment inhibition.** Candidate, visible, readiness-supported, and pressured states do not automatically become truth, permission, or implementation.
- **Stopping pressure.** When every continuation path would exceed support, lawful stop can become the visible behavior.

The physiology architecture states this cleanly: constitution establishes lawful conditions; physiology converts them into tendencies such as preservation, curiosity, candidate formation, evidence gating, clarification readiness, and stopping pressure; behavior is the visible result. Repository authority still limits this to characterization until implementation earns a runtime owner.

## Constitutional transition evidence

Implementation evidence supports a recurring constitutional progression:

```text
Null / unknown / uncommitted
  -> observation or preserved signal
  -> bounded unknown / orientation
  -> bounded inquiry or explicit stop
  -> evidence-bearing result or explicit stop
  -> supported local transition, handoff, or continued non-promotion
  -> visible behavior only where a bounded surface owns it
```

This progression recurs across constitutional text, inquiry orientation, question eligibility, transition-family characterizations, pressure audit, observation agreement, promotion readiness, responsibility recovery, and physiology characterization.

However, evidence does **not** support collapsing this progression into one generic constitutional transition or one runtime lifecycle. `constitutional_transition_family_characterization.md` explicitly rejects one generic constitutional movement and instead identifies different transition families with different effects and authority boundaries. `architectural_physiology_characterization.md` recognizes physiology as recurring bounded operation while rejecting a physiology subsystem, runtime owner, planner, scheduler, lifecycle engine, workflow runtime, or autonomous execution model.

Therefore the relationship is:

```text
Null
  -> recovered physiology as recurring bounded movement and guarded competencies
  -> possible living constitutional organism only if future implementation earns runtime ownership
```

The first two steps are supported. The final step remains guarded.

## Counterexamples and permanent-boundary candidates

Important constitutional responsibilities remain impossible to internalize on current evidence. These do not prove a permanent metaphysical boundary, but they do define the current constitutional boundary between surveyors and organism.

1. **Repository recovery.** Current recovery depends on external reading, comparison, characterization, and implementation-worker judgment. The repository has not implemented a self-recovery organism that can select and execute architectural recovery.
2. **Architectural evolution.** Roadmaps and frontiers preserve future orientation but do not authorize execution, scheduling, governance, or workflow. Architectural evolution remains surveyor/operator/Codex-mediated.
3. **Constitutional promotion.** Promotion authority remains bounded by repository-visible evidence and local admission rules. Capability promotion readiness can be supported while still not creating verified facts, writing events, selecting capabilities, invoking tools, or authorizing execution.
4. **Slice planning.** Implementation readiness and slice methodology remain external development practices. Pressure can start readiness investigation but cannot finish readiness or select implementation.
5. **Competency interrogation methodology.** The methodology can recover competency boundaries and negative authority, but it remains a notebook/recovery practice rather than runtime physiology.
6. **Runtime generalization.** Recent physiology reports explicitly reject workflow engine, state machine, planner, scheduler, lifecycle manager, autonomous reviewer, semantic question generator, general evidence pipeline, and general transition runtime claims.

These are current boundary facts. Whether they are permanent is unsupported. Repository authority only supports saying they remain outside the organism now.

## Supported conclusions

1. **Null is not merely an implementation placeholder.** It is a recurring constitutional non-commitment condition and evidence gate.
2. **Null is not absence.** It applies to identifiable subjects whose movement is constrained by evidence.
3. **Null is not startup or initialization.** No reviewed evidence ties Null to runtime boot or activation; the Constitution explicitly rejects runtime behavior claims.
4. **Null is broader than unknown/unavailable.** Unknown and unavailable often appear as implementation statuses; Null is the constitutional posture preventing unsupported promotion.
5. **Recovered physiology exists as characterization-level recurrence.** The repository has recovered recurring capacities: observation, bounded unknowns, eligibility, evidence gating, agreement preservation, pressure visibility, responsibility discrimination, conservation, handoff, and lawful stop.
6. **The repository has evidence of developmental progression.** Null can be followed by recovered physiological competencies and visible behavior, but only through bounded evidence and local authority.
7. **The organism has not earned runtime physiological autonomy.** Current evidence rejects treating physiology as a subsystem, planner, scheduler, lifecycle, workflow, activation framework, or autonomous recovery owner.

## Unsupported conclusions

The following conclusions are not supported by repository implementation evidence:

- Seed is constitutionally ready to leave Null.
- Null is a startup state, boot phase, initialization status, activation mode, or runtime lifecycle state.
- Recovered physiology is already an implemented runtime subsystem.
- A generic transition engine exists for `Null -> readiness -> behavior`.
- Pressure, roadmap, frontier, or operator concern selects work.
- Promotion readiness is promotion.
- Observation Agreement, Grammar Observation, or repeated vocabulary proves semantic truth.
- Surveyor methodology can now be internalized into organism runtime.
- Architectural recovery, slice selection, competency interrogation, or constitutional promotion can be performed autonomously by Seed.
